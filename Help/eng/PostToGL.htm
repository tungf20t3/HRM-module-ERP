Post transactions to general ledger

Created vouchers will be transferred to General Ledger (GL) to be kept in accounting books.
To businesses with simple accounting system, vouchers can be posted to ledger right after created by selecting status “Posted to GL”.
To businesses having vouchers transferred to general accounting, it can be directly processed in voucher by opening each voucher. However, it will takes much time in bookkeeping if the amount of vouchers is so large.
“Post Transaction” function helps to post mass of vouchers in “Release Later” status created in other modules to general ledger.
In reality, this function is usually used to limit access right to voucher statuses. Therefore, functional accountants can only change vouchers to ‘Release Later” status, general accountants will post these vouchers to general ledger.

“Post Transaction” screen

Common operations

Edit voucher

To edit voucher, select the required voucher and click - “Edit voucher”, the selected voucher will be opened as the entry screen.

Select voucher

In “Post Transaction” screen, there are vouchers in “Release Later” status. To post details of each voucher to ledger, tick the -“Select” checkbox option in front of the required voucher and click -“Post to GL”.

Note
To use this function, it is required the reasonable access right assignment by function and unit.
If this function is not used, it is required to assign the posting right to functional accountants so that they can post the created vouchers to general ledger during voucher entry.
If this function is used, it is required to assign the right to change “Release Later” status to vouchers (“Waiting for Approval” status to General Voucher) and not to assign the posting right to functional accountants.
It is required to assign “Edit” right by unit so that a user without right to post a voucher to GL but with the right to edit voucher by unit and assigned “Post Transaction” function, he/she can completely perform the posting as to be assigned by voucher status.
