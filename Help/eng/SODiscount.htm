Discount List

Path: Modules/ Sales Management/ List Maintenance/ Discount List
Discount list definition supports in setting up detail discount codes in sales.
According to the discount type (in 13 listed types) defined in discount information, there will be detail information for data entry in discount detail entry.

For example:
If discount type of the discount is “Site”, it is required to define site information and discount rate in discount detail entry. If discount type of the discount is “Site and Item Price Class”, it is required to define site information, item price class and discount rate in discount detail entry.

Field description

Discount Type

Discount type shows to which discount object/group the discount is applied such as customer, item, site, quantity. There is now an available list of 13 types in the system, un-editable.

Priority

Priority information shows if the customer is given many discount types in one same transaction, which one will be applied in priority.

Flat Price Discount

This checkbox option, when activated, indicates that discount amount in transaction will be directly extracted from Discount List. So it is required to input the discount amount in Discount Details. If this option is not activated, discount amount will be calculated by discount rate.
This option is applied only when discount code is one of follows:

    - Item Code
    - Site Code & Item Code
    - Customer & Item
    - Customer Price Class & Item
    · Applied to Extended Amt.

This checkbox option, when activated, indicates that discount amount is directly deducted on sales price. Then, amount in SO will be calculated by formula:
Amount=(Standard Unit Price * Quantity) – Discount amount
If this option is not activated, amount in SO will be calculated by formula:
Amount=(Standard Unit Price – Discount amount) * Quantity
Discount amount can be directly input or calculated by discount rate (depending on “Flat Price Discount” option).

Added Discount

This checkbox option, when activated, indicates that in one transaction, the customer is given many types of discount, the 2nd and next discounts will be added to the 1st one if they satisfy the discount condition.
If it is not activated, the 2nd discount type will not be considered when SO has already been applied the 1st one.

Apply to Remaining

This checkbox option, when activated, indicates that in one transaction, the customer is given many types of discount, the 2nd discount type can be applied to the remaining amount after deducting the 1st discount amount (in the priority of discount type). This option is available if “Added Discount” option is activated.
If this option is not activated, the 2nd discount amount will be calculated in total amount and added to the previous one.

Effective from/ to

Effective date shows the period in which the discount is effective. The system will compare this definition with the “Effective to” in Discount Details to calculate discount amount.
