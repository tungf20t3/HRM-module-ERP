Loan Contract List

Loan Contract is created when there is a credit contract with bank or a loan with other parties. There must be declaration of fields used in voucher list of related voucher to make loan contract reports.

Field description

Contract Code

A contract code is unique for one disbursement of the bank.

Contract Date/ Number

Contract date is the date when the loan contract is created. Contract number is used to record vouchers controlled by loan contract. This field is usually used together with loan contract code. “Contract Number” is default as “Contract Code”.

Contract Type

There are 2 types: 1 - Borrow, 2 - Lend. In case contract type is 2 - Lend, the payment will be recorded in credit and loan amount will be recorded in debit of account receivable. Contract type is used in filter condition of following reports:

    - Transaction List by Loan Contract;
    - Summary Report by Loan Contract;
    - Loan Contract Balance;
    - Loan Contract Opening Balance;
    - Loan Contract Closing Balance;
    - Loan Contract List;
    - Interest Sheet Detailed by Loan Contract;
    - Loan Contract Status Detailed Report;
    - Loan Contract Status Summary Report;
    - Loan Payment Term Tracking Report;
    - Loan Contracts with Interest Exceeding Permissible Rate.

Balance Tracking

This option defines whether the system will keep tract of contract balance or not. If you select balance tracking, contract balance will be copied to next year, the contract opening balance entry is enabled and vice versa.

Currency Code

The monetary unit used in loan contract

FC Loan Amount

The value of loan contract in the currency used in that contract

Loan Amount

The value of loan contract exchanged to base currency

Loan Date

Default as “Contract Date” but editable

Due Date

The date when the loan contract has to be paid, used in “Interest sheet detailed by loan contract”
The interest paid after the due date is overdue interest.

Max Interest Rate (%)

The maximum rate of interest used in “Loan contracts with interest exceeding permissible rate”
This is reference report to define reasonable expenses in corporation income tax finalization.

Payment Cycle

The period between payment times of loan contract; used in “Loan payment term tracking report”

Number of Payments

Number of payment times by payment cycle

Loan Account/ Bank

The account used to record loan and the bank giving loan; bank is extracted from customer list; loan account must be sub-account, extracted from chart of accounts.

Department

Sales department and the department executing the loan contract; department is extracted from department list, reference information.

Contract/ Job

Purchase contract and job in which loan is used and reported in “Loan contract list” and “Interest sheet detailed by loan contract”.

“Interest Rate” – Field descriptions

Interest Rate (%)

Interest rate applied at each time when comparing due date and effective date of loan contract

Effective Date

Effective date, loan data and due date are bases to define in due and overdue interest rates.

For example: Loan contract ABC, loan amount 100.000.000, loan date 01/01/2009 and due date 31/01/2009

Result (Unit: 1,000,000)

“Other” – Field descriptions

Group 1/2/3

Groups are used to classify loan contracts supporting data analysis. Groups are extracted from “Loan contract group list”.

Some tasks in list maintenance

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.
The data importing process:

    - Enter Loan Contract List;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to the List.

Notes of inputting data in template file:
- Contract Code: in form of string with 8 characters maximum. One code is unique in this file. If some code already exists in List, the system will replace by one in the template file (if the copy type is "Overwrite"). The new codes are certainly added to List. Contract code cannot be nested each other or consist of system master or operator characters: '`~!@#$%^&*(),;"<>?/,_;
- Contract Number: manually input with 16 characters maximum;
- Contract Type: accepted 1 in 2 options (1 - Borrow or 2 - Lend);
- Balance Tracking: accepted 1 in 2 options (1 - Yes or 0 - No);
- Contract, Job, Currency Code, Loan Account, Bank, Department: must be one existing in the corresponding list (contract, job, currency, customer, department and chart of accounts) with the length in turn: 16, 16, 3, 16, 16, 16 characters maximum;
- Loan Amount: number format, 14 digits and 2 decimal digits maximum;
- FC Loan Amount: number format, 12 digits and 4 decimal digits maximum;
- Payment Cycle, Number of Payments: number format, 3 digits maximum;
- Max Interest Rate (%), Interest Rate (%): number format, 12 digits and 4 decimal digits maximum;
- Date, Loan Date, Due Date: date format;
- Contract Description, Other Name: user-defined information, 128 characters maximum.

Other notes:
- If you do not have the "New" right in "Loan Contract List", you cannot import new data from spreadsheet to this list;
- If you do not have the "Edit" right in "Loan Contract List", you cannot edit data in this list from spreadsheet;
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
