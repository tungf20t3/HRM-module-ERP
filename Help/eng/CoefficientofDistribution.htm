Set up Coefficient of Distribution

Costs which can not be detailed to each product when entering must be distributed to aggregating objects by coefficient of distribution.
Program supports the following ways of distribution:

    - Distribute by BOM towards cost related to raw materials;
    - Distribute by arising of other cost factor calculated; this way is used for cost related to value.

Besides, if coefficient is not in those 2 options, it must be set up by the most possible cost aggregation unit. We now consider what the “cost aggregation unit” is.
The most general and smallest cost aggregation unit of program is combination of [Product Code + Dept. + Cost Factor Code + MO]; it means that costs will always be detailed to this combination in any case. In special case, if there is less than 1 element of combination in aggregation, it means that field is “NULL”.

Entry Screen

Field Descriptions

Period / Year

Period / Year are closing time, defined by order from the beginning of fiscal year.

Factor Code

Code of cost factor having coefficient of distribution

Product

Code of product having coefficient of distribution

Department

Code of production line, workshop in case product WIP by line, workshop. This is a required field if cost is calculated by department in system parameter.

MO Number

Number of MO WIP in case product WIP in MO. This is a required field if cost is calculated by MO in system parameter.

Indirect Dept.

It is used as intermediary section to distribute to other sections.

Item Code

It is used in case there is no BOM; items are distributed by density in coefficient table.

Important note:
Selecting objects to be updated coefficient of distribution depends on cost factor declaration. If there is no declaration of cost factor aggregation types, program will not help to extract data from that field when entering coefficient.
Coefficient updated by material is used only when there is no BOM. If cost factors have coefficient defined by ratio but recalculated by BOM, the 2nd result will be used.

To save hours of manual effort each month, the system enables you to copy the declaration of previous month. When opening this menu, in filter screen, select “Copy” field is 1 - Copy data from previous period.

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

The data importing process:

- Enter "Setup Coefficient of Distribution" menu;
- Click the icon to download template file to put data in;
- After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file into the system.

Notes of inputting data in template file:
- Factor Code, Item Code, Product, Department, MO Number: must be one existing in the corresponding lists (Cost Factor List, Item List, Section List, Manufacturing Order) with the same length in add (edit) screen in list;
- Department, MO Number: are optional fields depending on structure of the costing problem;
- Indirect Department: must be one existing in Section List, which is not checked in "Direct" option;
- Coefficient: number format, 12 digits and 4 decimal digits maximum (standard format);
- Product, Item Name: user-defined information, 128 characters maximum.

Other notes:
- Cannot import data after system data closing date (the data closing date after the beginning input date);
- The current user must have the "Edit" right in this function to update the available data in the system (Copy type 1 - Overwrite);
- The current user must have the "New" right in this function to add new data to the system (Copy type 1 - No);
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
