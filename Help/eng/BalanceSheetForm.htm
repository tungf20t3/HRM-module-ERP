Report Form Definition – Balance Sheet

To each report form, the articles can be defined by the “Create report form” function of that report.

“Filter Condition” screen

Field descriptions

Report Form

Enable to select the report form for definition, lookup from report form list

Type

There are 3 types: Type 1 - Create, Type 2 - Edit, Type 3 - Delete

    Type 1 - Create: Enable to create new report form to satisfy business management requirement
    Type 2 - Edit: Enable to edit the selected report form: add, edit, delete articles in report
    Type 3 - Delete: Enable to delete unnecessary report form, check the “Confirm delete” option to delete

Report Name

The name of the report form

Other Name

The name of the report form in English

Information

Reference information of report such as form number, date and place issued

Other Information

Reference information of report in English

Confirm delete

Check-box option to confirm to delete the report form

Article definition screen

Field descriptions:

Other, Number

Ordinal number of the article in definition and in report

Code, When Print

Code of the article in definition (supporting in calculation) and code shown in report (accordance with the code in report issued by the Ministry of Finance)

Article

Name of the article shown in report

Other Article

Name of the article in English

Interpretation

Shows the position where this article is described in the interpretation of financial statements.

Print

    - If 1- Print, this article will be shown in report.
    - If 0- Not Print, this article will be hidden.

Font Bond

    - If 1- Bold, this article name will be in bold font
    - If 0- Regular, this article name will be in regular font.

Classify

1- Asset, 2- Capital
To classify asset or capital so that the balance will be extracted exactly. If asset, the system will get balance in debit (and vice versa).
In balance sheet, asset accounts, in some cases, are get balance in credit such as depreciation, adjustment, provision or AR-AP accounts with balance in both debit and credit. The same argument for capital accounts.

Off Balance Sheet

To define if the article is off or on balance sheet

    - If 1- Off, arising amount in period of the defined articles will not be shown in balance sheet.
    - If 0- On, arising amount in period of the defined articles will be shown in balance sheet.

Format

    - 1- Foreign currency: used for “Foreign currencies” off balance sheet article in case the report form in base currency is not allowed decimal number. If this option is selected, this article can be input data with decimal number in browse screen and shown in report. Number of decimal digits is defined in “FC Amount Field ” format parameter in System – Optional Parameters – Format.
    - 0- Base currency: the values of articles with this option in report are defined in “Amount Field” format parameters in System – Optional Parameters – Format.

Calculating Way

    - If 1- Base on Account Balance, the account balance will be get from debit or credit site according to the “Classify” above.
    - If 0- Based on Formula, the calculation of the article can be defined in the “Formula” field below.

Account

The account in accordance with each article, extracted from chart of accounts.

Type

Used in case the account has balance in both debit and credit, detailed by customer/supplier.

Note: If the account in not AR/AP account, it is required to select 0 – No.

Mode

If the account is defined to get balance in Debit but it has balance in Credit, the balance will be shown in negative number. In case the article is not allowed negative number but zero, we have to select “Value is at least 0”.

Formula

The formula field is activated when the calculating way is 0 – Base on formula. The defined codes can be added or subtracted and must be put in square brackets [ ].
To the division expression, if the denominator is predicted as 0, declare as example 2

For example 1: article “1. Cash” coded “111” is sum of articles coded [11A]+[111B]+[111C]”
For example 2: [111A]+[111C] + CASE WHEN [100] > 0 THEN 12 ELSE 0 END
