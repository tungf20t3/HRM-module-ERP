Voucher Book List

Field descriptions

Form

Code identifying form of invoice in case your company applies the user-defined form; for example: 02GTTT3/001
Refer to Automatic Serial Voucher Number Definition for more details

Invoice Form

This field is used to input the form number of invoice in case of using user-defined invoice form; extracted from Invoice Form List. This is an optional field.

Series

Series of invoice in case your company applies the user-defined form; for example: AB/12P

Number Start

Number start defines the rule of voucher number increase in this voucher book.
If voucher number is desired to be “PT01”, PT01/09/11…, it can be defined by day, month, year and voucher name. Number start is a required field and must be defined in the same format as Number end and Current number.
To define voucher number by month and year, there must be “/” or “_” mark among them. Number of the longer character string will be auto increased; if there are 2 equal character strings, the 1st one will be auto increased.

For example: PT01/09/11 means Receipt no. 1 in September, 2011. If there is a new voucher created, the voucher number will be auto increased to PT02/09/11. If PT01/09/2011, it will be auto increased to PT01/09/2012

Number End

Number end shows the number of vouchers in a voucher book. During voucher entry using voucher book, if the number end is used, the system will give message “Voucher book has been used all”. The number end must be defined in the same format as Number start and Current number.
If the “Number Start” has n characters, the “Number End” will be system-generated to the greatest one having n characters. However, it is editable.

Current Number

Current number shows the voucher number being used in book. It will be system-generated after the “Number Start” is input and less than the “Number Start” 1 unit. If the “Suggest the Next Voucher Number” parameter is “Yes”, the duplicate voucher number will be system-generated by adding 1 to the “Current number” during opening data entry (in case of using voucher book).

Check Duplication

Check the duplication of voucher number by one of following options:

    0- No
    1- Daily: check voucher numbers created within a day
    2- Monthly: check voucher numbers created within a month
    3- Quarterly: check voucher numbers created within a quarter
    4- Annually: check voucher numbers created within a year
    5- All: check all voucher numbers in the system

“Warning only” option

If you select to check voucher number duplication but not activate “Warning only” option, when you create a new voucher with the same number to the existing one, the system will give message and does not allow saving the voucher.
On the contrary, if the “Warning only” option is activated, the system will give message but allow saving the voucher.

Issuing Date

The date when the voucher book is issued in case your company applies the user-defined form
