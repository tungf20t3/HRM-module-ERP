Service Purchase Invoice

Field descriptions - format view

Supplier

Supplier ID and name; extracted from “Supplier List”.

Delivered by

Person delivers goods, extracted from Supplier List (“Partner” field), corresponding with supplier ID default or entered by user. This field is declared to use or not in “Voucher Entry Declaration”.

Credit Account

Credit account used to reflect business activities, extracted from “Chart of Accounts”.

Invoice Number/ Serial Number/ Invoice Date

Invoice information created by supplier; used in input VAT reporting or invoices listing for payment detailed by invoice.

Memo

Services purchase reasons/ memo- an optional field.

Voucher Number

Voucher number can be automatically increased 1 when creating new voucher but editable.

Voucher Date

The date the voucher is created. Reference information only. Program will use GL date to calculate.

Posting Date

The date the voucher is posted; used in calculating and default as created date. It can only be updated when user click mouse. GL date must be greater than closing date.

Ex. Rate

Exchange rate is used for invoice in foreign currency. Select FC code and enter exchange rate. Exchange rate is extracted from “Exchange Rate List”, in the latest date before invoice date.

Status

There are 3 statuses:

    - No Action: the voucher has not been posted to any book;
    - Release Later: If there is VAT data, the invoice will be shown in VAT reports. If this is AR/AP invoice, it will be shown in AR/AP by invoice reports;
    - Release: the voucher has been posted to all related books.

Payment Term

Extracted from Payment Term List, used for receivable tracking by payment term but optional.

Field descriptions - “Detail” tab

Debit Account

Debit account; extracted from “chart of accounts”

FC Amount

Amount in transaction currency

Amount

Amount in base currency

Description

Description of arising transaction;

Job

Job relating to Service Purchase Invoice; extracted from Job List

Field descriptions - “Tax’ tab

“VAT” tab is used to input tax data of invoices so that the system will auto run input VAT listing.

Report Form

Report forms are used as filter condition of reports in Tax Reports module;
Consist of:

    - 3 – VAT Invoice (default);
    - 4 – Goods, Services Purchased without Invoice;
    - 5 – Sales Invoice.

Tax Type

VAT types of purchased goods, services; used to classify taxes when reporting “Input VAT Invoice Listing” (Form 01-2/GTGT)”.
Consist of:

    - 1 - Goods, services used for VAT taxable business activities with revenue;
    - 2 - Goods, Services used for VAT non-taxable business activities;
    - 3 - Goods, Services used for both VAT taxable and non-taxable business activities;
    - 4 - Goods, services used for investment projects which meet tax deduction condition;
    - 5 - Goods, services not be declared in Form No. 01/GTGT.

Invoice Form

This "Invoice Form" is default as one in optional parameter but editable.
The "Invoice Form" will be used in tax reports by the following priority:

    - The "Invoice Form" input in this voucher is the first priority;
    - If this field is blank, the system will get the "Default Input Invoice Form" in optional parameter;
    - If the "Default Input Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Invoice Number / Serial Number/ Invoice Date

Data in these fields are extracted from format view but editable.

Supplier ID/ Supplier Name/ Supplier Tax Code/ Goods – services name

Information of supplier and goods in invoice

Tax Code / Tax Rate / Tax Account

Tax code is extracted from “defaulted input VAT code” parameter in “optional parameter definition” by default.
Tax rate and tax account are also extracted from tax code in VAT rate list as the second priority.

Tax Authority

State Tax authority, where company registers tax declaration
If tax account is AR/AP Account, Tax Authority is a required field
