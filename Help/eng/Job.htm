Job List

Job list can be used in cost aggregating, product cost calculating; to control turnover and profit of construction jobs, services or commercial contracts, industrial products.

Field descriptions

Job Date

The date when the job arises

Job Number

Number of job is used to number job-related documents controlled by job. This field is usually used together with job code.

Balance Tracking

This option defines whether the system will keep tract of account – job balance or not. If you select balance tracking, the job balance will be copied to next year, the job opening balance entry is enabled and vice versa.

Tracking by Customer

This option defines whether the system will keep tract of account – job – customer balance or not. If you select balance tracking by customer, the job balance detailed by customer will be copied to next year and vice versa.

Currency

The monetary unit used in job; extracted from “Currency List” and required field

FC Amount

The value of job in the currency used in job

Base Currency Amount

The value of job exchanged to base currency

Start Date/End Date

The date when the job begins and finish; reference information

Parent Job

Job parent is also created in this Job List; supports in analysis, summary reporting by parent job - job.

Customer/Employee/Department

Customer, sales employee and department executing the job are extracted from corresponding lists, reference information.

Group 1/2/3

Groups are used to classify jobs in data analysis for reporting; extracted from “Job Group List”.

Some tasks in list maintenance

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.
The data importing process:

    - Enter Job List;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to Item List.

Notes of inputting data in template file:
- Job Code: If some code already exists in Job List, the system will replace by one in the template file (if the copy type is "Overwrite"). The new codes are certainly added to Job List. The length of code is the same as one in add (edit) screen;
- Description, Other Name: manually input in form of string with the same length as one in add (edit) screen;
- Job Date, Start Date, End Date: input in date format;
- Balance Tracking: accepted with value 1 or 0;
- Currency, Customer, Employee, Department, Group 1/2/3: must be one existing in the corresponding list with the same length as one in add (edit) screen;
- FC Amount, Base Currency Amount: input in format form, 14 digits maximum and 2 decimal digits (standard format).

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field;
- Invalid data type (for example: 1,0 or yes/no), in this case, the system indicate the specific invalid cell;
- Duplicate or nested codes, in this case, the system indicate the specific invalid cell.

Other notes:
- If you do not have the "New" right in "Job List", you cannot import new data from spreadsheet to this list;
- If you do not have the "Edit" right in "Job List", you cannot edit data in this list from spreadsheet.
