Physical Stock Opening Balance

Opening balance of item is input once when you start using the system; opening balance of next periods and next year will be automatically calculated and transferred.
All opening balance of items is input in this section except items calculated by FIFO method (balance of these items must be input in each receipt in FIFO balance entry; and then total balance of each item will be calculated and saved in each site).

Entry screen

Field Descriptions

Item code

Stocked item code

Site ID

Site having stock; only sites of current unit are displayed.

Location ID

Location in site stocking opening quantity; an item can be stocked in many locations.

Lot ID

Lot ID of opening stock if this item is declared “Lot control”

Opening Qty

Opening quantity of stock

Opening balance / FC Opening balance

Opening value of stock in GL currency and foreign currency

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

The data importing process:

- Enter "Physical Stock Opening Balance" menu;
- Click the icon to download template file to put data in;
- After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to account opening balance file in the system.

Notes of inputting data in template file:
- Date: input in date format;
- Item, Site, Location, Lot: must be one existing in the corresponding list with the same length in add (edit) screen in list. The input items must not be ones with FIFO costing method;
- Site: input sites belonging to assigned units;
- Location: input only when item with location track;
- Lot: input only when item with lot control;
- Opening Quantity: input in number format, 9 digits maximum and 3 decimal digits (standard format);
- Opening Balance: input in number format, 14 digits maximum and 2 decimal digits (standard format);
- FC Opening Balance: input in number format, 12 digits maximum and 4 decimal digits (standard format).

The system will give error message in some following cases:
- Data in column containing item, site, lot, location codes do not exist in the corresponding lists. In this case, the system will indicate the specific invalid cell;
- Data closing by unit (different from the current unit); in this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field; the field length is the same as one manually input.

Other notes:
- Cannot import data after system or unit data closing (the data closing data after the beginning input date);
- The current user must have the "Edit" right in "Physical Stock Opening Balance" to edit the balance rows available in data (select the copy Type "Overwrite");
- The current user must have the "New" right in "Physical Stock Opening Balance" to add the new balance rows into data (select the copy Type "No");
- The current user must have the "Edit" right in "Access Right by Unit" to edit the balance rows available in data (select the copy Type "Overwrite");
- The current user must have the "New" right in "Access Right by Unit" to add the new balance rows into data (select the copy Type "No");
- Enable to import data for many units. If the data of the current unit has been closed, the system will give warning message when you click the icon;
- If data of other units has been closed, the system will also give warning message during importing process. In this case, the system will indicate the specific invalid cell;
- Enable to import physical opening balance of items with "FIFO" valuation method;
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
