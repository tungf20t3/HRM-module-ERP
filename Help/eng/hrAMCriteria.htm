Some field descriptions

No.

Ordinal numbers in level 1 (1, 2, 3…) are system-generated during data entry in grid. Ones in lower levels (1.1, 1.2, 2.1, 3.1…) are user-defined by clicking on icon.
Level 2 numbers are lower level numbers of the right upper level 1 number. You cannot delete the upper level number if it contains the lower level ones. It is possible to move up and down among rows.

Some tasks in data entry

Import data from excel spreadsheet (mass importing)

The system is enabled to import data from excel spreadsheet to save hours of manual effort. The data importing process: - Enter “Appraisal Criteria”;
- Click the icon to download template file to put data in;
- After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to grid view of the voucher.
Select type of copy as 1 - Overwrite if you want to update the available timekeeping data. If you select 0 - No, the system will insert required timekeeping data to blank only.

Notes of inputting data in template file:
- Employee ID: must be ones existing in the Employee List with the same length in add (edit) screen in list; (only employees belonging to departments in the assigned units);
-Period: in number format with 2 characters maximum;
- Note: will be imported from the first row only;
- Level: 1 or 2 in accordance with employee level. Level 2 is lower level of the right upper level 1.

For example: input 1, 1, 2, 2 in the spreadsheet, the grid rows after being imported will be numbered with 1, 2, 2.1, 2.2 levels.

The system will give error message in some following cases:
- Data in column containing object codes do not exist in the corresponding lists. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field (indefinite error);
- An employee is input with level 2 before level 1 or without level 1;
-Does not exist one in following information: employee ID, period, year, level or criterion.

Other notes:
- You must have the "New" right in “Appraisal Criteria” to import new data to voucher detail grid;
- You must have the "Edit" right in “Appraisal Criteria” to edit data in list.
