First In – First Out

First In – First Out Logic

First in first out cost (FIFO) is calculated to apply to Issues by gradually deducting from Receipts following the principle that first receipt is issued first.

The system only gives out the last issue unit cost, does not show from which Receipt Issue is created.

Priority in FIFO

In FIFO calculation, it is important for you to define which Receipt is issued first. The order can be defined as following:

    - 1. Issue date
    - 2. Calculate and update cost to all Transfers
    - 3. Calculate and update cost to Invoice and other Issues

In each step 2 and 3, there are many sites, many kinds of transaction with different numbering system so calculation will follow priority order:

    - 1. Priority ordinal number of sites
    - 2. Voucher number of Issues
    - 3. Priority ordinal number of Receipts
    - 4. Voucher number of Receipts

The reason of the priority order above:

    - Transfer is first priority to solve 02 unclear problems: cost of items in Transfer will be cost in Receipt at receiving site and other Issues will be calculated then; voucher number of Receipt is the same as one of Transfer so its numbering system will be different from that of other Receipts.
    - In case there are many sites and transfers among sites, the program can not recognize which voucher is created first because they can be numbered by different systems. So user should number priority order to transferred sites in “Site List” definition.
    - There are many transactions which have different numbering system. So you should number priority order to know which voucher is first. You can view in “No.-FIFO” field in “Transaction List” definition.

Receipt voucher date order in accordance with Issue

First in first out cost (FIFO) is calculated by gradually deducting from Receipts. So if inventory quantity is not enough to issue, program will stop calculation and ask to check Receipt vouchers date though closing inventory quantity is enough to issue.
To avoid these problems when using FIFO cost, you should select “Calculate monthly FIFO cost” and apply reasonable management method so that on-hand inventory always supplies requirement for effective use of the program.
Items cost by FIFO method are defined in Item information entry. FIFO cost is calculated at “Calculate FIFO Cost” function.

Related system parameters

Calculating method

    - 1 – “By day”: if quantity of an issue applied FIFO cost is greater than stock to issue date, the program can not calculate cost for this issue and gives message “there is not enough item to issue” to check.
    - 2 – “By month”: it is necessary for Issues or Stocks satisfy issue quantity monthly.

Field descriptions

From period / to period / year

FIFO cost is usually calculated monthly, continuously. If it hasn’t been calculated in some month, you must recalculate from that month.

Update Types

There are 3 options for update result of calculation to system:

    - 0 – “Calculate and Update Stock”: warehouse dept. calculates issuing cost for reference, does not post to GL.
    - 1 – “Calculate and Update Stock and GL”: adequate entry
    - 2 – “Post to GL”: calculate cost and not update to stock card.
