Service Return to Supplier

Field descriptions - format view

Supplier ID

Code identify supplier; extracted from “Supplier List” (required field)

Note: when moving the cursor to Supplier ID field, the debit balance and address of this supplier will appear.

Recipient

Person receives goods, free entered

Inv. No.

Number of “Service Return to Supplier” invoice; user-defined or system-generated

- If the invoice is numbered by voucher book which has been defined the "Invoice Form", this "Invoice Form" will be used in tax reports.
- If the invoice is numbered by voucher book which has not been defined the "Invoice Form", the system will get the "Default Output Invoice Form" in optional parameter.
- If the "Default Output Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Serial Number

Serial number of Service Return to Supplier invoice.

Inv. Date / GL Date

The date the Service Return to Supplier invoice is created and the date all entries in invoice are posted to GL.

Ex. Rate

Select FC code and declare exchange rate in date of creating transaction.

Item group

Name of goods returned; for input VAT invoice listing purpose

Note

Number, date of purchase invoice; for input VAT invoice listing purpose

Field descriptions - “Details” tab

UOM

Default unit of measurement in “Item Master”. It can also be changed by conversion of units of measurement in some special cases.

Quantity

Returned quantity (required field).

Price

Issuing unit price of item in GL currency

Amount

Amount of item in GL currency

Cr. Account

Item account

Note:
- If the “Show quantity and unit cost in service purchase invoice” parameter is "0 - No", the following columns will be displayed: Credit Account, Account Name, FC Amount, Description, Amount, User-defined Fields;
- If the “Show quantity and unit cost in service purchase invoice” parameter is "1 - Yes", the following columns will be displayed: Credit Account, Account Name, UOM, Quantity, FC Sales Price, FC Amount, Description, Sales Price, Amount, User-defined Fields.

Field descriptions – “Input VAT” tab

Report Form

Report forms are used as filter condition of reports in Tax Reports module;
Consist of:

    - 3 – VAT Invoice (default);
    - 4 – Goods, services purchased without invoice;
    - 5 – Common sales invoice.

Tax Type

VAT types of purchased goods, services; used to classify taxes when reporting “Input VAT Invoice Listing” (Form 01-2/VAT)”.
Consist of:

    - 1 - Goods, services used for taxable business activities which meet tax-deduction condition;
    - 2 - Goods, services which do not meet tax-deduction condition;
    - 3 - Goods, services used for both taxable and non-taxable activities which meet tax-deduction condition;
    - 4 - Goods, services used for investment projects which meet tax-deduction condition;
    - 5 - Goods, services not be included in Form No. 01/GTGT

Tax Code

Code of VAT; extracted from VAT list

Tax Acct.

Account of VAT; in accordance with tax code but editable.

Tax Authority

State authority to which your company has tax declared.

Post to Output VAT

This voucher is as the same processing as "Return to Supplier Invoice". In case of check, VAT will be recorded in "Output VAT Invoice Listing", In contrast, VAT will be recorded in "Intput VAT Invoice Listing" with negative in value.

Task

Extract Data from Service Invoice

Press the icon , the program allow extract data from Service Invoice. The invoices meet the condition of greater than value "Invoice Date From" and less than voucher date.
