Tool & Supply Information

Entry screen

Field Descriptions

General Information

TS Code

Tool & Supply code is used like tool & supply card number.
Utility: in case the code is defined as a character string in which last characters are numbers in sequence, when you add new tool/supply and key in some first characters of any code, the system will auto search the greatest number of the entered string and auto increase with 1.

Description

Description name of tool & supply;

TS Type

Tool & Supply type is fixed; it is used in reports of normal TS types:

    - Tools
    - Supplies

Acquisition Reason

Increase reason is used to define increase/decrease reason, extracted from increase/decrease reason list.

Acquisition Date / Alloc. Start Date

Increase date and distribution date of tool & supply

If you input a distributing tool/supply, increase date is date on the card and distribution date is date of inputting.

Allocation Method

There are 2 methods to calculate distribution:

    0 - Line
    1 – Two times

If select 0 - Line, a period distribution will be equal to original cost or remaining amount divided by periods (depending on optional parameter definition)
If select 1- Two times:

    - 1st time allocation bases on “allocation date”.
    - 2nd time allocation will be created if it meets 1in 2 following conditions:
    (1) If tool is declared to be broken down, 2nd allocation will be created in the month of damage.
    (2) If tool is declared to end allocation, 2nd allocation will be created in ending month.

Allocation Periods

Allocation periods (months); original cost or current value will be divided by allocated months to receive monthly allocation.
If allocation type is 1- Two times, default allocation period is as 2.

Alloc. End Date

Allocation ending date of tool & supply.
To allocated by “Line method”, with option “Allocate the whole remaining amount in last period” defined in “”System/ List Maintenance/ Optional parameters”, the program will base on this allocation ending date to define the last allocation period. This is an optional field.
To allocated by two times method, the program will base on this allocation ending date to calculate allocation in the second time.

Quantity

Quantity of tool & supply issued to use. This is reference information only, is not used in calculating and reporting.

Voucher Number / Voucher Date

TS increase voucher number

Currency Code / Ex. Rate

Code of transaction currency of current vouchers; extracted from Currency list
This rate is transaction exchange rate; extracted from “Exchange Rate List”, based on the latest exchange rate. If currency code is GL currency, exchange rate will be 1.
The cursor is invalid at this field, you have to use mouse to select currency type.

Department

Code of department using tools & supplies; extracted from TS using department list. It shows by which department tools & supplies are used.

T&S Account

Tool & Supply account is account type “153*”.

Allocation Account

When creating allocation transaction entry, program will create credit entry in reference to expense account.

Expense Account

When creating allocation transaction entry, program will create debit entry in reference to allocation account.

“Detail” tab

Capital

Sourced capital of tools & supplies. The code is extracted from capital list.

Original Cost

Original cost of tools & supplies is used to calculate allocation of periods.

Accumulated Allocation

Allocated amount in case inputting information of tools & supplies being used

Remaining Value

Current value of tool & supply is original cost minus allocated amount.

Allocation per Period

Allocation amount in a period (month) is used to update monthly allocation details. It equals original cost or current amount divided by periods (depending on optional parameter definition) and can be adjusted.
Allocation is monthly calculated and saved in the database. In case of any changes, users need to recalculate allocation to update related books.

“Attached Tools/Supplies” information

This information is defined as reference information.

“Further Information” tab

Other Name

English name of tools & supplies

TS Number / Technical Parameter / Made in / MFG Year/ Susp. Reason / Suspending Date

More information of tool & supply: notation, technical parameter, manufacturing country, manufacturing year...

Group 1 / 2 / 3

Tool & supply groups are used in group reports. These codes are extracted from “Tool & Supply Group List”.
This information may not be input in this section but grouped in mass in “Group List”.

“Other” tab

Round Value

The round allocation amount of the last period. If the remaining value is less than this round value, total remaining value will be added to the last period allocation amount. On the contrary, the remaining value will be allocated in a new period.

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

The data importing process:

- Enter "Tool and Supply Information" menu;
- Click the icon to download template file to put data in;
- After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to voucher.

The system will give error message in some following cases:
- Data in column containing account code does not exist in chart of accounts, is general account or AR/AP account. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field which is the same as one manually input.

Other notes:
- The current user must have the "Edit" right in this function;
- The current user must have the "Edit" right in "Access Right by Unit";
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
