Interpretation of Financial Statements

Interpretation of Financial Statements Definition

Definition screen

Field descriptions

Type

There are 3 types:

    - Normal: used to define articles in form of 2 columns: current year/ previous year (or year-end/ year-opening);
    - Cross tab: used to define articles in form of many columns: “Fixed asset increase/decrease (tangible, intangible, financial lease)", “Owner Capital”. Each column is an account defined in "Accounts" field. For example: If there are 6 accounts: 2111, 2112, 2113, 2114, 2115, 2118, the report will include 6 columns;
    - Balance: used to define articles in form of 4 columns: (Opening Balance/ Increase/Decrease/Closing Balance) as “Property (Real Estate) increase/decrease”.

Calculating Way

There are 3 ways:

    - Based on Formula: calculating by code defined in formula section;
    - Balance: calculating by account balance defined in account field;
    - Arising: calculating by account arising amount defined in account field.

Data

The period in which data are retrieved; there are 2 options: 1 – This period, 2 – Previous period.

Classify

There are 2 types: Debit and Credit; define to get debit or credit of account.

Opening/Closing

Define to get opening or closing balance of accounts

Plus/Minus

Define to show the system-calculated results in positive (plus) or negative (minus)

Value is at least 0

If this option is "Yes", when the calculated balances of articles type 3 - Balance are negative numbers, they will be shown as 0.

Type

1 - AR/AP Items, 0 - No
If this option is 1, to AR/AP accounts which can have balances in 2 sides (debit/credit), only debit or credit balance in accordance with debit/credit AR/AP objects is processed. There is no AR/AP balancing between these objects.

Display

1 - Show Balance Detailed by AR-AP Object, 0 - No
If this option is 1, AR/AP account balances are shown detailed by AR/AP object (customer/supplier).

Accounts

List of accounts from which data are retrieved. The result depends on the defined calculating way: based on balance or arising; debit or credit balance; opening or closing balances…

Reference Accounts

List of reference accounts corresponding to “Accounts” above. This information is used only when calculating way is based on arising.

None Deduction Accounts

This information is used only when calculating way is based on arising.
In case there are "Reference Accounts", "None Deduction Accounts" is only used to inform that there is "deduction" (any account can be input). Then, the result will be equal to all arising of “Accounts” (debit/credit) minus a deduction amount.
This deduction amount will be arising amounts in the other side of “Accounts” (credit/debit) corresponding to "Reference Accounts".

For example:
Debit on 111/Credit on 511: 1,000,000;(A)
Debit on 111/Credit on 112: 2,000,000;(B)
Credit on 111/Debit on 112: 150,000.(C)
Credit on 111/Debit on 642: 200,000.(D)

Define an article xxx: Debit;
Account: 111;
Reference account: 112;
None deduction account: any account.

The result of article xxx will be: 2,000,000(B) - 150,000(C) = 1,850,000.

In case there is no "Reference Account", "None Deduction Account" will be used to inform that the arising amounts in the other side of "Accounts" corresponding to "None Deduction Accounts" will not be deducted in the article calculation. Then, the result will be equal to all arising amount of "Accounts" (debit/credit) minus a deduction amount.
This deduction amount will be arising amounts in the other side of "Accounts" (credit/debit) corresponding to any accounts, except "None Deduction Accounts".

For example:
The same double entries (ABCD) as above

Define an article yyy: Debit;
Account: 111;
Reference account: None;
None deduction account: 642.

The result of article yyy will be: 2,000,000(A) + 1,000,000(B) - 150,000(C) = 2,850,000.

This way of definition is useful in case the number of deduction accounts is greater than none deduction accounts. On the contrary, it should be defined to directly get from accounts and reference accounts.

Formula

Define the expression to calculate articles by code. This formula is only valid in case calculating way is 1 - “Based on Formula”. This expression includes addition (+), subtraction (-), multiplication (*) and division (/); codes are put in square brackets ([]).

Some guides in Interpretation of Financial Statement - Export to Excel

Common guides

Print

This report is specially designed to be exported to Excel file in a default template instead of directly printed from the system. It be edited and printed from Excel. Therefore, you have to defined the way for the system to dump correct data into Excel file.

Upload/Download

When you want to edit the default Excel template, download the temple to edit and upload the edited one to the system.
- To upload the completed template to server, click icon;
- To download the default template to client to edit, click icon.

Note
- The browse screen includes articles selected in filter condition only. If you want to print the whole interpretation of statements, you should print from the Excel file.
- The statement is supported in Excel (Microsoft Office) version 2007 or newer only, the uploaded file should have the extension ".xlsx".

Guides in Excel template definition

"Normal" articles

The structure for row title getting: used to export article name in report form setup to Excel file
!1.xkey#ma_so = 'code - define in report form setup'

For example:
"Short-term Security Investments" article will be defined as:
!1.xkey#ma_so = '121'.
'121' is the code defined in report form setup.

Structure for data getting
+ Year-end
!1.ps#ma_so = 'code - define in report form setup'

!1.ps#ma_so = '121'

+ Year-beginning
!1.ps0#ma_so = 'code - define in report form setup'

!1.ps0#ma_so = '121'

"Crosstab" articles

- First column
!2.ps01#ma_so = 'code - define in report form setup'

For example: "Historical cost of tangible fixed asset" - Opening balance - Houses, Architectural objects us defined as:
!2.ps01#ma_so = '2221'

- Second column
!2.ps02#ma_so = 'code - define in report form setup'

Machines and Equipment
!2.ps02#ma_so = '2221'

"Balance" articles

- Opening Balance
!3.du_dau#ma_so = 'code - define in report form setup'

For example: "Historical cost of investment property" will be defined as:
!3.du_dau#ma_so = '241'

- Increase in year
!3.tang#ma_so = 'code - define in report form setup'

!3.tang#ma_so = '241'

- Decrease in year
!3.giam#ma_so = 'code - define in report form setup'

!3.giam#ma_so = '241'

- Closing balance
!3.du_cuoi#ma_so = 'code - define in report form setup'

!3.du_cuoi#ma_so = '241'
