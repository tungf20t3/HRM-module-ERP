Field Descriptions

Format View

Voucher Type

There are 2 types of voucher: 1 - Negative Voucher and 2 - Reversing Voucher.
These 2 types of reverse transaction voucher are used to adjust the recorded vouchers. The vouchers selected to reverse transaction must be ones which have been posted to general ledger and belong to unit of the current voucher.

Voucher No.

Voucher number is systematically increased 1 when creating a new voucher but editable.

Date Created

The date when the voucher is created. This is reference information only.

GL Date

The date when the voucher is posted to general ledger. This information is used for calculation and default as Date Created. This field can be changed when you click on. The GL date must be after data closing date.

Currency

Code of the currency in current transaction; extracted from currency list. You can only click to select currency code.

Ex. Rate

If currency is base currency, exchange rate will be 1 and this field may not be updated. If the exchange rate is changed, the system will re-calculate all foreign currency and base currency amounts.

Memo

Description for the arising transaction

Note:
For description in account ledgers, description of an account is now used for the corresponding account and vice versa. Thus, to record right description of account, remember to reverse the description of accounts in data entry.

For example:
We have the corresponding relation between:
- Credit 3348: Salary cost in Nov 2011: 600
- Debit 6421-BP 1: Salary cost in Nov 2011- Dept. 1: 200
- Debit 6421-BP 2: Salary cost in Nov 2011- Dept. 2: 400


In case of reversing description, description in account ledger 3348 will be:
- Corresponding account 6421-BP 1: Salary cost in Nov 2011- Dept. 1: 200
- Corresponding account 6421-BP 2: Salary cost in Nov 2011- Dept. 2: 400

Then, description in account ledger 6421 will be:
- Corresponding account 3348: Salary cost in Nov 2011: 200
- Corresponding account 3348: Salary cost in Nov 2011: 400


In case of original description, description in account ledger 3348 will be:
- Corresponding account 6421-BP 1: Salary cost in Nov 2011: 200
- Corresponding account 6421-BP 2: Salary cost in Nov 2011: 400

Then, description in account ledger 6421 will be:
- Corresponding account 3348: Salary cost in Nov 2011- Dept. 1: 200
- Corresponding account 3348: Salary cost in Nov 2011- Dept. 2: 400

Original Voucher No./ Original Voucher Date

Number and date of the voucher get by "Select Voucher" function, un-editable

Status

Status of the current voucher, there are 3 statuses:

    - Hold;
    - Release Later;
    - Posted to GL.

The voucher can be viewed in voucher journal or financial statements only when it has been posted to GL.

Grid view

Information in grid view appears after "Select Voucher". The double entries in selected vouchers will be recorded in negative or reversing of ones in original vouchers. The information fields are un-editable, except "Description".

Some tasks in Reversing Transaction voucher entry

Select Voucher

To select vouchers for reverse transaction, click on the icon - "Extract data from voucher", filter condition form will be opened as follow:

VC Date from/to

To filter voucher need to reverse.

Type

To filter type of voucher need to reverse, choose in 3 type: 1 - General Voucher, 2 - Misc. Receivable Trans., 3 - Misc. Payable Trans.

Customer

To filter customer code of voucher need to reverse

Voucher Number

Choose voucher number need to reverse
