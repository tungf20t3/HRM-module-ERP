Purchase Adjustment Transaction

Field descriptions - format view

Supplier ID

Extracted from “Supplier List”. This is a required field.

Deliverer

Person delivers goods; this is an optional field.

Trans. code

Code of Purchase Adjustment transaction is a required field. There are 2 codes: 1- Decreases Price, 2- Increases Price.

AP Account

AP account corresponding with purchase account. Purchase Adjustment transaction is one of vouchers used to write-off duplicate entries, so accounts in cash, cash in bank must be entered in Cash Management.

Inv. No./ Serial/ Inv. Date

Invoice information created by supplier; used in input VAT reporting or invoices listing for payment detailed by invoice.

Memo

More description of Purchase Adjustment; this is an optional field.

Voucher No.

Purchase Receipt Transaction number can increase as declaration in “Voucher List” or be entered following voucher number system of each company.

Voucher Date

The date the voucher is created and the date the voucher is posted.

Ex. Rate

Exchange rate is used for vouchers in foreign currency. Select FC code, the exchange rate is extracted from “Exchange Rate List”, in the latest date before voucher date.

Status

There are 3 statuses of voucher: “No Action”, “Release Later” and “Posted to GL”.

Field descriptions - "Details” Tab

Item code

Item code in Receipt or Invoice.

UOM

Default unit of measurement in “Item Master”. It can also be changed by conversion of units of measurement in some special cases.

Site ID / Location ID / Lot ID / Quantity

These data are extracted from invoice and un-editable.

Dr. Acct.

Debit account is extracted from item account in Purchase Receipt entry and un-editable.

Inv. No. / VC Line

Invoice number and line number in invoice. This field has value only when data in receipt extracted from invoice.

Field descriptions - “VAT” Tab

Data of VAT are extracted from format view but you can edit in some special cases.

Report Form

Report forms are used as filter condition of reports in Tax Reports module;
Consist of:

    - 3 – VAT Invoice (default);
    - 4 – Goods, services purchased without invoice;
    - 5 – Common sales invoice.

Tax Property

VAT types of purchased goods, services; used to classify taxes when reporting “Input VAT Invoice Listing” (Form 01-2/VAT)”.
Consist of:

    - 1 - Goods, services used for VAT taxable business activities with revenue (default);
    - 2 - Goods, Services which do not meet tax deduction condition;
    - 3 - Goods, Services used for both VAT taxable and business activities which do not meet tax deduction condition;
    - 4 - Goods, services used for investment projects which meet tax deduction condition;
    - 5 - Goods, services which are not required to be declared in VAT Declaration Form 01/GTGT.

Invoice Form

The "Invoice Form" is default as one in optional parameter but editable. This "Invoice Form" will be used in tax reports

Inv. No. / Serial No./ Inv. Date

Data in these fields are extracted from format view but editable.

Supplier ID/ Supplier Name/ Supplier Tax Code/ Goods – services name

Information of supplier and goods in invoice

Tax Code / Tax Acct.

Data are extracted from VAT List.

Notes:
+ If the transaction code is 1 – “Decrease”: default with “Input VAT account” (In VAT List)
+ If the transaction code is 2 – “Increase”: default with “Input Deduction account” (In VAT List)

Tax Authority

State Tax authority, where company registers tax declaration

Field descriptions - format view - lower area

Dr. Term

Extracted from Payment Term list

Note:
If the transaction code of this voucher is 1 - Decrease, this "Dr. Term" field is disabled to input.

Common operation in creating Purchase Adjustment Transaction

Extract from invoice

Add a new voucher, click in the icon -“Extract from invoice”. This voucher can be get data from Domestic Purchase Invoice, Import Purchase Invoice, Purchase Receipt - Direct Issue Voucher. After selecting invoice, choose entry process and the system will automatically copy selected item rows to Purchase Adjustment Transaction.

Notes:
+ If the transaction code is 1 – “Decrease”:
In case this receipt is in the same month with the inherited one: subtract (-) the amount of the inherited receipt;
In case this receipt is in the different month from the inherited one: do not process this voucher.
+ If the transaction code is 2 – “Increase”:
In case this receipt is in the same month with the inherited one: add (+) to the amount of the inherited receipt;
In case this receipt is in the different month from the inherited one: do not process this voucher.
