Declaration of Detailed Statement Of Cost

It is necessary to estimate cost of a specific product without book closing at the end of month. Example, there is a table of planned cost for manufacturing product SPTP01:

This table is usually physically calculated so it is not absolutely exact (because of the lack of exactness in statistics and selection of cost target or overdue BOM). Besides, the targets in this table are usually not stable, so it is difficult to calculate by computer. For example, how will cost of product be if there is a cost factor added?!
In AI ERP, task described above is systemized and carried out in menu “Detailed Statement of Cost”.

Process Description

Detailed Statement of Cost Process:

    - Receive SO, consider products having the same properties as products needing to be manufactured; - After SO created, users consider products with the same BOM as required items;
    - Select report date;
    - Select product in “Filter condition” screen (“Product Code” field);
    - Click to view;
    - Edit or add calculating the articles, (add reporting items);
    - Input unit price of materials and other expenses;
    - Save report after printing, click icon;
    - Recalculate all, click icon;
    - Adjust value (BOM or Cost) if necessary to have the best cost table.

Detail for adding report

We can add one detailed statement of cost from a similar one by following tasks:

    - Select products with the same BOM as required items (“Report Form” field);
    - Select product in “Filter condition” screen (“Product Code” field);
    - Click icon to add a new cost article; Click icon to edit a cost article. According to type of article, value is entered in “Classification” field.

Classification types:

0 - Normal

Norm in this type can be input value or calculated by other norms;

1 - BOM

used for materials; in case product has bill of materials, these materials will be extracted to detail statement of cost;

2 - Quantity

Used for quantity norms;

3 - Unit Cost

Value quantity fields.

Besides common target adding, users can have simple calculation by entering in “Formula” field:

    - Division: CASE WHEN [1010]=0 THEN 0 ELSE [2100] / [1010] END
    - Round: ROUND (19*[221]/100, 0)
    - The calculation targets in formula must be in square brackets ([)

Note:
- The products which have the same structure and are only different in BOM or in figures entered after making report are grouped into 1 same Detailed Statement of Cost. Products in the same type are usually grouped together.
- If there is only 1 different cost target, a new report form must be created. Select the same form to create a new one.
- It is necessary to create Detailed Statement of Cost (Plan) for materials first. Normally, BOM includes raw material cost. If there has not been cost of materials in Detailed Statement of Cost (Plan) (effective date of Statement < effective date of BOM), cost will be automatically updated in Statement.

Main tasks

Create Report form structure

- Open Report form
- Select form with similar structure, choose “Type” as 1 – New
- Click OK to create

Edit Report form structure

- Open Report form
- Select form with similar structure, choose “Type” as 2 – Edit
- Click OK to edit

Delete Report form structure

- Open Report form
- Select form with similar structure, choose “Type” as 3 – Delete
- Choose “Confirm delete”
- Click OK to delete

Go to “Other” tab for information in the right top corner of the report.
