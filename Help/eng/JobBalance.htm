Job Opening Balance

This function is used to enter opening balance (if any) of job, item before using the system. Opening date is date when beginning entering in “First Entry”. Balance at the beginning of year (if any) will be entered in account in General Ledger.
You enter opening balance only when you start using the system. Job balance to next periods in year and next years will be calculated and transferred.

Entry Screen

Field Descriptions

Account

Account related to job, item having balance

Job

Code of job having balance

Customer

Customer relating to job. This field can only be entered if you select “1 – Yes” in “Balance Tracking by Customer” option in Job information.

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

The data importing process:

- Enter "Job Opening Balance" menu;
- Click the icon to download template file to put data in;
- After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to opening balance file in the system.

Notes of inputting data in template file:
- Unit, Account: Unit is one of the assigned units; account is neither general account nor AR/AP account. The input unit, account code must be one existing in the list with the same length in add (edit) screen in list;
- Job, Customer: must be one existing in the list with the same length as one in add (edit) screen. Job must be one with balance tracking;
- Debit Balance, Credit Balance: input in number format, 14 digits maximum and 2 decimal digits (standard format);
- FC Debit Balance, FC Credit Balance: input in number format, 12 digits maximum and 2 decimal digits (standard format).

The system will give error message in some following cases:
- Data in column containing account code does not exist in chart of accounts, is general account or AR/AP account. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field which is the same as one manually input.

Other notes:
- The system will not get data of accounts without balance;
- Check the existence of "Unit, Account": if some code already exists, the system will replace by one in the template file (if the copy type is "Overwrite"). The new codes are certainly added;
- Cannot import data after system or unit data closing (the data closing date after the beginning input date);
- The current user must have the "Edit" right in "Job Opening Balance" function;
- The current user must have the "Edit" right in "Access Right by Unit";
- Enable to import data for many units;
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
