Delivery Note

Process

Delivery is the next step after Pick List. According to each enterprise, sales processes will be different; therefore, Delivery Note will be created before or after Sales Invoice.

    Previous step: Creating Issuing Order (or Invoice in case invoicing before issuing)
    Next step: Issuing (or invoicing if invoicing after delivering)
    Creating Dept.: Delivery Dept.

Field descriptions - Format view

Cust. ID

Extracted from “Customer List” (required field)

Recipient

The person receiving goods (in customer side)

Memo

Delivery reason/ memo

Trans. Code

There is now 1 default code: 1 – “Delivery Note”.

Voucher No.

Delivery Note number can be automatically numbered by program following declaration in voucher list system or entered by user.

Date

Date when Delivery Note is created

Status

Status of Delivery Note, 3 options: “No Action”, “Deliver” and “Completed”.

Field descriptions - “Details” Tab

Item code

Item code in Delivery Note; directly entered in “Item Master” screen or extracted from previous steps such as “Sales Invoice” or “Sales Order”.

UOM

Unit of measurement of purchased items, extracted from “Item Master”

Site ID

Site issuing items in Delivery Note; directly entered in “Site List” screen or extracted from previous steps such as “Sales Invoice” or “Sales Order”.

Location ID

Location of items issued in site; extracted from “Location List” or from previous steps such as “Sales Invoice” or “Pick List”, used in case you wish to control items by location.

Lot ID

Lot number of goods; extracted from “Lot Maintenance” or from previous steps such as “Sales Invoice” or “Pick List”, used in case that enterprise controls items by lot. Lot declaration has been done in Inventory Management.

Quantity

Quantity of each item type; extracted from allocation transport quantity in selecting Sales Invoice / Pick List. It is detailed allocated to each transport type and each shipment in Delivery Note.

Deliver No.

Number of shipments in this Delivery Note in case it is impossible to deliver all goods by 1 shipment; automatically numbered by program.

Due Date

Deliver date of each ship; directly entered when creating note or extracted from delivery confirmation.

Transport Type

Extracted from Transport Type List

Transport

Means of transport; extracted from Transport List.

Note

More description of delivery

Confirm

Used to confirm quantity of goods delivered in case of quantity allocation without another operation.

Inv. No. / Inv. Line/ Pick. No./ Pick. Line

Number of Invoice, Pick List and lines of items in Pick List, Invoice (if data of Delivery Note are extracted from Pick List or Invoice).

Job

Job relating to the delivery

Some common tasks in creating Delivery Note

Extract from Invoice

Create new Delivery Note, click on function “ - Extract Data", select “Extract from invoice” function and type start date. Lookup and choose invoice in Sales Invoice selecting screen.
Input deivery quantity then exit Sales Invoice screen. Data in selected rows in this list will be updated to Delivery Note.

Extract from Pick List

Create new Delivery Note, click on function “ - Extract Data", select function Pick List and type start date. Lookup and choose Pick Lists. Input deivery quantity then exit Pick List to update allocated lines of items into Delivery Note.
Quantity of goods in selected Pick List must be allocated all to shipments.
