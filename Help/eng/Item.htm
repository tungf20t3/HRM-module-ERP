Item List

Field descriptions - General information


Unit of Measure

Unit of measure is the default one used in inventory reports although the entered unit of measure is another one.

Multi-UOM

The checkbox option, if activated, indicates that the item is controlled by many units of measure. If unit conversion list is defined, during voucher entry, you can select the transaction UOM, the system will automatically convert the input quantity of the item to original UOM when saving voucher.

Stock Control

The checkbox option, if activated, indicates that the item is controlled in stock. An item not controlled in stock will not be shown in inventory reports though it has been used in voucher. Items are usually controlled in stock except ones which has not been controlled any more or transaction code is recorded in sales invoice.

Lot Control

The checkbox option, if activated, indicates that the item is controlled by lot. An item controlled by lot must be input lot information in receipt-issue vouchers.
If the item is defined to check in “Lot control” and “Allow creating lot when receiving” option, it is unnecessary to define lot information in “Lot Maintenance”. Lot will be defined during receipt entries.

Physical Count

The checkbox option, if activated, indicates that the item is controlled by physical count. An item not controlled by physical count will not be shown in closing physical count reports. However, this information is now for reference only.

Valuation Method

There are 4 methods to valuate inventory cost of an item: 1 – Average, 2 – Specific, 3 – FIFO, 4 – Moving average.
To select the valuation methods above, it is required to define in System/Customization/Inventory Costing Method with status 1 – Active.
An unused method will be defined the status as 0 – Inactive in order to avoid mistake in selecting costing method during data entry relating to inventory.
Refer to: Average Cost Logic and FIFO Cost Logic for more details

Item Type

Item type, extracted from “Item Type List”, is used to categorize items in specific functions which will be developed during application deploying. Item type 51- “Finished goods” is now used to define costing objects in cost calculation. There are 5 options: 21- Raw materials, 31- Tools &Supplies, 41- Semi-finished Products, 51- Finished Products, 61- Goods.
Besides, item type is also used to group items in inventory reports.

Item Group

Item group is used in classification in reports; extracted from “Item List”.

Site

Site, extracted from Site List, is selected in data entries relating to item receipt or issue as default site.

Location

Location, extracted from Location List, is corresponding to selected site, used in data entries relating to item receipt or issue as default location of site.

VAT

The default VAT, extracted from VAT list, is used in data entries relating to item sales, purchase with VAT.

Import Tax

The default import tax, extracted from Import Tax List, is used in import purchase invoice.

Field descriptions – Account information

Inventory Account

Inventory account is defined in chart of accounts, used to create entries in keeping account of receipt &issue transactions. There will be reference account of this account in specific case.

For example: in Receipt, this account is “Debit”, reference account will be cash or payable account. In Issue, this account is “Credit”, reference account will be materials cost, overheads or others. In Sales Invoice, this account is “Credit” and reference account will be COGS…


Edit Inventory Account

Inventory account entries are usually not changed except some special cases, so “Edit inventory account” option is used to edit the account.

COGS Account

Cost account is defined in chart of accounts, used to create entries in sales invoices. In sales invoice, it is “Debit”; in Sales Return, it is “Credit”, reference account will be inventory account. This account is default in voucher but editable.

Sales Account

Sales account is defined in chart of accounts, used to create entries in keeping account of sales turnover. In sales invoice, this account is “Credit”, reference account will be cash or receivables account. This account is default but editable in voucher.

Sales Return Account

This is a type of sales turnover decrease account defined in chart of accounts, used to create entries in sales return. In Sales Return, this account is “Debit”, reference account will be cash or receivables account. This account is default but editable in voucher.

Discount Account

This is a type of sales turnover decrease account defined in chart of accounts, used to create entries in "Sales Price Adjustment Invoice". When Transaction Code is "1- Decrease in Price", this account will be recorded as “Debit”, AR account will be "Credit". This account is default but editable in voucher.

Consignment Account

This is a type of inventory account defined in chart of accounts, used to create entries stock transfers in/out. There will be reference account in specific case.

For example: in Stock Transfer in, this account is “Debit”; in Stock Transfer out, it is “Credit” instead of item account. This account is default but editable in voucher.


WIP Account

This is a type of inventory account defined in chart of accounts, used to create entries in finished goods receipt &issue vouchers if advanced costing is used in your enterprise. This account is default but editable in voucher.

Cost Difference Acct.

This account is used to create entries in stock value difference vouchers.

Discount Account

This is a type of sales turnover decrease account defined in chart of accounts, used to create entries in sales invoices. In sales invoice, this account is “Debit”, reference account will be cash or receivable account. This account is default but editable in voucher.

Field descriptions – Lot information


Assignment

Methods to define lot expiry; there are 2 options of lot issue type: “When received” and “When used”

    - When received: the system will define expiry date of lot by adding “shelf life” of item to receipt date.
    - When used: the system will completely base on expiry date defined in Lot Maintenance.

Issue Method

There are 4 options to define issuing method:

    - Expiration: lots having the nearest expiry date are issued first. The expiry date of lot is base to select.
    - FIFO: first received lot is issued first. The receipt date of lot is base to select.
    - Sequential: select from list only (sequence).
    - LIFO: late received lot is issued first. The receipt date of lot is base to select.

Shelf Life Days/ Warranty Days

The information is base to define the expiry date or the last warranty date of each item. These are optional fields.

Allow creating lot when receiving

If the item is checked in “Lot control” and “Allow creating lot when receiving” options, it is unnecessary to define lot information in “Lot Maintenance”. Lot can be defined during receipt entries.

Field descriptions – Other information

Minimum/ Maximum Quantity

The acceptable minimum &maximum stock quantity of item
If the item has been defined these 2 quantities and there is definition to deal with stock (Warning or Cannot Save), the system will not allow saving voucher or give message when receiving or issuing making stock quantity over max or under min.
The maximum &minimum quantities are used in Stock by Norm Report.

Volume / Weight

More reference information about item characteristics in shipping
Especially, volume and weight are used in charge apportionment in "Purchase Expense Transaction".General formula:

Item Volume (/Weight)*Item qty]/[Total Volume (/Weight)*Total qty]

Some tasks in list maintenance

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.
The data importing process:

    - Enter Item List;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to Item List.

Notes of inputting data in template file:
- Item Code: one code is unique in this column. If some code already exists in Item List, the system will replace by one in the template file (if the copy type is "Overwrite"). The new codes are certainly added to Item List. Item code cannot consist of system master or operator characters (‘ ? _ % ! @| # $ ” ? ~ ` ^ &);
- Description, Other Name: manually input in form of string with the same length as one in add (edit) screen;
- UOM, Item Group 1/2/3, Item Price Class, Site, Location, VAT, Import Tax: must be one existing in the corresponding list;
- Multi-UOM, Stock Control, Lot Control, Physical Count, Edit Inventory Account, Allow creating lot when receiving: accepted with value 1 or 0;
- Valuation Method: accepted one value (1, 2, 3 or 4) corresponding to list of item issuing costing method;
- Item Type: accepted one value (21, 31, 41, 51 or 61) corresponding to list of item types (21- Raw material / 31- Tools and Supplies / 41- Semi-finished Products / 51- Finished Products / 61- Goods);
- Inventory Account, COGS Account, Consignment Account, WIP Account, Cost Difference Account, Discount Account, Promotion Account: must be a detailed account existing in Chart of Accounts;
- Sales Account, Internal Sales Account, Sales Return Account: must be an account existing in Chart of Accounts (regardless of account type);
- Lot Assignment: accepted 1 value (1 or 2);
- Issue Method: accepted 1 value (1, 2, 3 or 4) (1 - Expiration, 2 - FIFO, 3 - Sequential, 4 - LIFO);
- ABC: accepted 1 value A,B or C, case-insensitive;
- Shelf Life Days, Warranty Days, Minimum Quantity, Maximum Quantity, Volume, Weight: input in number format with the same length as one in add (edit) screen;
- Unit of Volume, Weight: user-defined with the same length as one in add (edit) screen.

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field;
- Invalid data type (for example: 1,0 or yes/no), in this case, the system indicate the specific invalid cell;
- Duplicate or nested codes, in this case, the system indicate the specific invalid cell;
- Required fields are blank: Item Code, Item Name, Unit of Measure, Item Type, Inventory Account.

Other notes:
- To fields with 1-0 or true-false value, if there is no choice, the "Stock Control", "Lot Control" and "Edit Inventory Account" options will be auto-checked (the value is 1);
- If you do not have the "New" right in "Item List", you cannot import new data from spreadsheet to this list;
- If you do not have the "Edit" right in "Item List", you cannot edit data in this list from spreadsheet.

Refer to “Hints and Tips in List Setting” for details of common tasks in list maintenance
