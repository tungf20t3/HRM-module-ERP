Purchase Requisition

Purchase Requisition Process

Creating Purchase Requisition is the 1st step in purchasing management process. However, it will not be used in process if enterprise has purchasing management from purchase step without Purchase Requisition.

    Previous step: No
    Next step: PR Approval (Creating Purchase Order)
    Create Dept.: Require Dept.

PR Status Description

No Action

Voucher is being created in this status. It is default status when creating a new voucher. PR in this status can be edited.

Open

After creating PR, you will transfer it in this status to wait for approval in next step. Open PR can be edited.

Approved

PR has been approved and waiting for Quotation and Order. When creating PO, program will filter PR in “Approved” status
This status is set up in “Approve” step in “PR Approval” function and right in PR which user has right to approve. Approved PR can not be edited if user does not have right to approve.

Ordered

Items in PR have been ordered and prepared to Receipt step.
This status is set up in PO Approval step when there is at least 1 item in PR having been created PO and approved. Ordered PR cannot be edited.

Complete

That is status when all items in PR having been ordered.
It is set up when PO approved. Complete PR cannot be edited.

Closed

This is ending status; there is no more order for PR.
It is set up in PR Approval or PR Close. Closed PR cannot be edited.

Detailed lines status descriptions

Open

Item in PR has not been approved. This is default status of detailed line when creating new PR.
This status is set up in lines having requirement but disapproved in “PR Approval” step.

Approved

Item in PR has been approved. This status is set up in “PR Approval” step or in PR creating step if user has right to approve this PR

Ordered

Item in PR has been ordered (PO has been approved) and prepared to Receipt step.
This status is set up in PO Approval step when item has been created PO and approved.

Disapproved

Items in PR are disapproved. This status is set up in PR Approval step if it is chosen “Cancel” in PR Approval or PR Close step.

Approved by

Input the user who has the right to approve the requisition defined in “Approving Officers” menu. To process the approval online, it is required to declare the phone number and e-mail of the approving user in “User Define”,“Other” tab.

Guides of online approval function

Approve via mobile phone

To use this function, it is required to register 3G service and 3G USB device. To relevant configuration, please refer to consultant during implementation process.
How to use: Select the requisition in level 1 – Urgent and “Open” status, input user code and save the requisition. The system will send the approval request to the approving user’s mobile phone with brief information including PR number, total amount, approval code and cancel code. The approving user simply replies to the message with approval or cancel code, the system will change the PR status to “Approved” or “Closed”.

For example:
The approving user receives a message with following information:

"PR#: NC4-01-12
Total Amount: 24 000 000
Approval Code: 7529D7
Cancel Code: 77AA9F""

To approve the requisition, reply with ""7529D7""; if not, reply with ""77AA9F""

Approve via e-mail

To use this function, assume that the company has registered the internet service; to relevant configuration, please refer to consultant during implementation process.
How to use:
Select the requisition in “Open” status, input approver code and save the requisition. The system will send the approval request to approving user’s e-mail. The user simply clicks on “PR Approve” or “PR Cancel” link to approve or cancel, the system will change the PR status to “Approved” or “Closed”.

- When you select the requisition in level 1 – Urgent, AI ERP will send the approval request to both mobile phone (SMS) and e-mail of the approving user.
- When you select the requisition in level 2 – ASAP or 3 – Normal, AI ERP will send the approval request to e-mail of the approving user only.
