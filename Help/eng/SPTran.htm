Sales Price Adjustment Invoice

This invoice is used to record the adjustment of the service invoices before.
This function is used to create invoice in case the amount in the invoice issued to customer is greater or smaller than the actual amount of delivered goods, services and the customer requests an invoice of the adjustment..

Sales Price Adjustment Invoice entry screen

Field descriptions

Buyer

The representative of customer who directly do the purchasing; be extracted from Customer List (“Partner” field) in accordance with customer ID or manually entered. This field is defined to be used or not in “Voucher information” function.

Transaction Code

There are 2 types of transaction code: 1 – Decrease and 2- Increase.
- If transaction code is “1 – Decrease”, the program will record: debit with "Reference account" and credit with "AR/AP account". In other entry, debit with "Tax account" and credit with "Tax reference account" simultaneously.
- If transaction code is “1 – Increase”, the program will record: debit with "AR/AP account" and credit with "Reference account". In other entry, debit with "Reference account" and credit with "Tax reference account" simultaneously.

Account Receivable

The account used to record the decrease of receivable from customer

Invoice No.

Number of adjustment invoice. In case of using voucher book, this number will be auto-increased 1 when a new voucher is created but editable. The system will check the number duplicate when saving.

- If the invoice is numbered by voucher book which has been defined the "Invoice Form", this "Invoice Form" will be used in tax reports.
- If the invoice is numbered by voucher book which has not been defined the "Invoice Form", the system will get the "Default Output Invoice Form" in optional parameter.
- If the "Default Output Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Serial No.

Serial number of invoice; manually input

GL Date

The date the invoice is posted to GL; used in calculating and default as invoice date. GL date must be greater than closing date. This field can be updated only when you click mouse.

Ex. Rate

If transaction currency is the same as base currency, exchange rate will be 1 and not be updated. If exchange rate is changed, the system will recalculate value in foreign and base currency.

Item Group /Note

In order to descript the items of invoice in "Output VAT Invoice Listing" report.
More description informations in "Output VAT Invoice Listing" report.

Tax Account

Tax Account will appear after transaction code is inputed. If transaction code is “1 – Decrease”, default with "Output Deduction Account" in "Value Added Tax List"
If transaction code is “2 – Increase”, default with "Output Account" in "Value Added Tax List"

Tax Reference Account

By default, see "Transaction Code", but editable

Payment Term

Get the payment term defined in Customer List of looked up from Payment Term List, valid only when transaction code with “Voucher Type” 2 – Increase

Field descriptions – “Details” tab

For more the same fields, please refer to "Invoice" help.

Reference account

- If transaction code is “1 – Decrease”, the program will get value from “Discount Account” of Item list (but editable).
- If transaction code is “1 – Increase”, the program will get value from “Sales Account” of Item list (but editable).

Task

Extract Data from Invoice

Press the icon , the program allow extract data from Invoice. The invoices meet the condition of greater than value "Invoice Date From" and less than voucher date.
