Voucher Number Mass Update

This function is used to mass update the voucher number system in case of need to re-order voucher. Step to perform:

Step 1: Select the required vouchers to update:

Step 2: Input the start number:

Period/Year

According to the calendar period, it will return the data of the field "From/to date" to filter by posting date of the voucher in the general ledger; ;

Type

- 1 - By voucher: The program will replace the current number of the vouchers which has codes in the selected list.

Note:
The voucher codes in the selected list must have the common parent voucher code, if not the program will not perform updates.

- 2 - By voucher book: The program will replace the current number of the vouchers which use the selected book code.

Note:
The selected book code must not be used for voucher codes listed below (*), if not the program will not perform updates.

(*): List of voucher codes should not be setted up to use voucher book:
- CO1 - Production Overhead Allocation Entry;
- PK2 - Exchange Rate Different Reversing;
- PK3 - Auto-Posting Entry;
- PK4 - Auto-Allocation Entry;
- PK5 - Depreciation Posting Entry;
- PK6 - Rate Difference Entry;
- PK7 - Distribution Posting Entry;
- PK8 - Average Book Rate Difference Entry;
- PXF - Auto Issuing Transaction for Cost Difference;
- PXG - Purchase Receipt - Direct Issue Voucher (Issue).

Book Code

Extracted from "Voucher Book List" in case "Type" is "2", only show the code which has "Issuing Date" <= "From date".

Voucher List

Extracted from "Voucher Infomation" in case "Type" is "1".

Start Number

Require input before the operation will be performed. The increasing rule is the same as rule of "Start Number" in Voucher Book List.

Unit

Extracted from Unit List. Allow chose multi units which assigned the right to user.

Other Notes:
- Performed ascending order: the browser or replacement is odered by posting date-> Ordinal Number(Voucher Information) -> voucher number ;
- Allow drilldown;
- Can not perform this funtionc in case of the period is not between working date;
- Can not perform this funtionc in case of data closing (general, unit or voucher data closing).
