Field descriptions - format view

Customer

Customer ID and name, extracted from Customer/Supplier List, is usually used in debt control in case of Issue related to debt transactions as lending items from other units. In case transactions which are not related to debt control, this information is used to control internal objects as production line, shop floor…

Buyer

Person buys goods; extracted from Customer List (“Partner” field) correlative with Customer ID. Information of this field is defined in “Voucher Enter Screen Declaration”

Debit Account

Debit account is extracted from the “default account” field in customer List by default. When Dr. Account changes, Tax Ref. Account also changes. Dr. Account is chosen from Chart of Accounts. Dr. Account can show the lowest level Account or Parent Account depending on definition of “Search by code or name definition” menu. If Parent Account shows, this Account will be blear and not be chosen.

Credit Term

Payment term is extracted from payment term list, used in controlling debit aging of invoice.

Sales Employee

Sales employee ID and name, extracted from “Sales Employee List”

Memo

More information in buying services

Invoice Number

Invoice number will be auto-generated with 1 increase when a new voucher is created but editable.

Serial Number

Series is entered by user. In case of using the voucher book list, is extracted from “invoice series” field in voucher book list by default.

- If the invoice is numbered by voucher book which has been defined the "Invoice Form", this "Invoice Form" will be used in tax reports.
- If the invoice is numbered by voucher book which has not been defined the "Invoice Form", the system will get the "Default Output Invoice Form" in optional parameter.
- If the "Default Output Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Invoice Date

The voucher’s date is created; reference information only. Program will use GL date to calculate. When create voucher, invoice date is returned from system date by default.

Posting Date

The date the voucher is posted; used in calculating and default as created date. This field can only be updated when you click mouse. GL date must be greater than closing date.

Ex. Rate

If transaction currency is the same as GL currency, exchange rate will be 1 and not be updated. If exchange rate is changed, program will recalculate value in foreign and GL currency.

Status

    - No Action: the voucher has not been posted to any book;
    - Release Later: If there is VAT data, the invoice will be shown in VAT reports. If this is AR/AP invoice, it will be shown in AR/AP by invoice reports;
    - Release: the voucher has been posted to all related books.

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

The data importing process:

    - Enter Invoice;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to grid view of the invoice.

Notes of inputting data in template file:
- Unit, Debit Account, Customer: Account must be detailed account. Unit is one of the assigned units with unit locking date before invoice date;
- Currency, Tax Code, Tax Authority, Job, Department, MO, Product, Contract, Expense, Loan Contract: must be one existing in the corresponding list with the same length in add (edit) screen in list;
- Tax Account: must be detailed account. If there is no tax account in the imported file, the system will get one in accordance with tax code;
- Date: input in date format. Data closing date and voucher locking date must be before voucher creating date;
- Quantity: input in number format, 9 digits maximum and 3 decimal digits (standard format);
- Price: input in number format, 12 digits maximum and 4 decimal digits (standard format);
- Amount: input in number format, 14 digits maximum and 2 decimal digits (standard format). The system will import the right amount input in the template file, not recalculate.

The system will give error message in some following cases:
- Data in column containing customer, supplier, account do not exist in the corresponding lists. Account is general account. In this case, the system will indicate the specific invalid cell;
- The required fields (Invoice Number, Unit, Tax Authority,...) are blank;
- String data is longer than the available length of field; the field length is the same as one manually input.

Other notes:
- Cannot import data after system data closing, unit or voucher locking date (the data closing date after the beginning input date);
- The current user must have the "New" right in "Service Invoice" menu;
- The current user must have the "New" right in "Access Right by Unit" menu;
- Enable to import data for many units. If the data of the current unit has been closed, the system will give warning message when you click the icon; if data of other units has been closed, the system will also give warning message during importing process and indicate the specific invalid cell;
- Check the duplicate vouchers by number only (if available);
- If the invoice currency is the same as base currency, it is required to input exchange rate as 1;
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
