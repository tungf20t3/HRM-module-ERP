Optional Parameters

This function is used to define optional parameters to adapt specific requirements of each business.

Definition screen

Company Parameters

Decision Issuing Date / Number of Decision Regulating Accounting System

Information of the accounting system that your company is following

Director's Name

Name of company director in Vietnamese and English.

Chief Accountant’s Name

Name of company chief accountant in Vietnamese and English.

Base Currency

Base currency following Vietnamese accounting regulations; therefore, all transactions in foreign currency will be converted to base currency when reporting.

Company’s Address/ Province/City/ District/ Phone Number/ Fax Number/ Email Address

Vietnamese and English information of company; used in VAT declaration

Tax Code

Tax code of company; used in VAT reports

Bank Account Used in Invoice

Bank account number and bank name used when printing sales invoices directly from the system.

Default Tax Type

Default value in input VAT voucher entry. There are 5 types:

    1- Goods, services used for taxable business activities eligible for tax deduction
    2- Goods, services ineligible for tax deduction
    3- Goods, services used for both taxable business activities and ineligible for tax deduction
    4- Goods, services used for investment projects eligible for tax deduction
    5- Goods, services not required to be declared in report form 01/VAT

Tax Program Directory

The folder storing tax data when AI ERP auto transfers data to tax program of the General Department of Tax. This is not used in case of multi-unit. If the current unit has the directory defined in Unit List, the system will use the defined directory.

Tax Accountant

The person who gives signature in tax reports when exported to tax program.

Backup Parameters

Backup Folder on Server

Define the directory for backup files; it can be addressed to data-containing computer (server).

For example: “D:\Databases\Backup\”.

Other Backup Folder

Define the other directory for backup files; it can be addressed to data-containing computer.

For example: “C:\Databases\Backup\”.

Note:
If the backup folder directory is defined in client, it is known as network directory (relative); since the data compression is required from client.
This is the directory that the client can view, not the absolute one in server. If it is absolutely defined in server, when processing backup in client, the address cannot be understood; because the system has to rely on this directory to compress the backup file.
Specifically, we have to define the directory as a shared folder from the server such as \\server1\backup\, \\server2\luutru\...

GL Parameters

Duplicate accounts in sales & purchasing modules

This parameter is used to check vouchers posted to GL in case there are duplicate vouchers in goods, materials sales and purchases by cash. Purchasing by cash transaction has payment voucher and purchase receipt. These two vouchers have the same posting to GL but input by two different accountants, and the information may be different. For example, in case of sales paid at site, sales invoice made by AR accountant contains quantity, unit price, item value and receipt voucher made by cash accountant contains item value only. Default parameter values are 111,112. The system will check the reason code when saving in and out vouchers, if the reason belongs to duplicate accounts, the voucher entry will not be posted to GL, GL entry will be extracted from receipt - payment vouchers. See "Deal with coincident entry relating to cash and bank deposit" in “Other guides” in Cash Management module for more details

Accounts Receivable/ Payable

Only accounts defined in this list will be shown in viewing reports relating to AR/AP accounts. The accounts defined in Chart of Accounts but not defined in this AR/AP list will not be AR/AP accounts.

Cost Variance Account

When calculating average cost, the system will auto generate accounting entry of the stock different amount by this account if the item has not been defined the “Cost Difference Account” information in Item List.

Revenue Account for Write-off

When running write-off to invoices in “Accounts Receivable/Payable/ Data Entry/ Write-off invoices”, the system will auto post to credit of this revenue account, corresponding with AP-AR account of written-off invoices. The default account is 7113- Other revenue.

Expense Account for Write-off

When running write-off to invoices in “Accounts Receivable/Payable/ Data Entry/ Write-off invoices”, the system will auto post to debit of this expense account, corresponding with AP-AR account of written-off invoices. The default account is 8113- Other expenses.

Evaluation of Business Results Account Code

Define an account for the evaluation of business results; default account is 911.

List of the first characters of accounts without balance

This definition helps the system recognize accounts without balance in opening balance entry. Default values are “5, 6, 7, 8, and 9”.

Default Input VAT Code

Define the default tax code when creating new voucher having output VAT

Default Output VAT Code

Define the default tax code when creating new voucher having input VAT

Advanced Payment Control

If this parameter is "Yes", corresponding to following cases:

    - Account input in column “Account” (Detail grid) is “AR Account” in list of parameter “Accounts Receivable” and “Debit” is not equal 0 , AR Invoice will be posted to AR report;
    - Account input in column “Account” (Detail grid) is “AP Account” in list of parameter “Accounts Payable” and “Credit” is not equal 0 , AP Invoice will be posted to AP report.

Assess the Excise Duty on Imported Goods

If this parameter is “Yes”, in data entry screens of import purchase invoice, import purchase order, custom declaration sheet, there will be more fields to input Excise tax information.

Customer/ Supplier Opening Balance of the Fiscal Year

If this parameter is “Yes”, in case the first data entry date is different from the beginning date of fiscal year, year-opening balance can be input by customer/supplier in “Customer/ Supplier Opening Balance” functions in Accounts Receivable/ Payable modules. This parameter is used only when year-opening balance of one customer/supplier having both Credit and Debit balances in 2 sub-accounts (of 1 parent AR-AP account). This ensures that accounting data will be balanced in “Balance Sheet”.
This detailed entry can replace the account balance entry in “Account Opening Balance” function in General Leger module

Note: In case there are both Debit and Credit balances but you want to enter by account (not by customer/supplier), choose “No”. Then, you have to manually balance Debit and Credit balances between 2 sub-accounts before entering to the system (enter only Debit or Credit balance after balancing the account with the greater balance)

Show sales promotion (in invoice)

If this parameter is “Yes”, there will be “Promotion” and “Promotion expense account” fields shown in Sales invoice entry screen.

Note: in case goods on promotion are invoiced separately from sales invoice, this parameter must be “No”. Goods on promotion information will be entered the same as goods sold.

Calculate Tax on Promotion Goods

If this parameter is “Yes”, in data entry screens of sales invoice, or sales return, the system will add the VAT of the promotion goods to the VAT of goods sold. Otherwise, the system will calculate the VAT of the goods sold only.

Payment Control Detailed by Invoice defined in Input VAT

If this parameter is “Yes”, invoice data in “VAT” tabs in Domestic Purchase Invoice, Service Purchase Invoice, Purchase Expenses Transaction will be transferred to a file to keep track of payment. This option allows inputting many invoices of one supplier in one voucher. If there is no data in “VAT” tab, it’s default to get data from “Details” tab.
If this parameter is “No”, invoice data are get from “Details” tab; then, only one invoice is input in a voucher.

Display quantity and unit price on sales service invoice

If this parameter is "1 - Yes":

    In “Service sales invoice” and “Service sales return invoice” screens (in case transaction currency is the same as base currency): there will be 3 more columns “UOM”, “Quantity” and “Sales Price” before “Amount” column;
    In “Service sales invoice” and “Service sales return invoice” screens (in case transaction currency is different from base currency): there will be 4 more columns “UOM”, “Quantity”, “FC Sales Price” before “FC Amount” column and “Sales Price” before “Amount” column.

List of Exchange Rate Difference Reversing Accounts

According to the Circular No. 201/TT/BTC, cash account and short-term receivable/payable accounts which are evaluated the exchange rate differences at the end of fiscal year must be created the reversing entries of the difference entries at the beginning of next fiscal year to delete the balances in relating accounts (111, 112, 131_, 331_, 413). There is no reversing entry for long-term receivable/payable accounts.
It means that the reversing entry is not created for all accounts with foreign currency balances, it’s required to define list of accounts which need creating reversing entry at the beginning of next period.
Definition: Key in the general or sub-accounts separated by comma (,)
See “Reversing Entry of Exchange Rate Difference Entry” guides in General Ledger or Accounts Receivable/Payable modules for more details

Inventory Parameters

Average Cost Method

Select to calculate average cost by 2 options: one cost for all sites and separate cost for each site.
Calculate Common Average Cost of All Sites in Each Site Group
This parameter is selected to calculate cost of a group of sites; designed to be used in case of calculating cost of each group of sites in a separate unit (or a group of sites that are in the same properties such as site of goods on promotion). All processes will only affect that unit.

FIFO Cost Method

Select to calculate FIFO cost by day or by month:

    - “By day”: FIFO for each issue voucher daily. If the quantity in an FIFO issue voucher is bigger than the remained quantity in warehouse, the system cannot calculate FIFO price for this issue voucher and gives a message: “Not enough item to issue” for user to recheck.
    - “By month”: Receipt or inventory quantity is required to adapt issue quantity in month only.

Assign Difference Cost to the Last Issue

If this parameter is “Yes”, the system will auto assign the different amount calculated by average costing method to the last issue voucher.

Create Difference Issue of an Item when There is no Issue in Period

If this parameter is “Yes”, the system will auto create an issue voucher for inventory cost difference of an item even if there is no issue for that item in period but there is difference.

Note: In case “Average Cost Method” parameter is “One cost for all sites” and when running “Calculate Average Cost” function, the “Create Difference Issues” option is defined as “2 - All”, this parameter must be “Yes” so that the inventory data is still reasonable and balance.

Calculate FC Average Cost (Yes/No)

This parameter shows that average cost is calculated in foreign currency or base currency. If it is calculated in FC, it is required to input arising in FC.

Tracking Slow Moving Item

If the “Tracking slow moving item” parameter is “Yes”, “Last Issue date” will be also updated to next year when run “Transfer stock to next year” process.
The rule of processing: base on the last issuing date in stock card data.

Note:
About the items which have arising of receiving in period but has not yet made issue, the process will update the “Last Issue date” by receiving date.
About the opening balance stock items which have no arising of issuing in period, the process will update the “Last Issue date” by “Last Issue date” of opening balance.

Automatic Lot Code

About Auto lot generation tool:
If the item is checked in “Lot Control” and “Allow creating lot when receiving” options (in Item List), it is unnecessary to define lot information in “Lot Maintenance”. Lot can be defined during data entries of following vouchers.

    - Domestic Purchase Invoice;
    - Import Purchase Invoice;
    - Purchase Receipt – Direct Issue Voucher;
    - Receipt

AI ERP supports auto lot generation. User inputs item in detail grid, then click on - “Lot maintenance” to open “Lot maintenance” form. This form will show grid lines for items with auto lot definition and lot IDs wrongly shown. If “Lot ID” typed in detail does not exist, the system will auto generate lot id by the rule defined in system parameters. Lots are generated by “Automatic Lot Code”: the system uses 04 characters “d,m,y,#” with replacement as follows:

    - "d" for day;
    - "m" for month;
    - "y" for year;
    - # for ordinal number (auto increase In case of multi-lot receiving in a day).

Example:
We configured the parameter “Automatic Lot Code” as “LO.ddmmyyyy.##” In the day Oct 15th 2012 we have 2 receipts, the first will generate lot ID as “LO.15102012.01”, and the second will generate lot ID as “LO.15102012.02”.

Note:
After user presses “OK” in “Lot maintenance” form, a new lot is created although the voucher has not been saved yet. Therefore, do not use Auto lot function when editing open vouchers to avoid new lot ID; key in lot ID manually in this case.

Coefficient field round

Define the odd number when creating coefficient; default value is 4.

Multi-level in BOM

There are 2 options:

    - If “Yes”, the system will calculate to the last components of BOM in finished goods;
    - If “No”, the system will consider the BOM in finished goods only.

Costing by unit

This is a hidden parameter with 2 options:

    - If “Yes”, the “Unit” field will appear on cost factor list, cost aggregation, calculation and report so that cost can be calculating by unit;
    - If “No”, the “Unit” field will not appear and cost will be aggregated and calculated for all units.

Fixed Asset Parameters

Depreciate the Whole Remaining Amount in Last Period

If this parameter is “Yes”, the remaining value of asset will be depreciated all in the last period of fixed asset depreciation.
This parameter is valid only when “Depreciation Ending Date” information is defined in “Fixed Asset Information” entry.

Human Resources - Payroll - Insurance Parameters

SI Agency Name

Used in print form of insurance reports as Written Request

Apply State Salary Scale

If this option is "Yes", the system will calculate base salary of employees based on salary coefficient, allowance coefficient and common minimum wage levels. On the contrary, the system will use the user-defined salary.

Monthly SI Report Dates

Used as default report date in "Employee SI History" input

Employee Leaving/ Working Status

These 2 parameters are used to inform that the employee is working or has left job, used to filter data in reports.

Min Time Block (Seconds)

This option is used to remove the scrap timekeeping data during workday computation.

For example: Min time block is 120 seconds. An employee has checked in 2 times at 07:58 and 08:00 (maybe check in again), the system will keep the check in time at 7:58, remove 08:00.

Number of hours per day

This parameter is used exchange timekeeping data from hour-minute into day during workday and leave computation. It is usually 8 hours per day. 2 decimal digits are allowed.

Annual leave sign

This parameter is used to inform which timekeeping sign is annual leave, support in annual leave computation and annual leave reports.

Number of annual leave in month

Used to define the number of annual leave days of employees who have not worked for a full year. One day of annual leave for a working month (from the official hire date).

Number of annual leave in year

Used to define the number of annual leave days of employees who have worked for a full year and over. It supports in annual leave reporting and calculating the remaining days of annual leave in year.

Years of seniority to be added a day of annual leave

This is the number of working years that an employee is added a day of annual leave. It is 5 years (from the official hire date) according to the current regulation.

Working Days per Month

This parameter is used to calculate employee salary by working time in case you have not define the "Regular Days" in "Salary Computing Period".

Default Status of Hired Applicant

Used as default value in updating the status of applicant profile in the function "Recruitment Decision"

Default Status of Application Form when Received

Used as default value in "Applicant Profile Input", but editable; Or in case of delete all test/interview result, it is also used for update the profile status.

Operations

Edit parameter value

To edit a parameter value, please do the following steps:

    - Click optional button - Edit,
    - “Edit Parameter” screen appears with “Value” field be active,
    - Input new parameter value,
    - Click <Save> to accept new value.

Reset default value

To restore the default value of the program, please use the “Reset default value” function

Note that these optional parameters are common to all units in the same database.
