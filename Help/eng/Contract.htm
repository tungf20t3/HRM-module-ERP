Contract List

Field descriptions

Customer

Customer ID & name, extracted from Customer List

Employee

This information is used in sales contract only. Sales employee information is extracted from Employee List.

Department

The department implementing the contract; extracted from Department List

Date Signed

The date when the contract is signed

Contract Number

Ordinal number of the contract; manually input

Balance Tracking

The option defines whether the system will keep tract of contract balance or not. If you select balance tracking, the contract opening balance entry will be enabled to input. On the contrary, you cannot input contract opening balance.

Start Date

The date when the contract is effective

Finish Date

The date when the contract is finished

Currency

Monetary unit used in the contract

FC Amount

Value of contract in the currency used in the contract

Base Currency Amount

Value of contract exchanged to base currency

Group 1/ 2/ 3

Groups classified by characteristic 1/ 2/ 3 are used in report filter conditions or creating reports classified by contract. These fields are extracted from groups with group type as 1/ 2/ 3 in Contract Group List.

Some tasks in list maintenance

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.
The data importing process:

    - Enter Contract List;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to upload data to the system, click the icon , select the file, the system will get data from this file to Contract List.

Notes of inputting data in template file:
- Contract: if some code already exists in Contract List, the system will replace by one in the template file (if the copy type is "Overwrite"). The new codes are certainly added to Contract List;
- Description, Other Name, Contract Number: manually input with the same length as one in add (edit) screen in list;
- Date: Signed, Start Date, End Date: input in date format;
- Balance Tracking: accepted with value 1 or 0;
- Currency, Customer, Employee, Department, Group 1/2/3: must be one existing in the corresponding list with the same length in add (edit) screen in list;
- FC Amount, Base Currency Amount: input in number format, 14 digits maximum and 2 decimal digits (standard format).

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field;
- Invalid data type (for example: 1,0 or yes/no), in this case, the system indicate the specific invalid cell;
- Duplicate or nested codes, in this case, the system indicate the specific invalid cell.

Other notes:
- If you do not have the "New" right in "Customer List", you cannot import new data from spreadsheet to this list;
- If you do not have the "Edit" right in "Customer List", you cannot edit data in this list from spreadsheet.
