Lot Maintenance

Lot maintenance is designed for stock controlling by lot purpose.

Field descriptions

Item

Item received in lot; an item can be received in many lots if there is check in “Lot control” option in Item List.

Receipt Date

The date when the lot is received

Manufacturing Date

More information about the date when the item is manufactured

Expiry Date

The date when the item’s useful life is expired; if the lot assignment in Item List is “When received” option, the expiry date will be equal to receipt date plus “useful life (days)” of item.

Warranty Date

Warranty date of item will be system-generated by adding “warranty days” to receipt date.
If the lot assignment in Item List is “When used” option, the expiry date and warranty date must be manually input.

Create lot during voucher entry

If the item is checked in “Lot Control” (in Item List), you can define lot information in this “Lot Maintenance”. Besides, lot can be created during data entries of following vouchers:

    - Domestic Purchase Invoice;
    - Import Purchase Invoice;
    - Purchase Receipt – Direct Issue Voucher;
    - Receipt

If the item is checked in “Lot Control” and “Allow creating lot when receiving” options (in Item List), the system will auto-generate lot code in the format defined in "Automatic Lot Code" parameter to put in the voucher.

In voucher entry screen (grid view), click the icon - “Lot maintenance” to open “Lot maintenance” screen. This screen will only show the detail lines of items with "Auto creating lot when receiving" option and the current code of lot is incorrect. If the “Lot” typed in detail does not exist, the system will auto generate lot code by the rule defined in system parameters.
Lots are generated by the format defined the “Automatic Lot Code” parameter with 04 special characters “d,m,y,#”:

    - "d" stands for day;
    - "m" stands for month;
    - "y" stands for year;
    - "#" stands for ordinal number (auto increase in case of multi-lot receiving in a day).

The character “dmy” must be lower case, if they are in upper case, the system will understand as a part of lot code.

Example:
We configured the parameter “Automatic Lot Code” as “LO.ddmmyyyy.##” In the day Oct 15th 2012 we have 2 receipts, the first one will have lot ID as “LO.15102012.01”, and the second one will have lot ID as “LO.15102012.02”.

Other related information:
- Receipt Date: being default as voucher date in the first data entry, but editable;
- Lot Description: system-generated in the form of Lot + lot ID + [Receipt Date] + voucher date, but editable;
- Manufacturing Date: being default as Receipt Date, but editable;
- Expiry Date: being equal to "Receipt Date" plus "Shelf Life Days" (if the "Assignment" is "1 - When Received");
- Warranty Date: being equal to "Receipt Date" plus "Warranty Days", but editable.

Note:
After you press “OK” in “Lot maintenance” form, a new lot has been created in the list although the voucher has not been saved yet. Therefore, when editing an unfinished voucher, you should manually input the created code of lot to avoid creating another code in the system.

Some tasks in list maintenance

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.
The data importing process:

    - Enter List;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to List.

Notes of inputting data in template file:
- Item Code: check the existence in Item List;
- Item and lot: check the existence of these 2 codes in Lot Maintenance; if the input codes already exist in Lot Maintenance, the system will replace by one in the template file (if the copy type is "Overwrite"). The new codes are certainly added to List. The length of code is the same as one in add (edit) screen;
- Receipt Date, Manufacturing Date, Warranty Date, Expiry Date: input in date format;
- Description, Other Name, Part Number: manually input with the same length as one in add (edit) screen.

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field;
- Invalid data type (for example: 1,0 or yes/no), in this case, the system indicate the specific invalid cell;
- Duplicate or nested codes, in this case, the system indicate the specific invalid cell.

Other notes:
- If you do not have the "New" right in "Lot Maintenance", you cannot import new data from spreadsheet to this list;
- If you do not have the "Edit" right in "Lot Maintenance", you cannot edit data in this list from spreadsheet.
