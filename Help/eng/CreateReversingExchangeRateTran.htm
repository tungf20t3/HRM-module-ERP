Reversing Entry of Exchange Rate Difference Entry

Path: General → Closing Transaction → Reversing Entry of Exchange Rate Difference Entry
The reversing entry is used to create an entry in opposite direction to the exchange rate difference entry created at the end of previous period.
It is created in the same way to the exchange rate difference entry at the end of the previous period.


Note:
- This entry can not be created/deleted after data closing date/time;
- Only records defined in “Exchange Rate Entry Definition” menu with “Account” belonging to “List of Exchange Rate Difference Reversing Accounts” parameter are filtered to show in this screen;
- Logic: Basing the selected entries and the required period to create reversing entries, the system will search the entries with voucher code having exchange rate difference entries at the end of previous period (PK6) in ledger, record these entries in the opposite direction (debit to credit and vice versa). The “Voucher Date” is the first date of the next month.

For example, the required period is 01/2009=> Voucher date of the reversing entry will be 01/02/2009
