Description of Transaction Codes in Sales Invoice

There are many transaction codes in Sales Invoice to use applications of the system in different environments. System will be defined in accordance with the applications of modules.
These are some cases:

Note:
Only Sales Management: Sales is involved in on hand inventory, not financial inventory
Only Accounting: Accounting is involved in financial inventory and accounting entry, not on hand inventory
Both Sales + Accounting: Sales is involved in on hand inventory, accounting is involved in financial inventory, accounting entry

Entry screen

Field descriptions - format view

Customer

Customer ID and name, extracted from Customer/Supplier List, is usually used in debt control in case of Issue related to debt transactions as lending items from other units. In case transactions which are not related to debt control, this information is used to control internal objects as production line, shop floor…

Transaction Code

The default transaction code of sales invoice is 2 - “Immediate SO Invoice”.

Note:
If sales invoices, domestic/import purchase invoices are wrongly entered following the data entry process, the system will not keep track of inherited data in process of invoice.

For example:
Creating sales invoice transaction code 3, retrieve data from sales order/ pick list, then change transaction code to 2. At that time, the invoice creating process is cancelled, the data inherited from sales order/ pick list are 0 (sales invoice transaction code 2 cannot be retrieved data from sales order/pick list).

Debit Account

Debit Account in revenue record

Credit Term

Extracted from “Customer List” but editable

Invoice Number / Serial Number

Invoice number and serial number is extracted from “Voucher Book List”. If “Voucher Book using definition” is not declared, they will be input by users.

- If the invoice is numbered by voucher book which has been defined the "Invoice Form", this "Invoice Form" will be used in tax reports.
- If the invoice is numbered by voucher book which has not been defined the "Invoice Form", the system will get the "Default Output Invoice Form" in optional parameter.
- If the "Default Output Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Invoice Date / Posting Date

The date the invoice is created and the date the invoice is posted. Invoice date is un-editable if extracted from SO.

Ex. Rate

Exchange rate when invoicing in foreign currency. Choose FC code and enter exchange rate. If currency is base currency, Ex. Rate will be 1 and not be updated. On the Contrary, it is not base currency; it is extracted from the latest Ex. rate in “Exchange rate list” by default. When Ex. Rate changes, “amount”, “tax amount” and “total amount” field will also change.

Tax code

Select tax code (tax rate) of item

Tax Account

Extracted from “Tax Rate List” and general information of sales invoice; editable

Tax Authority

Tax authority (branch) relating to goods sold taxing following VAT regulations. This is an optional field due to management requirements of each company.
“Tax Authority” is defined in “Supplier/ Customer List”.

Note: in case tax account is a receivable account, “Tax authority” is considered as a receivable object and required.

Amount

Total goods amount

Tax Amount

Note: if definition “calculate tax on promotion goods” is “yes”, taxable amount will be included promotion revenue.

Total Amount

Total payable goods and tax amount

Status

There are 3 statuses:

    - No Action: the voucher has not been posted to any book;
    - Release Later: If there is VAT data, the invoice will be shown in VAT reports. If this is AR/AP invoice, it will be shown in AR/AP by invoice reports;
    - Release: the voucher has been posted to all related books.

Field descriptions - “Details” Tab

Item Code

Item code in invoice; extracted from “Item Master”

UOM

UOM of item, extracted from item master by default and it can be changed in case of condition that “controls multi UOM” configuration and in UOM conversion List declaration for this item.

Promotion

In case of sales with promotion, check in this box. This option is defined to hide (unhide) in “System/ List Maintenance/ Optional Parameters”.
The revenue of goods on promotion will be excluding for calculating VAT on invoice (Show Sales Promotion)

Note: If goods on promotion are invoiced in another voucher, do not check in this box.

Site

Sites issuing goods; extracted from “Site List”

Location

Location of goods in site; extracted from “Location List”, used in case you wish to control items by location.

Lot

Lot of goods; required for items with lot control. Lot declaration has been done in Inventory Management.

Quantity

Quantity of items in invoice

SP.(specific) Cost

If you check in this option, you have to manually define “cost of goods sold” and enter it in the system. Otherwise, the system will auto calculate COGS following the costing method of goods defined in Item List.

Sales Price

Price of item in GL currency.

Discount

Discount amount

Sales Account / Discount Account / Inventory Account / COGS Account

Sales, Discount, Inventory, and Cost Price Accounts are accounts of goods in invoice; declared in “Item Master”.

"Note:
If discount account is input, discount entry will be separately recorded. On the contrary, revenue will be decreased equal to goods amount minus discount."

Other Information - Field descriptions

Customer/ address/ tax code

In case of customer name show on VAT invoice (for VAT reports suppose) and AR/AP customer are not the same, users can input value for this field.

Customer name: Show customer name in general information and editable. Customer Name will be posted to customer column in output VAT report.

Customer Address: Show customer Address in general information and editable. Customer Address will be posted to Address column in output VAT report.

Tax code: Tax code of “customer ID” appears in general information and editable. Tax code posts to Tax code column in output VAT report.

Item group

Name of goods, services in output VAT list. If you do not input date in this field, the system will auto get the item with the greatest sales figures.

Note

Information enter in tax report (output VAT invoice list)

Some common tasks in creating Sales Invoice

Create Sales Invoice - Extract from Pick List

Create a new Sales Invoice, click on function “ - Extract Data", select “Extract from Pick List” function and type start date. Lookup and choose Pick List in Pick List selecting screen.
Input invoice quantity then exit Pick List screen. Data in selected rows in this list will be system-updated to Sales Invoice.

Create Sales Invoice - Extract from Order

Create a new Sales Invoice, click on function “ - Extract Data", select “Extract from Order” function and type start date. Lookup and choose Orders in Order selecting screen.
Input invoice quantity then exit Order screen. Data in selected rows in this list will be system-updated to Sales Invoice.

View Receipt

If you want to refer specific calculated cost in some Receipt, press the icon to filter these Receipts.

Import data from excel file

The data importing process:

    - Enter Sales Invoice;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to grid view of the invoice.

Notes of inputting data in template file:
- Unit, Debit Account, Customer: Account must be detailed account;
Unit is one of the assigned units with unit locking date before invoice date;
- UOM: is one defined in list. If an item has many units of measure, define in Unit Conversion List;
- Site: input sites belonging to assigned units;
- Location: required when site having location track;
- Lot: required when item with lot control;
- Sales Account: get from one defined in item list; otherwise the system will get from the imported file;
- Currency: if voucher consists of many rows, the currency of the first row will be considered as currency of the voucher;
- Site, Lot, Location, Currency, Tax Code, Tax Authority, Job, Department, MO, Product, Contract, Expense, Loan Contract: must be one existing in the corresponding list with the same length in add (edit) screen in list;
- Tax Account: must be detailed account. If there is no tax account in the imported file, the system will get one in accordance with tax code;
- Date: input in date format. Data closing date and voucher locking date must be before voucher creating date;
- Quantity: input in number format, 9 digits maximum and 3 decimal digits (standard format);
- Price: input in number format, 12 digits maximum and 4 decimal digits (standard format);
- Amount: input in number format, 14 digits maximum and 2 decimal digits (standard format). The system will import the right amount input in the template file, not recalculate.

The system will give error message in some following cases:
- Data in column containing customer, supplier, account do not exist in the corresponding lists. Account is general account. In this case, the system will indicate the specific invalid cell;
- The required fields (Invoice Number, Unit, Tax Authority,...) are blank;
- String data is longer than the available length of field; the field length is the same as one manually input.

Other notes:
- Cannot import data after system data closing, unit or voucher locking date (the data closing date after the beginning input date);
- The current user must have the "New" right in "Sales Invoice" menu;
- The current user must have the "New" right in "Access Right by Unit" menu;
- Enable to import data for many units. If the data of the current unit has been closed, the system will give warning message when you click the icon; if data of other units has been closed, the system will also give warning message during importing process and indicate the specific invalid cell;
- Check the duplicate vouchers by number only (if available);
- If the invoice currency is the same as base currency, it is required to input exchange rate as 1;
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
