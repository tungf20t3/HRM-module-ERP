Invoice Opening Balance

To businesses having payables and suppliers tracked by invoice, this function is used to input opening balance of available invoices before using the program so that the payables by invoice of supplier can be exactly calculated and reflected.

Invoice opening balance screen

Invoice opening balance entry screen

Field Descriptions

Invoice Date / Invoice No.

Date and number of invoice

Customer

Code of supplier in invoice

AP Account

Account used to control payables to supplier, extracted from Chart of Accounts which is defined as AR-AP account

Description

Information related to invoice being entered

Currency

Currency in opening payable invoice

FC Total Amount

Invoice amount in foreign currency

Exchange Rate

Exchange rate of the invoice

Total Amount

Invoice amount in VND

Payable Amount

Payable amount in invoice

Paid Amount

Amount paid for invoice

Remaining Amount

Amount to be paid for invoice (in transaction currency); equal to Total Amount minus Paid Amount

Debit Term

Payment term of the invoice

Write-off

This option informs that the invoice is being tracked or not. It will be auto-changed to 1 when the remaining amount is 0. The written-off invoice will not appear in next paying process.

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

The data importing process:

- Enter "Invoice Opening Balance" menu;
- Click the icon to download template file to put data in;
- After keying data in the spreadsheet, to upload data to the system, click the icon , select the file, the system will get data from this file to opening balance file in the system.

Notes of inputting data in template file:
- Unit, AP Account, Customer (Supplier), Sales Employee, Currency, Debit Term: Unit is one of the assigned units; account must be detailed account and AP account. The input codes must be ones existing in the lists with the same length in add (edit) screen in lists;
- Total Amount, Paid Amount: number format, 14 digits maximum and 2 decimal digits (standard format);
- Exchange Rate: number format, 12 digits maximum and 2 decimal digits (standard format);
- Invoice Number, Description: manually input with the same length in add (edit) screen in lists;
- Posting Date, Invoice Date: date format;
- Write off: accepted with the value 1 or 0.

The system will give error message in some following cases:
- Data in column containing customer, supplier ID and account codes do not exist in chart of accounts; being neither detailed account nor AP account. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field which is the same as one manually input.

Other notes:
- Cannot import data after system or unit data closing (the data closing date after the beginning input date);
- The current user must have the "Edit" right in "Invoice Opening Balance" to edit the balance rows available in data (select the copy Type "Overwrite");
- The current user must have the "New" right in "Invoice Opening Balance" to add the new balance rows into data (select the copy Type "No");
- The current user must have the "Edit" right in "Access Right by Unit" to edit the balance rows available in data (select the copy Type "Overwrite");
- The current user must have the "New" right in "Access Right by Unit" to add the new balance rows into data (select the copy Type "No");
- Enable to import data for many units. If the data of the current unit has been closed, the system will give warning message when you click the icon;
- If data of other units has been closed, the system will also give warning message during importing process. In this case, the system will indicate the specific invalid cell;
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
