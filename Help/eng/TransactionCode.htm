Transaction Code List

Path: Modules → Inventory Management → List Maintenance → Transaction Code List

Field descriptions

Voucher Code

Voucher code is selected from available list designed in system, can not be added or edited.

For example, issuing transactions can be Pick List, Issue, Return to Supplier…

Voucher Type

Transaction type is selected from available list designed in system, can not be added or edited. Transaction type defines actions in voucher.

For example, there are 2 transaction types of Pick List: “Issue Transaction” and “Issue Order”.

Transaction Code

Transaction code can be added or edited depending on business statistic. It cannot define actions in voucher but transaction type.

For example, Pick List can be divided into many codes to many factories.

Description

The description of transaction; e.g. “Issuing for factory A”, “Issuing for factory B”

See “Hints and Tips in List Setting” for details of common tasks in list maintenance
