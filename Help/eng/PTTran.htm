QA Transaction

Entry screen

Field descriptions - Format view

Receipt Number

Number of Receipt having items in “Open” status; extracted from Receipt list created.

Date

Date of Receipt having items in “Open” status; extracted from Receipt list created.

QC Staff/Memo

Staff in charge of checking quality and related description; free enter.

QC Slip No.

Number of QA Transaction; free entered by user or automatically numbered by program depending on declaration in system

QC Slip Date

Date when QA Transaction is created

Status

Status of QA Transaction is automatically processed by program. There are 02 statuses: “Open” and “QC Finished”.

Field descriptions – Grid view

Item Code

Codes of received items selected in “QC” field in Purchase Receipt.

UOM

Unit of measurement of item; extracted from “Item Master” or changed by conversion of units of measurement in some special cases.

Site ID

Site receiving items, extracted from “Site List”

Location ID

Location where items are stored in site, extracted from “Location List” and used in case enterprise has controlled items by location.

Lot ID

Number of lot of received items; used in case enterprise has controlled item by lot. Lot declaration is implemented in Inventory Management.

QC Result

There are 5 types of result that have been declared by program: 1 - Approve, 2 – Return to Supplier, 3 - Reject, 4 – Sample, 5 – Receipt without QC.

Description

Description of QC result

Quantity

Quantity of items received required field

Reason

Reason of issuing and receiving in next process; extracted from Reason List.

Note

Other description of items in checking

Some operations in creating QA Transaction

Create Purchase Order by Extract data from Purchase Receipt

Create new QA Transaction, click " - Extract Data from Purchase Receipt”, Input the date from which the Purchase Receipt is selected, then click to lookup Purchase Receipt (Select Purchase Receipt). Only Purchase Receipt with “Receipt” status and detail item in “For Inspection” of the selected supplier appears in the screen. Input inspected quantity and QC Result. Result code included:

    - 1 Approve
    - 2 Reject to supplier
    - 3 Reject as Scrap
    - 4 Sample
    - 5 Receive without QC

Click "OK" to exit the Purchase Receipt list screen. The selected rows in the Purchase Receipt will be systematically updated to QA Transaction.
