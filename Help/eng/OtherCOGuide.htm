Common notes before applying advanced costing

Definition in related lists

Optional Parameters

It is required to define for which object costing calculation is applied

    - Product;
    - Department;
    - Manufacturing Order (MO);
    - Multi-level BOM.

More definition of management information in voucher entry

Menu: System/ List Maintenance/ Voucher Information
According to the costing object defined in optional parameter, check information about department code, MO number, product code in Receipt, Issue, General Voucher.
To Receipt, carefully check if the costing object has been selected group when saving to ledger so that the system can update the right product cost. This check should be done for Issue, General Voucher, Allocation & Closing Entries to avoid wasting time to check process error (although there are functions to check costing steps)

Chart of Accounts

WIP account: it is required to create WIP accounts (154.1, 154.2,154.n...) if the costing calculation has many steps.
To cost accounts, each account or account couple (corresponding accounts) must be unique, because each cost factor is unique and corresponding to each account or account couple.

Item List

When defining items related to finished products, WIP, notice:
Define item type 41, 51 for WIP, finished products. If using multi-step costing with WIP accounts 154.1, 154.2, define item account of WIP as 154.1
In multi-step costing, item groups should be created in multi-step so that the period-end issue cost will be calculated by group instead of all items.

Receipt voucher entry

Product Receipt voucher must be used type 4

Costing by unit

AI ERP helps to select if you want to have cost calculated by unit or not
