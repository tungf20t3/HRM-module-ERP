Stock Transfer In

Entry Screen

Field descriptions

Site ID

Receiving site ID (target)

Deliverer

Information of the person who delivers goods; extracted from “Contact” field in Customer/ Supplier List and editable

Trans. Code

Transaction code is used to classify usage of Transfer. There are now 2 codes: “Internal Transfer” and “Misc. Receipt”. In data process, all transactions have the same action.

Voucher No. / Voucher Date / GL Date

Information of voucher number, the date when voucher is created and posted to GL. Voucher number can be system-generated (figure form) or manually input. Voucher date and GL date can be different.

Ex. Rate

Used in case transaction is input in foreign currency, extracted from latest “Exchange Rate” list before creating date.

Item code / Site ID / Location ID / Lot ID

If stocks are controlled by lot, lot ID will be extracted from Stock Transfer and un-editable. In case stocks in receiving site are also controlled by location, location in receiving site will be extracted from Stock Transfer but editable in Stock Transfer In.

The requirement for the location and lot input depends on definition in “Item Location Tracked by Accountants” and “Item Lot Tracked by Accountants”.

Job

Extracted from “Job List” if Issue relates to jobs, events

Reason Code/ Dr. Acct. / Cr. Acct.

Transfer form and entry information
Reason code is extracted from “Reason List”. Debit account is extracted from item account but can be entered if this item code is declared to allow changing item account. Credit account is extracted from “Account” field in selected Reason code but can be entered.
“Transfer” and “Transfer In” entries can be posted to general ledger. According to the entries assignment in your enterprise, these 2 entries will be chosen 1 to post to GL. Otherwise, there will be coincident entries.

Avg. Cost

This is receiving cost in case item cost is calculated by average method. It is used to reassign item cost in Stock Transfer In when item cost in Stock Transfer is changed because of running cost calculating function at the end of month.

Status / Action

Current status of Stock Transfer In; system-generated when you choose action when saving voucher. There are 4 actions of Stock Transfer In: “No action”, “Open”, “Release Later”, and “Posted to GL”.
