Customer List

Field descriptions

Customer Name/ Other Name

Name of customer and English name (if any)

Address/ Partner

Contact address and contact person of customer; used as default information in voucher entry and printing

Tax Code

Tax code of customer; used in VAT reports and invoice print forms

Sales Employee

Sales employee who is in charge of this customer

Default Account

Default AR account when recording receivables from this customer

Default Term

Default payment terms granted from this customer

Credit Limit

It is base to suspend sales order if this customer’s credit is over limit. If total receivables are greater than credit limit, the sales order of this customer will be auto defined as “Suspended” status when being saved. Credit limit is now used in Sales management module only.

To Date

The last effective date of “Credit limit”

“Customer”, “Supplier” or “Employee” option

The checkbox options define to manage customer, supplier or employee in list. Based on this option, if voucher entries or reports are grouped in accounts receivable or payable, the system will search for customer or supplier or employee.

Customer Group 1/ 2/ 3

Customers are classified by group 1/ 2/ 3, used to filter data in reports or create customer classifying reports, extracted from Customer Group List.

Customer Price Class

It is used to define in which price class the customer is grouped. Each price class is applied a different discount. If there is no customer price class selected, the discount by customer price class will not be applied when calculating discount in sales transaction.

Phone/Fax Number/ Email/ Home page

Phone number/ fax number/ email address, website of the customer; reference information

Bank Account/ Bank Name/ City/Province

Bank account/ bank name/ city/province of the bank which the customer registers account. These fields are used to print bank vouchers.

Some tasks in list maintenance

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.
The data importing process:

    - Enter Customer, Supplier List;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to upload data to the system, click the icon , select the file, the system will get data from this file to Customer, Supplier List.

Notes of inputting data in template file:
- Customer ID: one ID is unique in this column. If some ID already exists in Customer List, the system will replace by one in the template file (if the copy type is "Overwrite"). The new IDs are certainly added to Customer List;
- Sales Employee, Payment Term, Group 1-2-3, Customer Price Class: must be ones existing in corresponding lists;
- Customer Name, Other Name, Address, Tax Code, Partner, Phone Number, Fax Number, Email, Homepage, Bank Account, Bank Name, Province/City: manually input in form of string with the same length as one in add (edit) screen;
- Default Account: must be detailed and AR/AP account;
- Credit Limit: input in number format with the same length as one in add (edit) screen;
- To Date: input in date format;
- Customer, Supplier, Employee: accepted with value 1 or 0.

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field;
- Invalid data type (for example: 1,0 or yes/no), in this case, the system indicate the specific invalid cell;
- Duplicate or nested codes, in this case, the system indicate the specific invalid cell.

Other notes:
- When getting data, if there is no choice of "Customer", "Supplier" or "Employee" option in the spreadsheet, the system will auto-check all 3 options;
- If you do not have the "New" right in "Customer List", you cannot import new data from spreadsheet to this list;
- If you do not have the "Edit" right in "Customer List", you cannot edit data in this list from spreadsheet.

Refer to “Hints and Tips in List Setting” for details of common tasks in list maintenance
