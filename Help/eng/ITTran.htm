Stock Transfer

Entry Screen

Field descriptions

Source/Destination Site

Issuing site code (source) and receiving site code(destination); only appear sites belonging to the current unit

Transaction Code

Stock Transfer is now designed with 1 transaction code "Stock Transfer Out" only.

Voucher Number

Voucher number can be system-generated (in number format) or user-defined in Serial Voucher Number List, but can be manually input.

Voucher Date / Posting Date

The date when the voucher is created / posted to GL. Voucher date and posting date can be different.

Ex. Rate

Used in case transaction is input in foreign currency, extracted from latest “Exchange Rate” list before creating date.

Item Code / Site / Location / Lot

If stocks are controlled by location or lot, it is required to input source location and lot. In case stocks in destination site are also controlled by location, it is also required to input destination location.

Reason Code / Debit Account / Credit Account

Transferring form and double-entry information
Reason code is extracted from “Reason List”. Credit account is extracted from issued item account but editable if this item is declared to allow editing item account. Debit account is extracted from “Account” field in selected Reason code but editable.
“Stock Transfer” and “Receipt” entries can be posted to general ledger. According to the entries assignment in your enterprise, these 2 entries will be chosen 1 to post to GL. Otherwise, there will be coincident entries.

Quantity

Quantity of item transferred; if the item is FIFO cost, quantity must not be 0.

Specific Cost

This check-box option is used to choose a specific issuing price to item. This price will not be changed during cost calculating.

Job

Extracted from “Job List” if Stock Transfer relates to any job, event

Step (1/2)

The stock transfer can be operated in 1 step or 2 steps. If transfer in 1 step, the system will create “Stock Transfer In” in target site which must not be adjusted. If transfer in 2 steps, the system will create “Stock Transfer In” which voucher number and date are the same as ones in "Stock Transfer" so that user can adjust actual quantity in “Transfer In” at target site.
One-step transfer is used in case in-out data are surely the same. Two-step transfer is used in case warehouse-keeper in target site wants to confirm the actual received quantity (for loss or damage during transportation).

Note:
- In case of two-step transfer, if there is any adjustment in Stock Transfer In, issued quantity in site card of source site will be different from received quantity in site card of target site.
- The parameter of 1 or 2 steps is unchangeable.

Some common tasks in voucher entry

Import data from excel file (detail importing)

The system is enabled to import data from excel spreadsheet to save hours of manual effort.
The data importing process:

    - Enter Stock Transfer;
    - Click the icon , the system allows user to download template file to put data in;
    - After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to grid view of the voucher.
    - Select copy type 1 - Overwrite if you want to replace the available data; if you select copy type "0 - No", the system will insert the new data only.

Notes of inputting data in template file:
- Specific Cost: accepted with value 1 or 0;
- Item Code, UOM, Source Location, Destination Location, Lot, Reason Code, Debit/Credit Account, Job, Department, MO, Product, Contract, Expense, Loan Contract: must be one existing in the corresponding list with the same length in add (edit) screen in list;
- UOM: the system will use the standard UOM of the item if you let it blank in template file;
- Source / Destination Location: input only when item with location track;
- Lot: input only when item with lot control;
- Quantity: input in number format, 9 digits maximum and 3 decimal digits (standard format);
- Price: input in number format, 12 digits maximum and 4 decimal digits (standard format);
- Amount: input in number format, 14 digits maximum and 2 decimal digits (standard format).

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists (Item, Site...). In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field (maximum length can be manually input.

Other notes:
- The function is only supported to get data for each voucher, used in case of frequently inputting the similar periodic vouchers.
- If you do not have the "New" right, you cannot import new data to voucher detail grid;
- It is not required to keying object name in lists, the system will auto update following their codes.

Import data from excel file (mass importing)

The system is enabled to import data from excel spreadsheet to save hours of manual effort.
The data importing process:

    - Enter Stock Transfer;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will import data from this file.

Notes of inputting data in template file:
- Voucher Number: input in form of string, 12 characters maximum;
- Voucher Date: input in date format. Voucher date must be after data closing date, unit and voucher locking date;
- Specific Cost: accepted with value 1 or 0;
- Customer, Item Code, UOM, Site, Reason Code, Debit/Credit Account, Job, Department, MO, Product, Contract, Expense, Loan Contract: must be one existing in the corresponding list with the same length in add (edit) screen in list. Account must be detailed account;
- Item Code: to item with monthly or moving average costing method, if the "Calculate Common Average Cost of all Sites in each Site Group" parameter is "Yes", the source and destination sites must be in the same group;
- Step: accepted with value 1 or 2;
- Source/Destination Site: input sites belonging to assigned units;
- Location: input only when item with location track;
- Lot: input only when item with lot control;
- Quantity: input in number format, 9 digits maximum and 3 decimal digits (standard format);
- Price: input in number format, 12 digits maximum and 4 decimal digits (standard format);
- Amount: input in number format, 14 digits maximum and 2 decimal digits (standard format).

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists (Item, Site...). Account does not exist in chart of account or is general account. In this case, the system will indicate the specific invalid cell;
- The required fields (Voucher Number, Voucher Date, Unit...) are blank;
- String data is longer than the available length of field (maximum length can be manually input.

Other notes:
- Cannot import data after system data closing, unit or voucher locking date (the data closing date after the beginning input date);
- Do not input exchange rate if the "Currency" column is blank;
- The current user must have the "New" right in "Stock Transfer" menu;
- The current user must have the "New" right in "Access Right by Unit";
- Check the duplicate vouchers by number only (if available);
- In order to import a voucher with many grid rows, you can input many rows in spreadsheet with the same voucher number and date. The system will create one voucher with these grid rows;
- Enable to import voucher in "No Action" status only; you can edit the imported voucher to post to ledger;
- It is not required to keying object name in lists, the system will auto update following their codes;
- To yes/no fields, it is required to input 1/0 or true/false;
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
