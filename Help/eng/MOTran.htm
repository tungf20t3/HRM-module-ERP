Manufacturing Order

Manufacturing Order is used to declare order to manufacturing, used in case of manufacturing by order, cost or product calculated by MO. Therefore, cost of product calculating process will base on declared manufacturing orders.

Entry Screen



Field Descriptions

Transaction Code

Transaction code of MO. There are 2 options for selection in specific case: “Standard” (new production) or “Rework”.

Priority

There are 3 levels of priority for selection: “Urgent”, “As soon as possible”, “Normal”.

Customer

Customer ID of MO; extracted from Customer List

Plan from/to; Execution from/to

interval time of MO in plan and in reality. This information typed by hand or automatically inherited when get data from production planning.

MO Number/ MO Date

Number and date of manufacturing order; used in cost aggregation

Status

There are 05 statuses of MO: “No action”, “Plan”, “Open”, “Complete”, and “Close”.

Item Code

Code of items required for manufacturing is directly entered in “Item Master” or extracted from “Sale Order”.

UOM

Unit of measure of item. If there are many units of measure in product, they can be extracted from Unit of Measure List.

Site

Code of site receiving products is directly entered in “Site List” or extracted from “Sale Order”. There is no interaction among sites in MO and real sites receiving products.

Location/Lot

Code of location/lot receiving products in site is directly entered in “Location/Lot List”. It is only used in case products are controlled by location/lot.

Quantity

Quantity of products can be directly entered in entry screen or in “Quantity” field in Sale Order.
