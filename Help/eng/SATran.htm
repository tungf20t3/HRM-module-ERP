Supplier Allocation

This is the next step after PR Approval. Data from Supplier Allocation will be used to create PO exactly by function “Order to allocated supplier” in creating PO screen.
This step will also not be used if your business has Purchasing Management from Purchase step without creating PR.
Before supplier allocation step, user must enter price of items in “Supplier Price List”.
Supplier Allocation is only implemented to approve PR.
It can be implemented in 2 cases:

    - Allocate supplier to PR having no supplier
    - Correct result of PR having allocated supplier

Supplier Allocation process

    Previous step: PR Approval, Supplier Price List
    Next step: Creating Purchase Order
    Created Dept.: Supplier Allocation Department

Allocation screen

Some operations in supplier allocation

PR needing allocation

Click " - Extract Data from Requisition” on "Supplier Allocation";
Choose Requisition date;
Type Requisition Number, program will list the approved requisitions;
Type or lookup item on Requisition;
Click "OK", program will list supplier price meets the date. Click the mark on "Select" column, program will assign the Order q'ty by Approved Q'ty by default. User may be edit Order q'ty in case of there are many prices in corresponding options.

The program will valid Order q'ty by the rule:

    - Order q'ty between "Quantity from" and "Quantity to"
    - Order q'ty between "Min Quantity" and "Max Quantity"

Note:
The program will not check the allocation if inserts other detail. In case of editing, you should delete detail before allocate again.
