Report Form Definition – General Profit & Loss Statement

To each report form, the articles can be defined by the “Create report form” function of that report.

“Filter Condition” screen



Field descriptions

Report Form

Enable to select the report form for definition, lookup from report form list

Type

There are 3 types: Type 1 - Create, Type 2 - Edit, Type 3 - Delete

    Type 1 - Create: Enable to create new report form to satisfy business management requirement
    Type 2 - Edit: Enable to edit the selected report form: add, edit, delete articles in report
    Type 3 - Delete: Enable to delete unnecessary report form, check the “Confirm delete” option to delete

Report Name

The name of the report form

Other Name

The name of the report form in English

Information

Reference information of report such as form number, date and place issued

Other Information

Reference information of report in English

Confirm delete

Check-box option to confirm to delete the report form

Article definition screen



Field descriptions

Number

Ordinal number of the article in definition

Code

Code of the article (supporting in calculation of the article in formula)

Article

Name of the article shown in report

Other Article

Name of the article in English

Interpretation:

Shows the position where this article is described in the interpretation of financial statements.

Print

    - If 1- Print, this article will be shown in report
    - If 0- Not Print, this article will be hidden

Font Bond

    - If 1- Bold, this article name will be in bold font
    - If 0- Regular, this article name will be in regular font

Debit/ Credit Accounts

When we select the calculating way 1- Base on account arising and declare both debit and credit accounts, the system will get arising of debit account corresponding to credit one to report. If only debit or credit accounts are declared, the system will get debit or credit arising of that account to report.

For example:
Case 1: To “Current income tax expenses” article, debit and credit accounts are declared as 911 and 8211, the system will get arising in debit of account 911 and in credit of account 8211 to this article in report.
Case 2: To “Administration expenses” article, debit account is declared as 642, all debit arising of this account will be get to this article in report.

List of Units

List of units of which data are processed. If this field is blank, it means that all units are in process.

List of Customers

List of customers (customers, suppliers) of which data are processed. This information is value only when “Account” is AR-AP account. If this field is blank, it means that all customers are in process.

Deduction Type

If deduction type is 1 - Include deduction, debit or credit arising of the declared accounts will be subtracted to the corresponding ones of other accounts except account 911.

For example: To “Gross revenue from sale of goods and supply of services” article, credit account is 511, deduction type is 1 – Include, credit arising of account 511 will be subtracted to its debit, the corresponding account is not account 911.

Formula

The formula field is activated when the calculating way is 0 – Base on formula. The defined codes can be added or subtracted and must be put in square brackets [].

Note:
In case of division, the denominator is required to be different 0. Therefore, we have to write the condition expression for the division.


For example:
01. [131]+[132]+[133]+[134]+[135]+[139]
02. CASE WHEN {A310} > 0 THEN {A100}/{A310} ELSE 0 END
