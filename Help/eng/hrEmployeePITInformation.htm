Employee PIT Information

“General Information” tab

PIT Object

1 – Resident, 2 – Non-resident
Having effect on “Income/Deduction by the Progress Tax Tariff” entry - Details/ Employee ID: only employees being “Resident” and having “Labor contract over 3 months” appear on look up screen.

Tax Assessment

0 – No, 1 – Yes. Usually “1 – Yes”. Only in cases the taxpayers have committed that their incomes are under a regulation threshold (500,000 dong) and they themselves declared personal income tax, this option will be “0 – No”.

Taxpayer Deduction

If this option is 1 – Yes, the system will generate the deductions for taxpayer’s circumstances, dependant, SI, HI, UI premium, professional insurance and other compulsory insurances during personal income tax calculation. It’s required to check on corresponding “Pay for SI/ HI/ UI” options.

Authorization

Authorization for tax finalization; effective only if the employee is “Resident” and has “Labor contract over 3 months” If this option is 1 – Yes, in case of “Income/Deduction by the Progress Tax Tariff” entry with “Voucher Type”=2 – Adjustment in tax finalization, only employees with “Authorization” appear on Employee ID look up screen.

Labor Contract

Having effect on “Income/Deduction by the Progress Tax Tariff” entry - Details/ Employee ID: only employees being “Resident” and having “Labor contract over 3 months” appear on look up screen.

Pay for SI/ HI/ UI

If there is check:

    - The system will generate the corresponding insurance deductions during entry in “Deduction Information” tab;
    - The system will generate the corresponding taxpayer deductions during personal income tax calculation.

“Dependant Information” tab

Effective Date from

Required field

Effective Date to

Optional field. If input, it’s required the date with month the same or after month in “Effective Date from”. In case of inputting the 2nd row for the same dependant, month in “Effective Date from” must be after one in “Effective Date to” of the row before.

Common tasks

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

Download template file

Click icon to download template file. Some notes of PIT information input:

    - Employee ID: only employees in the assigned units and departments;
    - Tax Code, Dependant ID, Dependant Name, ID Card No., Tax Code, Relationship, Birth Certificate, Book No., Nation, Province/City, District, Ward/Commune, Enclosed Documents, Description: input in form of string, the number of characters maximum in turn (32,16, 128, 50, 50, 16, 128, 16, 16, 64, 64, 64, 64, 64 and 128);
    - Kind: accepted with 1 in 2 options (1 - Resident and 2 - Non-resident);
    - Tax Assessment, Deduction, Labor Contract, Authorization: accepted with 1 in 2 options (1 and 0);
    - Nationality: must be one existing in Nationality List;
    - Adding Time: number format, less than 1000;
    - Effective Date from/to: date format.

Upload data from file

Click icon to upload "Employee PIT Information" to the system;
Click "Select..." to select the required file;
Select copy type 1 - Overwrite if you want to replace the available data; if you select copy type "0 - No", the system will insert the new data only.

Note:
- If the effective date in "Dependant Information" tab is different from the available one, the system will add a new row with related information: Dependant Code, Dependant Name, Date of Birth, ID Card No., Tax Code;
- In case of many effective periods or an employee has many dependants, it is possible to input many rows for that employee;
- If the system gives an "indefinite error" message, you should check number fields or date fields;
- If the system gives a specific error message, you can check list column and accepted value fields.
