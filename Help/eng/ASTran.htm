Payment of Advanced Amount

Field descriptions - format view

Customer

Customer ID and name, extracted from Customer/Supplier List, is usually used in debt control in case of Issue related to debt transactions as lending items from other units. In case transactions which are not related to debt control, this information is used to control internal objects as production line, shop floor…

Delivered by

Person delivers goods, extracted from Supplier List (“Partner” field), corresponding with supplier ID default or entered by user. This field is declared to use or not in “Voucher Entry Declaration”.

Memo

Advanced payment reason. This is an optional field.

Credit Account

Credit account used to control advanced amount, extracted from “chart of accounts”.

Voucher Type

Type of payment voucher. There are 2 types: Common Payment and AR-AP Clearing Payment.

Voucher Number

Payment of Advanced Amount number;

Voucher Date

The date the voucher is created. It is reference information only. Program will use GL date to calculate.

Posting Date

The date the voucher is posted; used in calculating and is default as created date. This field can only be updated when user click mouse. GL date must be greater than closing date.

Ex. rate

Exchange rate is used for invoice in foreign currency. Select FC code and enter exchange rate. Exchange rate is implicitly extracted from each “Exchange Rate List”, in the latest date before invoice date.

Status

There are 3 statuses:

    - No Action: the voucher has not been posted to any book;
    - Release Later: If there is VAT data, the invoice will be shown in VAT reports. If this is AR/AP invoice, it will be shown in AR/AP by invoice reports;
    - Release: the voucher has been posted to all related books.

Field descriptions - “Detail” tab

Debit Account

Debit account, extracted from “chart of accounts”.

Description

Advanced payment reasons/ memo.

FC Amount

Amount in transaction currency. If there is any change of this amount, the amount will be recalculated, equal to FC amount multiplied by exchange rate. If transaction currency is base currency, this field will not be displayed.

Amount

Amount in base currency

Job

Job code, extracted from Job List. This is an optional field.

“Tax” Information

Report Form

Report forms are used as filter condition of reports in Tax Reports module;
Consist of:

    - 3 – VAT Invoice (default);
    - 4 – Goods, Services Purchased without Invoice;
    - 5 – Sales invoice.

Tax Type

VAT types of purchased goods, services; used to classify taxes when reporting “Input VAT Invoice Listing” (Form 01-2/VAT)”.
Consist of:

    - 1 - Goods, services used for VAT taxable business activities with revenue;
    - 2 - Goods, Services used for VAT non-taxable business activities;
    - 3 - Goods, Services used for both VAT taxable and non-taxable business activities;
    - 4 - Goods, services used for investment projects which meet tax deduction condition;
    - 5 - Goods, services not be declared in Form No. 01/GTGT.

Invoice Form

This "Invoice Form" is default as one in optional parameter but editable.
The "Invoice Form" will be used in tax reports by the following priority:

    - The "Invoice Form" input in this voucher is the first priority;
    - If this field is blank, the system will get the "Default Input Invoice Form" in optional parameter;
    - If the "Default Input Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Invoice Number / Serial Number / Invoice Date

Original invoice data; used for advanced payment as well as to enter in input tax invoice list.

Tax Supplier ID

Tax ID of goods & services suppliers, default as Supplier ID in Payment of Advanced Amount but editable and extracted from Customer/Supplier List. After entering ID, program will update Name, Address, and Tax ID of suppliers. ID can be blank if you want to enter these fields in case of retail suppliers.

Supplier Tax Name/ Tax ID

Tax Name and Code of goods and services suppliers

Goods - Services Name

Name (group) of purchased goods, services; free entered

Tax Code / Tax Rate / Tax Account

Code of tax rate type, extracted from VAT List
Tax rate and tax account will be displayed after tax code is entered but editable.

FC Amount

Amount in transaction currency. If there is any change of this amount, the amount will be recalculated, equal to FC amount multiplied by exchange rate. If transaction currency is base currency, this field will not be displayed.

FC Tax

Tax amount in transaction currency

Amount

Amount in base currency

Tax

Tax amount in base currency. Tax=amount * tax rate correlative with defined tax code.

Tax Authority

Tax Authority code, extracted from Customer/ Supplier List, used in case there are many tax authorities, which usually occurs to construction companies having works wide spread in many towns.

Note

Notes of created invoice.
