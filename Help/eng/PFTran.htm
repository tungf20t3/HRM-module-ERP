Purchase Expense Transaction

Field descriptions - format view

Supplier

Extracted from “Supplier List”. This is a required field.

Delivered by

Person delivers goods; this is an optional field.

Memo

More description of purchase expense; this is an optional field.

Credit Account

Credit account corresponding with purchase account, this is an optional field. Purchase expense transaction is one of vouchers used to write-off duplicate entries, so accounts in cash, cash in bank must be entered in Cash Management.

Invoice Number/ Serial Number/ Invoice Date

Invoice information created by supplier; used in input VAT reporting or invoices listing for payment detailed by invoice.

Voucher Number

Purchase Receipt Transaction number can increase as declaration in “Voucher List” or be entered following voucher number system of each company.

Voucher Date

The date the voucher is created and the date the voucher is posted.

Ex. Rate

Exchange rate is used for vouchers in foreign currency. Select FC code, the exchange rate is extracted from “Exchange Rate List”, in the latest date before voucher date.

Status

There are 3 statuses:

    - No Action: the voucher has not been posted to any book;
    - Release Later: If there is VAT data, the invoice will be shown in VAT reports. If this is AR/AP invoice, it will be shown in AR/AP by invoice reports;
    - Release: the voucher has been posted to all related books.

Field descriptions - "Detail” Tab

Item code

Item code in Receipt or Invoice.

UOM

Default unit of measurement in “Item Master”. It can also be changed by conversion of units of measurement in some special cases.

Site / Location / Lot / Quantity

These data are extracted from invoice and un-editable.

Job

Purchase job… related to received items.

Expense

Expense allocated in details to items in invoice.

Debit Account

Debit account is extracted from item account in Purchase Receipt entry and un-editable.

Invoice No. / Line

Invoice number and line number in invoice. This field has value only when data in receipt extracted from invoice.

Field descriptions - “Charges” Tab

Charges tab in invoice is declared as information for reference, used to declare charges arising in purchasing.

Charge code

Enclosed in invoice, extracted from “Charge List”

Charges

Amount of charges relates to purchasing. Amount of every charge code will be allocated to every item in invoice by specific formula. Allocation formula is defined in “Charge List”.

Allocate Charge

Charge can be allocated to one or many invoices.

    - Allocate to 1 invoice:
    In detail of Purchase Expense Transaction, click “ - “Select Receipt Voucher and Clear Contents”. Enter the filter condition and click to filter the Receipt required to allocate charge. This function is allowed to select 1 invoice, when select another invoice, the data of previous invoice will be cleared.
    - Allocate to many invoice:
    In detail of Purchase Expense Transaction, click on the icon “ - “Select Receipt Voucher”. Enter the filter condition and click to filter the Receipt required to allocate charge. Enter the detail allocation to selected invoice and click "Ok". Then, select another invoice to allocate charge by the same tasks.
    - Manual Allocate:
    In case user does not accept the automatic allocated result by computer, click on the icon “, then enter detail charge allocated.

- If user create new voucher (Purchase Expense Transaction) then save, the program will automatically allocate the charges without any task;
- If user edit a voucher automatically allocated (before), then add more charge lines and then save it, the program will automatically allocate the charges without any task;
- Once user click on the icon and edit automatic result , the program will not automatically allocate the charges and always check the charge amount between detail tab and charges tab.

Field descriptions - “Tax” Information

Data of VAT are extracted from format view but you can edit in some special cases.

Report Form

Report forms are used as filter condition of reports in Tax Reports module;
Consist of:

    - 3 – VAT Invoice (default);
    - 4 – Goods, services purchased without invoice;
    - 5 – Common sales invoice.

Tax Type

VAT types of purchased goods, services; used to classify taxes when reporting “Input VAT Invoice Listing” (Form 01-2/GTGT)”.
Consist of:

    - 1 - Goods, services used for VAT taxable business activities with revenue;
    - 2 - Goods, Services used for VAT non-taxable business activities;
    - 3 - Goods, Services used for both VAT taxable and non-taxable business activities;
    - 4 - Goods, services used for investment projects which meet tax deduction condition;
    - 5 - Goods, services not be declared in Form No. 01/GTGT.

Invoice Form

This "Invoice Form" is default as one in optional parameter but editable.
The "Invoice Form" will be used in tax reports by the following priority:

    - The "Invoice Form" input in this voucher is the first priority;
    - If this field is blank, the system will get the "Default Input Invoice Form" in optional parameter;
    - If the "Default Input Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Invoice No. / Serial No./ Invoice Date

Data in these fields are extracted from format view but editable.

Supplier ID/ Supplier Name/ Supplier Tax Code/ Goods – services name

Information of supplier and goods in invoice

Tax Code / Tax Account

Data are extracted from VAT List.

Tax Authority

State Tax authority, where company registers tax declaration
