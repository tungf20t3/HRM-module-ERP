Create lot during voucher entry

If the item is checked in “Lot Control” (in Item List), you can define lot information in this “Lot Maintenance”. Besides, lot can be created during data entries of following vouchers:

    - Domestic Purchase Invoice;
    - Import Purchase Invoice;
    - Purchase Receipt – Direct Issue Voucher;
    - Receipt

If the item is checked in “Lot Control” and “Allow creating lot when receiving” options (in Item List), the system will auto-generate lot code in the format defined in "Automatic Lot Code" parameter to put in the voucher.

In voucher entry screen (grid view), click the icon - “Lot maintenance” to open “Lot maintenance” screen. This screen will only show the detail lines of items with "Auto creating lot when receiving" option and the current code of lot is incorrect. If the “Lot” typed in detail does not exist, the system will auto generate lot code by the rule defined in system parameters.
Lots are generated by the format defined the “Automatic Lot Code” parameter with 04 special characters “d,m,y,#”:

    - "d" stands for day;
    - "m" stands for month;
    - "y" stands for year;
    - "#" stands for ordinal number (auto increase in case of multi-lot receiving in a day).

The character “dmy” must be lower case, if they are in upper case, the system will understand as a part of lot code.

Example:
We configured the parameter “Automatic Lot Code” as “LO.ddmmyyyy.##” In the day Oct 15th 2012 we have 2 receipts, the first one will have lot ID as “LO.15102012.01”, and the second one will have lot ID as “LO.15102012.02”.

Other related information:
- Receipt Date: being default as voucher date in the first data entry, but editable;
- Lot Description: system-generated in the form of Lot + lot ID + [Receipt Date] + voucher date, but editable;
- Manufacturing Date: being default as Receipt Date, but editable;
- Expiry Date: being equal to "Receipt Date" plus "Shelf Life Days" (if the "Assignment" is "1 - When Received");
- Warranty Date: being equal to "Receipt Date" plus "Warranty Days", but editable.

Note:
After you press “OK” in “Lot maintenance” form, a new lot has been created in the list although the voucher has not been saved yet. Therefore, when editing an unfinished voucher, you should manually input the created code of lot to avoid creating another code in the system.

Import data from excel spreadsheet file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

    - Enter the browser of a voucher. If exist the icon and in the tools bar, the system allow mass importing vouchers from excel spreadsheet such as sales invoices, bill of materials...
    - Or during creating new voucher, if exist the icon and in the tools bar of grid. It's meaning that you can only import the detail of each voucher such as Cash Receipt, Cash Disbursement (type 3)... This function is used in case of frequently inputting the similar periodic transactions.
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will import data from this file to browse screen or to grid view of the voucher;
    - Select copy type 1 - Overwrite if you want to replace the available data. If "No", the system will insert the new data only.

Please read the help of specific voucher to get the rule of keying for each column.

The system will give error message in some following cases:
- Data in column containing objects do not exist in the corresponding lists (Customer, Account, Job...). In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field;
- etc.

Other notes:
- The current user must have the "New" right to add data to grid;
- The current user must have the "Edit" right to edit data in grid;
- It is not required to key in object name in lists, the system will auto update following their codes;
- etc.

Other Notes

Right in voucher operation

The operation in vouchers will also be affected by the authorization by function and unit.

    - If you are authorized by function with full right in operation (new, edit, delete, view other users' vouchers, print, ...) and authorized by a unit, you have full right to operate all vouchers in this unit.
    - If you are authorized by function with full right in operations (new, edit, delete) but not authorized to view the vouchers created by other users, you can only work in the vouchers created by yourself.
    - Warning "No right to perform this function or access denied. Contact your system administrator if you can get access to this function" will appear when you do not have the corresponding right.

System-generated voucher number

Number of new voucher will be generated to the next of current number in Voucher Book List used in the current voucher where the cursor is placed. In case of not using Voucher Book List, it will be system-generated as declaration in the voucher information.

Lookup

To lookup an item from any list, key in three characters into textbox, the list will appear to select.

First, the system will search in code list; for example, if you key in “KH0”, the following codes (suppose that they have existed) KH0A01, AB KH0A04, XX KH0,
After that, if the required item cannot be found by code, the system will search by item name with the key words “KH0”.
In case the required item is not found, click the "" icon to open lookup screen.

    - If you remember some characters of item name, key in the “Quick Search” to search by name;

    - If not, use the advance search function (Search Query - see “Filter” function in "Hints and Tips in List Setting")

- If you cannot remember both code and name, use quick search. Click the "" icon and then key the required characters in “Quick Search” box.
- If you remember code but name, you should search by field filter. Click "", icon and next click "" icon in code field, and key in the required characters.

“Check existence” function: if the keyed code is underlined by the system, it means the code does not exist in the list. You should check all characters keyed in.

Note:
01. Enable to add new item in lookup screen (see "Add new code in lookup screens" function in " Hints and Tips in List Setting" for more details)
02. Enable to re-filter the filtered result: for example, vouchers filtered by item code can also be filtered by site code, position code…until you find the desired result.
03. It is required to disable “Check spelling” function of the internet browser for this function to process correctly.

The functions of Checking exist

if you see the border line at the foot of code or ID after finished typing, meaning is not on the list. That case, should check all characters entered.

Common tasks in date fields

To input the date in data entry, we can directly key in or select from the system by clicking on the icon at the right of date fields.

In the calendar screen , we can select the day only. To choose the month, use the left and right arrows. In case of selecting in many months and years, click on “month, year” area (at the middle of 2 arrows), we can select the year. After that, we can select the month. Thus, at the date field, we can select by following order:

02. Month, year
03. Year - 10 years

Calculate in number field

The program allows simple calculating between the numbers. Put cursor in number field, press math “+ - x /” to activate calculation function. When has final result, press enter.

Resize object

The program allows scaling screen or fields in the grid to make things look better.

Expand and Restore Voucher Screen

You can simply expand voucher screen to entire screen and restore it by double-clicking on voucher title bar.
If the screen is maximized, double-clicking the title bar restores it. If it is not maximized, double-clicking the title bar maximizes it.

Moving between Vouchers

You can simply moving between voucher in view screen by click on the icon or in voucher title bar to move to the next or previous voucher.

Keyboard Shortcuts

The program also support alternate keyboard shortcuts. The buttons can be accessed by keyboard shortcuts by pressing and holding the Alt key + the corresponding underlined letter.

    1. Keyboard Shortcuts in Vietnamese interface: Mới [M], Sửa [S], Lưu [L], Nhận [N], Hủy [H], Đóng [g];
    2. Keyboard Shortcuts in English interface: New [N], Edit [E], Save [S], Ok [O], Cancel [C], Close [l];
    3. When working in the screen or form with multi tabs, the Tabs can be accessed by keyboard shortcuts by pressing and holding the Alt key + 1,2,3 (tab 1, tab 2, tab 3);
    4. Use Keyboard Shortcuts to move the focus to the fields under detail grid (voucher): The text box can be accessed by keyboard shortcuts by pressing and holding the Alt key + the corresponding underlined letter.

Guide to customize keyboard shortcuts for Firefox browser: use the [ALT] instead of [Alt] [Shift]
1. Open Firefox browser, type "about:config" in address bar;
2. Search "ui.key.generalAccessKey";
3. Change the value from -1 to 18;
4. Close and open Firefox again.
