Domestic purchase invoice

Field descriptions – format view

Supplier

Code to identify supplier; extracted from “Supplier List”.

Delivered by

If “contact person of trans” field is declared that be used in “voucher information”, deliverer field will be extracted from supplier list, but editable.

Memo

Purchase reasons/ memo, free entered

Credit Account

Credit account in invoice posting (Payable or cash, cash in bank accounts).
Cr. Account is extracted from “default account” field in supplier list by default but editable.

Invoice Number / Serial Number/ Invoice Date

Invoice information created by supplier; used in input VAT reporting or invoices listing for payment detailed by invoice.

Voucher Number

Accounting voucher number posted to Ledger.

Voucher Date/ Posting Date

The date the invoice is created and the date the entries in invoice are posted to GL.

Ex. Rate

Exchange rate is used for invoices in foreign currency. Select FC code and enter exchange rate. Exchange rate is extracted from “Exchange Rate List”, in the latest date before invoice date.

Status

There are 3 statuses:

    - No Action: the voucher has not been posted to any book;
    - Release Later: If there is VAT data, the invoice will be shown in VAT reports. If this is AR/AP invoice, it will be shown in AR/AP by invoice reports;
    - Release: the voucher has been posted to all related books.

Debit Term

Extracted from Payment Term List

Field descriptions - “Detail” Tab

Item Code

Item code in invoice, extracted from “Item Master”

UOM

UOM of item, extracted from item master by default but editable in condition that the item configured as “controls multi UOM” and have more than one UOM in conversion List.

Site

Sites issuing goods; extracted from “Site List”.

Location

Location of goods in site, extracted from “Location List”, used in case you wish to control items by location.
If item declare site ID, location ID will be defaulted in item list. When enter item code, location ID will be defaulted, but editable.

Lot

Lot/ Serial number, required for items with lot control. Lot control defined in Item Master. Lot declaration has been done in Inventory Management.

Job

Job code used to control a purchase process

Quantity

Quantity purchased, directly entered.

Price

Unit price of item in GL currency.

Debit Account

Dr. Account in receipt entry, usually taken default value declared in “Item Master’ but editable.

Field descriptions - “Charges” tab

Charges tab in invoice is declared to input expenses relating to purchasing. In case arising charges relate to another supplier, you have to input in “Purchase Expense Transaction” function instead of this tab.

Charge code

Enclosed in invoice, extracted from “Charge List”

Amount

Amount of charges relates to purchasing. Amount of every charge code will be allocated to every item in invoice by specific formula. Allocation formula is defined in ”Charge list”.

Field descriptions - “Tax” Information

Data of VAT are extracted from format view and can be edited in some special cases.

Report Form

Report forms are used as filter condition of reports in Tax Reports module;
Consist of:

    - 3 – VAT Invoice (default);
    - 4 – Goods, services purchased without invoice;
    - 5 – Common sales invoice.

Tax Type

VAT types of purchased goods, services; used to classify taxes when reporting “Input VAT Invoice Listing” (Form 01-2/GTGT)”.
Consist of:

    - 1 - Goods, services used for VAT taxable business activities with revenue;
    - 2 - Goods, Services used for VAT non-taxable business activities;
    - 3 - Goods, Services used for both VAT taxable and non-taxable business activities;
    - 4 - Goods, services used for investment projects which meet tax deduction condition;
    - 5 - Goods, services not be declared in Form No. 01/GTGT.

Default tax type is defined in optional parameters.

Invoice Form

This "Invoice Form" is default as one in optional parameter but editable.
The "Invoice Form" will be used in tax reports by the following priority:

    - The "Invoice Form" input in this voucher is the first priority;
    - If this field is blank, the system will get the "Default Input Invoice Form" in optional parameter;
    - If the "Default Input Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Invoice Number / Serial Number/ Invoice Date

Data in these fields are extracted from format view but editable.

Supplier ID/ Supplier Name/ Supplier Tax Code/ Goods – services name

Information of supplier and goods in invoice

Tax Code/ Tax Account

Data are extracted from VAT List.

Tax Authority

State Tax authority, where company registers tax declaration

Some tasks in voucher entry

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

The data importing process:

    - Enter Domestic Purchase Invoice;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to grid view of the invoice.

Notes of inputting data in template file:
- Item, UOM, Site, Reason Code, Debit/Credit Account, Job, Department, MO, Product, Contract, Expense, Loan Contract, Location, Lot: must be one existing in the corresponding list with the same length in add (edit) screen in list;
- Site: input sites belonging to the assigned units;
- Location: required when site having location track;
- Lot: required when item with lot control;
- Quantity: input in number format, 9 digits maximum and 3 decimal digits (standard format). To item with FIFO costing method, quantity must not be 0;
- Price: input in number format, 12 digits maximum and 4 decimal digits (standard format);
- Amount: input in number format, 14 digits maximum and 2 decimal digits (standard format).

The system will give error message in some following cases:
- Data in column containing customer, supplier, account do not exist in the corresponding lists. Account is general account. In this case, the system will also get data to grid but will give error message when saving;
- String data is longer than the available length of field.

Other notes:
- The function is only supported to get data for each voucher, used in case of frequently inputting the similar periodic transactions.
- You must have the "New" right to import new data to voucher detail grid;
- You must have the "Edit" right to edit data in voucher detail grid;
- It is not required to keying object name in lists, the system will auto update following their codes.
