Misc. Payable Transaction, AP-AR Interface

Misc. Payable, AP-AR Interface between 2 suppliers is used to update other payable amounts which do not belong to purchase invoice.

For example: FC Exchange Rate Transaction, small debit balance adjustment, AP-AR Interface between 2 suppliers or between 1 supplier and 1 customer.

Field descriptions - format view

Voucher Number

It is automatically increased 1 when a new voucher entered but editable.

Voucher Date

The date the voucher is created; reference information only. Program will use GL Date in calculation.

Posting Date

The date the voucher is posted; default as created date in calculation and greater than closing date.

Ex. Rate

If currency is GL currency, exchange rate will be 1 and this field will not be updated. If exchange rate is changed, program will recalculate amount related to foreign currency and GL currency.

Field descriptions - “Details’ Tab

Account

Extracted from Chart of Accounts

Customer

Customer ID, extracted from Customer/Supplier List, is usually used in debt control in case of Issue related to debt transactions as lending items from other units.

Debit / Credit

Debit/ Credit amount in GL currency

Description

Transaction reasons/ memo.

Contract

Information of contracts related to voucher.

Loan Contract

Information of loan contracts related to voucher.

Job

Code of job related to service purchase, extracted from Job List.

Payment Term /Invoice Number/Invoice Date

These fields only display when the parameter “Advanced Payment Control” is "Yes".
If this parameter is "Yes", corresponding to following cases:
- Account input in column “Account” (Detail grid) is “AR Account” in list of parameter “Accounts Receivable” and “Debit” not equal 0 , AR Invoice will be posted to AR report;
- Account input in column “Account” (Detail grid) is “AP Account” in list of parameter “Accounts Payable” and “Credit” not equal 0 , AP Invoice will be posted to AP report.

Group

Used in case enter many debit and credit in one same voucher. At that time, you must divide into groups: 1 debit/ many credits or many debits/ 1 credit. This grouping aims at helping create reference account of an account report. You must number groups to distinguish group of account. The simple way is gradually increase 1, 2, 3…

Field descriptions - “Input VAT” Tab

Tax information is extracted from format view but edited in some special cases.

Report Form

Report forms are used as filter condition of reports in Tax Reports module;
Consist of:

    - 3 – VAT Invoice (default);
    - 4 – Goods, services purchased without invoice;
    - 5 – Common sales invoice.

Tax Type

VAT properties of purchased goods, services; used to classify taxes when reporting “Input VAT Invoice Listing” (Form 01-2/VAT)”.
Consist of:

    - 1 - Goods, Services used for taxable business activities with revenue;
    - 2 - Goods, Services which do not meet tax deduction condition;
    - 3 - Goods, Services used for both taxable and non-taxable business activities;
    - 4 - Goods, services used for investment projects which meet tax deduction condition;
    - 5 - Goods, services not be declared in Form No. 01/GTGT

Invoice Form

This "Invoice Form" is default as one in optional parameter but editable.
The "Invoice Form" will be used in tax reports by the following priority:

    - The "Invoice Form" input in this voucher is the first priority;
    - If this field is blank, the system will get the "Default Input Invoice Form" in optional parameter;
    - If the "Default Input Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Inv. No. / Serial No./ Inv. Date

Invoice number, serial number and invoice date relating to arising transaction (if any)

Supplier ID/ Supplier Name/ Supplier Tax Code/ Goods – services name

Information of supplier and goods in invoice

Tax Code / Tax Acct./ Ref. Acct.

Tax code, tax account are extracted from VAT List. Reference account is used to enter the account corresponding to tax account.

Tax Authority

State Tax authority, where company registers tax declaration

Some common operations

Create different groups of account in voucher

Input debit and credit accounts in one same field, amount field shows that account is debit or credit. Accounts in groups have only 1-1 or 1 - many correspondence, total debit and credit amount in group must be equal.

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

For example: There are only account code, customer ID... in excel file. After importing data from the excel file, the voucher will be auto-updated with account name, customer name...

The data importing process:

    - Enter Voucher;
    - Click the icon , the system allows user to download template file to put data in;
    - After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to grid view of the voucher.

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists (Customer, Account, Job...). In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field.

Other notes:
- The function is only supported to get data for each voucher, used in case of frequently inputting the similar periodic transactions.
- If you do not have the "New" right, you cannot import new data to voucher detail grid;
- If you do not have the "Edit" right, you cannot edit data in voucher detail grid;
- It is not required to keying object name in lists, the system will auto update following their codes.

Import data from excel file (mass importing)

The same as importing data for each voucher with some following notes:

Notes of inputting data in template file:
- Unit: must be one of the assigned units with unit locking date before invoice date;
- Account: must be detailed account;
- Customer: required field if the account is foreign currency account;
- Currency: if voucher consists of many rows, the currency of the first row will be considered as currency of the voucher;
- Exchange Rate: to foreign currency vouchers, it is required to define exchange rate in detailed rows, the system will not generate exchange rate for details;
- Voucher Date: input in date format. Voucher date must be after data closing date and voucher locking date;
- Debit/Credit Amount: input in number format, 14 digits maximum and 2 decimal digits (standard format). The system will import the right amount input in the template file, not recalculate;
- Group (of double entries): there can be 1-1 or 1-multiple entries only, total debit and credit amount in a group must be balanced.

The system will give error message in some following cases:
- Data in column containing customer, account do not exist in the corresponding lists. Account is general account. In this case, the system will indicate the specific invalid cell;
- The required fields (Voucher Number, Unit...) are blank;
- String data is longer than the available length of field; the field length is the same as one manually input.

Other notes:
- Cannot import data after system data closing, unit or voucher locking date (the data closing date after the beginning input date);
- The current user must have the "New" right in "General Voucher" menu;
- The current user must have the "New" right in "Access Right by Unit" menu;
- Enable to import data for many units. If the data of the current unit has been closed, the system will give warning message when you click the icon; if data of other units has been closed, the system will also give warning message during importing process and indicate the specific invalid cell;
- Check the duplicate vouchers by number only (if available);
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
