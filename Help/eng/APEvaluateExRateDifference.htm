Invoice Exchange Rate Difference Revaluation

At period end, when you run “Evaluate Exchange Rate Difference” function, only differences in FC accounts are evaluated.. If you give payment (disbursement) by invoice next period, exchange rate will be the evaluated value, which is not correct. In this case, you should use “ Evaluate Exchange Rate Difference in Invoice” function to correct the data.

Note: it is unnecessary to run this function if you do not control payables and disbursement by invoice.

“Invoice Exchange Rate Difference Revaluation” Screen

Field Descriptions

Period/Year

Period and year of invoices have been evaluated the difference of exchange rate

Currency

Currency of invoices have been evaluated the difference of exchange rate

Ex. Rate

Exchange rate of foreign currency at closing time, entered by user, used to apprise difference in exchange rate of invoices.

Type

There are 2 types:

    - Evaluate;
    - Delete Evaluation.

Unit

Unit in which the invoices are evaluated the exchange rate difference

Operations

Select period

Select month in fiscal year in which exchange rate transaction of invoices are processed. Before you see “Exchange Rate Difference Transaction of Invoice” screen, select period in “Select Period” screen.

Select kind of foreign currency

Before processing “Exchange Rate Difference Transaction”, select foreign currency type which is evaluated in period and enter closing exchange rate.

Create Exchange Rate Difference evaluation/ Delete Exchange Rate Difference Transaction

Select type 1 - Evaluate to create exchange rate difference evaluation or type 0 - Delete Evaluation to delete exchange rate difference transaction of invoices;
Then, click "Ok" or "Cancel" to continue or cancel the evaluation.
