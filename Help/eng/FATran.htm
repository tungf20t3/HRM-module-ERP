Fixed Asset Information

Entry screen

Field Descriptions

FA Code

Fixed asset code is used like fixed asset card number. Utility: in case the code is defined as a character string in which last characters are numbers in sequence, when you add new asset and key in some first characters of any code, the system will auto search the greatest number of the entered string and auto increase with 1.

Description

Description name of asset

FA Type

Type of assets; extracted from FA type list

Acquisition Reason

Increase reason is used to define increase/decrease reason, extracted from increase/decrease reason list.

Acquisition Date / Depn Date

Increase date and depreciation date of asset. Normally, depreciation date is a month later than increase date. The system calculates depreciation of assets which have valid increase date and depreciation date.
If you input a depreciating asset, increase date is date on the card and depreciation date is date of inputting.

Depn Periods

Depreciation month. If depreciation calculation method is 1- “By month”, monthly depreciation will be equal to original cost divided by depreciated months.

Depn End Date

Depreciation ending date of asset. If you opt for “Depreciation the whole Remaining Amount in Period” in “System/ List Maintenance/ Optional parameters”, the program will base on it to define the depreciated ending period of asset. This is an optional field.

Diff. Amount

Different depreciation amount of the last period. If remaining amount is less than different amount, total remaining amount will be added to the last period depreciation amount; otherwise, remaining amount will be calculated to depreciation of new period.

Depn Method

There are 3 methods to calculate depreciation:

    0 – Straight-line
    1 - Quantity
    2 – Decreased Balance with Adjustment

If select 0- Straight-line, a period depreciation will be equal to original cost or remaining amount divided by periods (depending on optional parameter definition)
If select 1- Quantity, a period depreciation will be equal to original cost divided by total quantity and multiplied by quantity of depreciating period.
If select 2- Decreased Balance with Adjustment, a period depreciation will be equal to original cost multiplied by quick depreciation ration and divided by 12

Depreciated Periods at BoP

Depreciated period is used when depreciation type is 2- Decreased Balance with Adjustment.
Depreciated period is remainder of periods of depreciation (before running program) divided by 12.

Example: Fixed asset A has been depreciated in 27 periods, depreciated period will be 3.

Depn Rate

Quick depreciation rate is used when depreciation type is 2- Decreased Balance with Adjustment.
Quick depreciation ration is equal to FA depreciation type 1 multiplied by adjustment ratio.

Total Production

This field is displayed when depreciation type is 1- Quantity.
Total quantity depends on designed capacity of fixed assets.

Voucher Number / Voucher Date

Number of voucher related to assets increase

Currency Code / Ex. Rate

Code of transaction currency of current voucher; extracted from Currency list.
This rate is transaction exchange rate; extracted from “Exchange rate list” based on the latest exchange rate. If currency code is base currency, exchange rate equals 1.
The cursor is invalid at this field, you have to use mouse to select currency type.

Department

Code of department using assets is extracted from FA using department list. It shows by which department assets are used.

Asset Account

Asset account is account type “21*”.

Depreciation Account

Depreciation account is account type “214*”. When creating depreciation entry, program will create credit entry in reference to expense account.

Expense Account

Expense account is account type “6**4”. When creating depreciation entry, program will create debit entry in reference to depreciation account.

"Detail" tab

Capital

Sourced capital of assets. The code is extracted from capital list.

Original cost

Original purchasing cost of asset. Original cost is used to calculate depreciation of periods.

Accumulated Depreciation

Accumulated depreciation of asset

Remaining Value

Current amount of asset is equal to original cost deducted to depreciated amount.

Depreciation per Period

Monthly depreciation of asset is used to update details of monthly depreciation, equals original cost divided by months of depreciation and can be adjusted.
Depreciation is monthly calculated and saved in the database. In case of any changes, you need to recalculate depreciation to update related books.

Opening Depreciation

There will be “opening depreciation” column when depreciation type is 2- Decreased Balance with Adjustment.
Opening depreciation will be corresponding with periods of depreciation.

“Attached Tools/Supplies” tab


Attached information is defined as reference information.

“Further information” tab

Other Name

English name of asset

FA Number / Technical Parameters / Made in / MFG Year

More information of asset: notation, technical parameter, manufacturing country, manufacturing year...

Group 1 / 2 / 3

Asset groups are used in group reports. These codes are extracted from “Asset Group List”.
This information may not be input in this section but grouped in mass in “Group List”.

Regulation Depn Period(s)

Default to be equal to “Depn Periods” in general information, but editable

Exceeding Rate (%)

Exceeding rate=[(Regulation depreciation periods/ Depreciation periods) - 1] x 100

Permissible Exceeding Ratio

Permissible exceeding ratio; default as 2 but editable

Depriciation Periods Remain

"Remaining periods" is used when depreciation type is 2- Decreased Balance with Adjustment. "Remaining periods" is equal to depreciation periods of asset minus depreciated periods.

"Other" tab

Round Value

The round depreciation amount of the last period. If the remaining value is less than this round value, total remaining value will be added to the last period depreciation amount. On the contrary, the remaining value will be depreciated in a new period.

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

The data importing process:

- Enter "Fix Asset Information" menu;
- Click the icon to download template file to put data in;
- After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to voucher.

The system will give error message in some following cases:
- Data in column containing account code does not exist in chart of accounts, is general account or AR/AP account. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field which is the same as one manually input.

Other notes:
- The current user must have the "Edit" right in this function;
- The current user must have the "Edit" right in "Access Right by Unit";
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
