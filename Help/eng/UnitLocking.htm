Unit Locking

AI ERP supports this function for closing data by unit.

This function is useful in case database consists of data of many units but the data closing dates of units are different.
It is impossible to add, edit or delete any voucher in data of unit from the data closing date backward. If it is required to edit closed data, “open” by closing by the date of previous period or let the data closing date blank. For example: If you want to open period 05/2016, close data to date 30/04/2016 (period 04/2016).
The functions affected by data closing:

    - All vouchers manually input;
    - Processes in period (Average cost, Daily average cost, FIFO cost, Inventory cost, Write-off, Invoice exchange rate difference evaluation, Reserving of exchange rate difference revaluation);
    - Period-end processes (Transfer account balance to next year, Allocation entry, Closing entry, Exchange rate difference entry, Reversing entry of exchange rate difference entry);
    - Opening balance: Account opening balance, Customer opening balance , Invoice opening balance, Project/Job opening balance, Job norm opening balance...
    In case of balance related to many units, only balance of open unit can be edited.

Note:
- This function is not used for costing, payroll, book exchange rate calculations because they have not been processed by unit (there is no unit to select);
- In case unit field is blank or selected many units: if data of 1 unit is closed, the system will stop processing and give message (do not process for any unit);
- If you select specific unit to process: the system will check the “Unit Locking Date” condition and perform process if satisfied.
