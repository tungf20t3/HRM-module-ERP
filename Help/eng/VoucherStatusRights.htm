Limit access right to voucher statuses

Path: Modules → System → User Maintenance → Limit access right to voucher statuses
AI ERP allows limiting the right of users to edit vouchers by each voucher and its status.

For example: Staffs can be allowed to edit voucher only when its status is “No Action”. When voucher status is changed to another one, they can read only.

Definition screen

Field descriptions

User

Selected from the defined user list

Voucher

Selected from voucher list, which are limited by user

Status/ Status Name

The statuses of a voucher which are assigned to users

No edit

This check-box option restricts the user right to edit a voucher in a particular status.

For example, if check in the “Posted to GL” status, a voucher in this status cannot be edited.

No delete

This check-box option restricts the user right to delete a voucher in a particular status.
