Sales Price List

Sales price list is defined to support extracting sales price when creating quotation, sales orders or sales invoices.

Entry screen

Field descriptions

Item Code

The code of item being defined the sales price; extracted from Item List

Effective Date

The date when the sales price is effective; compared with system date to consider if the price has been effective or not.

For example:
Item HH1 has 3 sales prices (in order) 100,000 effective on 01/01/2010; 150,000 effective on 01/05/2010 and 200,000 effective on 01/09/2010. If a sales order is created on 01/06/2010, the sales price in sales order will be 150,000.

Currency

The monetary unit that the sales price follows

Sales Price

The sales price of item in the defined currency

Some tasks in list maintenance

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.
The data importing process:

    - Enter List;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to List.

Notes of inputting data in template file:
- Item and Effective Date: check the existence of item with the effective date in Sales Price List; if the input item already exist in sales Price List, the system will replace by one in the template file (if the copy type is "Overwrite"). The new items with new effective date are certainly added to List. The length of code is the same as one in add (edit) screen;
- Effective Date: input in date format;
- Item Code, Currency: must be one existing in the corresponding list with the same length in add (edit) screen in list;
- Sales Price: input in number format, 12 digits maximum with 4 decimal digits (standard format).

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field;
- Duplicate or nested codes, in this case, the system indicate the specific invalid cell.

Other notes:
- If you do not have the "New" right in "Sales Price List", you cannot import new data from spreadsheet to this list;
- If you do not have the "Edit" right in "Sales Price List", you cannot edit data in this list from spreadsheet.

See “Hints and Tips in List Setting” for details of common tasks in list maintenance
