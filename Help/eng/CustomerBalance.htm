Customer Opening Balance

This function is used to input customer opening balance (if any) before using the program. The data entry begin date is defined in “Define the Data Entry Begin Date” menu.
In case the data entry begin date is the same as the fiscal year begin date; the opening balance will be year-opening balance.
In case the data entry begin date is different from the fiscal year begin date and the “Input customer/supplier year-opening balance” optional parameter in System is activated, this function will also be used to input year-opening balance.
On the contrary, the “Input customer/supplier year-opening balance” optional parameter in System is not activated, the year-opening balance will be input in “Account opening balance” menu in General Ledger module.
Opening balance is entered once only when you start using the system. Opening balance of next periods and next year will be automatically calculated and transferred.

Entry screen

Field descriptions

Account

The AR-AP account in which customer balance is recorded; extracted from chart of accounts

Customer

The customer having opening balance; extracted from customer list

Debit/Credit Opening Balance

The opening balance of customer in debit or credit; input only in one side (debit or credit)

FC Dr./Cr. Opening Balance

The opening balance in foreign currency of customer in debit or credit; input only in one side (debit or credit)

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

The data importing process:

- Enter "Customer Opening Balance" menu;
- Click the icon to download template file to put data in;
- After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to opening balance file in the system.

Notes of inputting data in template file:
- Unit, Account, Customer (Supplier): Unit is one of the assigned units; account is detailed account and AR/AP account. The input unit, account code must be one existing in the list with the same length in add (edit) screen in list;
- Period-opening Debit Balance, Period-opening Credit Balance, Year-opening Debit Balance, Year-opening Credit Balance: number format, 14 digits maximum and 2 decimal digits (standard format);
- Period-opening FC Debit Balance, Period-opening FC Credit Balance, Year-opening FC Debit Balance, Year-opening FC Credit Balance: number format, 12 digits maximum and 2 decimal digits (standard format).

The system will give error message in some following cases:
- Data in column containing customer, supplier ID and account codes do not exist in chart of accounts; being general account or AR/AP account. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field which is the same as one manually input.

Other notes:
- Cannot import data after system or unit data closing (the data closing date after the beginning input date);
- The current user must have the "Edit" right in "Customer Opening Balance" to edit the balance rows available in data (select the copy Type "Overwrite");
- The current user must have the "New" right in "Customer Opening Balance" to add the new balance rows into data (select the copy Type "No");
- The current user must have the "Edit" right in "Access Right by Unit" to edit the balance rows available in data (select the copy Type "Overwrite");
- The current user must have the "New" right in "Access Right by Unit" to add the new balance rows into data (select the copy Type "No");
- Enable to import data for many units. If the data of the current unit has been closed, the system will give warning message when you click the icon;
- If data of other units has been closed, the system will also give warning message during importing process. In this case, the system will indicate the specific invalid cell;
- If the beginning input date is the same as the first day of fiscal year, opening balance is year-opening balance;
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
