Bill of Material

Bill of Material is used to declared material structure of a product, used as allocation formula. Basing on product structure, program will define coefficients to allocate other costs in cost of product.

Note: In this version, product structure is not declared in many levels (costing calculation follows only 1 operation). The calculations of this operation will be adjusted by specific demand.

Entry Screen

Field Descriptions

Product

Code of product which needs BOM; extracted from “Item Master”.

Department

Code of department having cost in production

Item

Code of items, materials used to form product structure

Quantity

Quantity of material in BOM

Scrap Factor

Acceptable quantity of scrap. Applying in:

    - Issuing item by MO;
    - Detail Statement Of Cost

Enter in program: enter in form of number.

Example: Scrap factor is 10%, enter “10”

Effective from/to

Effective time of BOM. There can be many BOMs at different times.

Note: “Effect Date from/To Date” menu is used in case product structure is completely changed; if there is only temporary replacement of items in equivalence, use “Material Replacement Declaration

Some tasks in voucher entry

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

The data importing process:

    - Enter "Bill of Material" menu;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to upload data to the system, click the icon , select the file, the system will get data from this file to grid view of the voucher.
    - Select copy type 1 - Overwrite if you want to replace the available data; if you select copy type "0 - No", the system will insert the new data only.

Notes of inputting data in template file:
- Product: only input item code with item type 51 - Finished Product;
- UOM: must be one existing in "Units of Measure" list; if blank, the system will get the standard UOM of the item;
- Department, Item: must be one existing in the corresponding lists with the same length in add (edit) screen in list;
- Quantity, Scrap Factor (%), Lead Time, Cost Allocation Rate (%): number format;
- Effective Date from/to: date format.

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists (Customer, Account, Job...). In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field.

Other notes:
- If you do not have the "New" right, you cannot import new data to voucher detail grid;
- If you do not have the "Edit" right, you cannot edit data in voucher detail grid;
- It is not required to key in object name in lists, the system will auto update following their codes.
