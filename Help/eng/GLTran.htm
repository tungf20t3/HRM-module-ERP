General voucher

Entry screen

Field descriptions

Format view

Voucher Number

Voucher number is system-generated to increase 1 when a new voucher is created; but it is editable.

Voucher Date

The date when the voucher is created; for reference only

Posting Date

The date when the voucher is posted to GL; be default as voucher date but used in calculation. This field is active only when you click on. Posting date must be after closing date.

Currency

Code of the currency used in current voucher; extracted from currency list. This field can be only selected by clicking mouse because the cursor does not appear.

Exchange Rate

If the currency is base currency, the exchange rate will be 1 (default). If the exchange rate is edited (the value changed), the system will process to re-calculate all values related to foreign currency and base currency

Memo

Interpreting content of the current transaction

Note: For description in account ledgers, description of an account is used for the corresponding account and vice versa. Thus, to record right description of account, remember to reverse the description of 2 accounts in the entry.

For example:
We have the corresponding relation between:
Credit 3348: Salary cost in Nov 2011: 1.000
Debit 6421-BP 1: Salary cost in Nov 2011- Dept. 1: 200
Debit 6421-BP 2: Salary cost in Nov 2011- Dept. 2: 400
Debit 6421-BP 3: Salary cost in Nov 2011- Dept. 3: 400

Then, description in account ledger 3348 will be:
Corresponding account 6421-BP 1: Salary cost in Nov 2011- Dept. 1: 200
Corresponding account 6421-BP 2: Salary cost in Nov 2011- Dept. 2: 400
Corresponding account 6421-BP 3: Salary cost in Nov 2011- Dept. 3: 400

Then, description in account ledger 6421 will be:
Corresponding account 3348: Salary cost in Nov 2011: 200
Corresponding account 3348: Salary cost in Nov 2011: 400
Corresponding account 3348: Salary cost in Nov 2011: 400

Status

There are 3 statuses:

    - No Action: the voucher has not been posted to any book;
    - Release Later: If there is VAT data, the invoice will be shown in VAT reports;
    - Release: the voucher has been posted to all related books.

Grid view - Detail

Account

Code of account; extracted from chart of accounts

FC Debit/ Credit

Debit/ credit arising amount in transaction currency. In the same row, if FC debit does not equal 0, FC credit must equal 0. If there is any change in FC debit amount, debit amount will be recalculated as FC debit amount multiplied by exchange rate. Negative number is allowed.

Debit/ Credit

Debit/ credit arising amount in base currency. If the transaction currency is base currency, these fields will not be shown. Debit/credit equals FC debit/credit multiplied by exchange rate, but editable.

Description

Interpreting content of the current transaction (See the example in Memo for reference)

Payment Term /Invoice Number/Invoice Date

These fields only display when the parameter “Advanced Payment Control” is "Yes".
If this parameter is "Yes", corresponding to following cases:
- Account input in column “Account” (Detail grid) is “AR Account” in list of parameter “Accounts Receivable” and “Debit” not equal 0 , AR Invoice will be posted to AR report;
- Account input in column “Account” (Detail grid) is “AP Account” in list of parameter “Accounts Payable” and “Credit” not equal 0 , AP Invoice will be posted to AP report.

Group

This field is used in case there is multi-debit and multi-credit entry in a voucher. It should be divided into groups of 1 debit/ multi-credit or multi-debit/ 1 credit. This helps in reports relating to corresponding accounts of an account. For These groups should be numbered in ascending order 1, 2, 3… for distinguishing purpose.

Note: Create entry groups in voucher
Enter debit and credit in the same field, the arising transaction will show if the account is debit or credit. Only 1-1 or 1- many entries are allowed in group, total debit must be equal to total credit in group.

Input VAT

Report Form

Used as filter condition in tax reports.
There are 3 forms:

    - 3- VAT Invoice;
    - 4- Goods, Services Purchased without Invoice;
    - 5- Sales Invoice.

Tax Type

This is VAT of goods, services purchased following State’s regulations; used to classify taxes in “Input VAT invoice listing (Form 01-2/GTGT)”.
With following option:

    - 1 - Goods, services used for VAT taxable business activities with revenue;
    - 2 - Goods, Services used for VAT non-taxable business activities;
    - 3 - Goods, Services used for both VAT taxable and non-taxable business activities;
    - 4 - Goods, services used for investment projects which meet tax deduction condition;
    - 5 - Goods, services not be declared in Form No. 01/GTGT.

Invoice Form

This "Invoice Form" is default as one in optional parameter but editable.
The "Invoice Form" will be used in tax reports by the following priority:

    - The "Invoice Form" input in this voucher is the first priority;
    - If this field is blank, the system will get the "Default Input Invoice Form" in optional parameter;
    - If the "Default Input Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Invoice Number/ Serial Number

Number and serial number of invoice from supplier

Invoice Date

The date when invoice is created by supplier

Supplier ID

Code to identify of goods, services supplier; extracted from customer list.

Supplier Name

Name of goods, services supplier; manually keyed in if there is no supplier ID

Address/ Supplier Tax Code

Address/ Tax code of the supplier

Goods, services

Name (group) of purchased goods, services

Amount

Total payment amount in transaction currency

Tax Code/ Tax Rate

VAT code; extracted from VAT list. After tax code is entered, the system will show the corresponding tax rate and tax account defined in VAT list.

Tax Account/ Corr. Tax Account

Tax account and corresponding account in VAT entry

FC Tax

Amount of tax in transaction currency; calculated on tax rate and invoice type, but editable

Tax Authority

Code of the Tax Authority where your business registers tax deduction; extracted from customer list.
This field is used in case a business has to register deduction in many Tax Authorities like construction companies with many projects in many towns.

Note/Job/Product/Department/MO/Loan Contract

More information to satisfy management requirements of each business

Some tasks in voucher entry

Import data from excel file

The system is enabled to get data from excel spreadsheet to save hours of manual effort.

For example: There are only account code, customer ID... in excel file. After importing data from the excel file, the general voucher will be auto-updated with account name, customer name...

The data importing process:

    - Enter General Voucher;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to upload data to the system, click the icon , select the file, the system will get data from this file to grid view of the voucher.

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists (Customer, Account, Job...). In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field.

Other notes:
- The function is only supported to get data for each voucher, used in case of frequently inputting the similar periodic transactions.
- You must have the "New" right to import new data to voucher detail grid;
- You must have the "Edit" right to edit data in voucher detail grid;
- It is not required to key in object name in lists, the system will auto update following their codes.

Import data from excel file (mass importing)

The same as importing data for each voucher with some following notes:

Notes of inputting data in template file:
- Unit: must be one of the assigned units with unit locking date before invoice date;
- Account: must be detailed account;
- Customer: required field if the account is foreign currency account;
- Currency: if voucher consists of many rows, the currency of the first row will be considered as currency of the voucher;
- Exchange Rate: to foreign currency vouchers, it is required to define exchange rate in detailed rows, the system will not generate exchange rate for details;
- Voucher Date: input in date format. Voucher date must be after data closing date and voucher locking date;
- Debit/Credit Amount: input in number format, 14 digits maximum and 2 decimal digits (standard format). The system will import the right amount input in the template file, not recalculate;
- Group (of double entries): there can be 1-1 or 1-multiple entries only, total debit and credit amount in a group must be balanced.

The system will give error message in some following cases:
- Data in column containing customer, account do not exist in the corresponding lists. Account is general account. In this case, the system will indicate the specific invalid cell;
- The required fields (Voucher Number, Unit...) are blank;
- String data is longer than the available length of field; the field length is the same as one manually input.

Other notes:
- Cannot import data after system data closing, unit or voucher locking date (the data closing date after the beginning input date);
- The current user must have the "New" right in "General Voucher" menu;
- The current user must have the "New" right in "Access Right by Unit" menu;
- Enable to import data for many units. If the data of the current unit has been closed, the system will give warning message when you click the icon; if data of other units has been closed, the system will also give warning message during importing process and indicate the specific invalid cell;
- Check the duplicate vouchers by number only (if available);
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.

See “Hints and Tips in voucher entry” for details of common tasks
