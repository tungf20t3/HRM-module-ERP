Income/Deduction by the Progressive Tax Tariff

Field descriptions – Format view

Pattern Code (Income Code)

Required field; extracted from Income/Deduction List (only income code having “Income Type” with “PIT Type” as Progressive -> defined in PIT Assessable Income List).

Voucher Type

1 – Arising in Period, 2 – Adjustment in Tax Finalization. Default as 1. Voucher type 2 – Adjustment in Tax Finalization: used in case there is any mistake in declaration of income or deduction after tax finalization. It’s used to create an adjustment voucher to increase/ decrease an income or deduction because the created vouchers cannot be edited after tax finalization.

Field descriptions – Grid view

Employee ID

Required field, extracted from “Employee Information” entry and only employees being “Resident” and having “Labor contract over 3 months” appear to look up.
To voucher type 2, only employees with “Authorization” (authorization for tax finalization) appear to look up.

Taxable Income

Manually input
Available to input only when income code with “Nature”=1 – Income (defined in Income/Deduction List)

Note:
Must not delete the employee ID having arising in “Income/Deduction by the Progressive Tax Tariff” or “Income by the Flat Rate Tariff”.
