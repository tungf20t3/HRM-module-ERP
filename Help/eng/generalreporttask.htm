Some common guide in reports

All report screens have some common tasks as follows:

Common tasks in date fields

To input the date in data entry, we can directly key in or select from the system by clicking on the icon at the right of date fields.

In the calendar screen, we can select the day only. To choose the month, use the left and right arrows. To choose the year, click on “month, year” area (at the middle of 2 arrows). Thus, at the date field, we can select by following order:
01. Date, month, year
02. Month, year
03. Year - 10 years
04. Month, year
05. Date, month, year


Filter condition – Fields selected from list

To look up an item from any list, key in three characters, the corresponding list will appear to select, or click on the magnifier icon to open lookup screen. When the list appear, process filter tasks as ones in list setting.

Note to filter condition selected from list: in some reports, it is required to key in the whole code (absolute condition); however, in some reports, it is required some wildcard characters (relative condition)
To learn more about wildcard characters, please search by the key word “wildcard” for online document.

- If you cannot remember both code and name, use quick search. Click the "" icon and then key the required characters in “Quick Search” box.
- If you remember code but name, you should search by field filter. Click "", icon and next click "" icon in code field, and key in the required characters.


Multi-item fields

To filter condition consisting of many items, open the lookup screen and select by checking in the desired items. When finishing the selection, click on 1 item before closing lookup screen; otherwise, there is no checked item selected.

Yes/No option or multiple choice fields

Press space bar to check. To some other options, click a to check.
To multi-option fields: 1, 2, 3 (group), key the number separately, the system will arrange.

For example: select report group in the order of group 1<-> group 2<-> group 3 (1/2/3); after that, delete group 2, you can key (1/0/2).

Select print form

There are usually 2 report print forms: “Standard Form” in base currency and “FC Form” in foreign currency. Some reports have more than 2 print forms.
The print form selected in the filter condition screen can be changed during printing process.

Note:
The order of report data in the sorted column will not be changed when report data are refreshed or re-filtered so that users do not have to sort again. This is useful for continuous reference.

Drill down

To view the detail voucher of a general data line in (summary) report, in data view screen, select the desired line, click the object link (

) to view the detail voucher.
The object link is specific to each report.

For example:
To sales summary by customer, the summarized object is customer; summary by item, the summarized object is item. When drilling down, the system shows the lowest level - voucher details.
To account summary report, when drilling down account details, the system ignores ledger level and showing list of vouchers with account arising.

Edit voucher in report

To edit voucher right in report, in data view screen, select the desired voucher, click “Voucher code” link icon, the voucher will be opened in another tab.

Pages in report

To move among previewed pages in report, click “Page 1, 2, 3, 4, …” to go to the desired page.
See “View list by page” in “Hints and tips in list setting” for reference.

Freeze columns

This function is used in case a report consists of many columns and you want to fix some columns.
Put the cursor at the column you want to fix, click “Freeze columns” icon. Then, when we use the horizontal scroll bar, the columns from the first one to the frozen one will not be scrolled or adjusted the width.
To unfreeze column, select the frozen columns and click “Freeze Columns” icon again.

Common information in report form definition

AI ERP is designed for users to self-define calculating way of articles in some reports from standard form to create specific ones.
To each report, the system supports specific definition and detail information. This part only presents some common information to avoid repetition in documents.

Print:

    - If 1- Print, this article will be shown in report.
    - If 0- Not Print, this article will be hidden.

Font Bond:

    - If 1- Bold, this article name will be in bold font
    - If 0- Regular, this article name will be in regular font.

Formula

The formula field is activated when the calculating way is 0 – Base on formula. The defined codes can be added or subtracted and must be put in square brackets [].

For example:
Article “1. Cash” coded “111” is sum of articles coded [11A]+[111B]+[111C]”
