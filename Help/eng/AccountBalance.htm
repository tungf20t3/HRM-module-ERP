Account Opening Balance

This function is used to input opening balance of account (if any) before using AI ERP. Opening date is the first date of using the system, declared in “Declare the beginning input date”.
In case the first date of using the system is different to the first date of fiscal year, this function is used to input the year-opening balance of account which will be shown in Balance Sheet articles at year-opening balance column. If the first date of using the system is the same to the first date of fiscal year, the opening balance will be used as year-opening balance to report.
To AR/AP accounts, year-opening balance can only be input if the first date of using the system is different to the first date of fiscal year. Opening balance can be input in “Customer Opening Balance” or “Supplier Opening Balance” function.
List of accounts with balance will be system-generated from chart of accounts, unit list. The system will ignore accounts without balance (defined in “List of the first characters of accounts without balance” parameter).

Entry screen

Field descriptions

Unit

Unit relating to account and balance

Account

The account of which balance is input

Debit Opening Balance

Opening balance of account in debit. If this account has credit balance, it will be deleted when the debit balance is input.

Credit Opening Balance

Opening balance of account in credit. If this account has debit balance, it will be deleted when the credit balance is input.

FC Debit Opening Balance

Opening balance of account in debit in foreign currency. If this account has credit balance in foreign currency, it will be deleted when the debit balance is input.

FC Credit Opening Balance

Opening balance of account in credit in foreign currency. If this account has debit balance in foreign currency, it will be deleted when the credit balance is input.

Debit Year-opening Balance

Year-opening balance of account in debit. It can be input only when this account has debit balance and the first date of using the system is different to the first date of fiscal year.

Credit Year-opening Balance

Year-opening balance of account in credit. It can be input only when this account has credit balance and the first date of using the system is different to the first date of fiscal year.

FC Debit Year-opening Balance

Year-opening balance of account in debit in foreign currency. It can be input only when this account has debit balance in foreign currency and the first date of using the system is different to the first date of fiscal year.

FC Credit Year-opening Balance

Year-opening balance of account in credit in foreign currency. It can be input only when this account has credit balance in foreign currency and the first date of using the system is different to the first date of fiscal year.

Import data from excel file

The system is enabled to import data from excel spreadsheet to save hours of manual effort.

The data importing process:

- Enter "Account Opening Balance" menu;
- Click the icon to download template file to put data in;
- After keying data in the spreadsheet, to up data to the system, click the icon , select the file, the system will get data from this file to opening balance file in the system.

Notes of inputting data in template file:
- Unit, Account: Unit is one of the assigned units; account is neither general account nor AR/AP account. The input unit, account code must be one existing in the list with the same length in add (edit) screen in list;
- Period-opening Debit Balance, Period-opening Credit Balance, Year-opening Debit Balance, Year-opening Credit Balance: number format, 14 digits maximum and 2 decimal digits (standard format);
- Period-opening FC Debit Balance, Period-opening FC Credit Balance, Year-opening FC Debit Balance, Year-opening FC Credit Balance: number format, 12 digits maximum and 2 decimal digits (standard format).

The system will give error message in some following cases:
- Data in column containing account code does not exist in chart of accounts. In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field which is the same as one manually input.

Other notes:
- The system will not get data of accounts without balance;
- Cannot import data after system or unit data closing (the data closing date after the beginning input date);
- The current user must have the 'Edit" right in "Account Opening Balance" function;
- The current user must have the 'Edit" right in "Access Right by Unit";
- Enable to import data for many units;
- If the beginning input date is the same as the first day of fiscal year, opening balance is year-opening balance;
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
