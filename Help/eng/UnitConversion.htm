Unit Conversion

Path: Modules → Inventory Management → List Maintenance → Unit Conversion
An item, in reality, can be received or issued by many units of measure depending on the business transaction. Therefore, it is required to have the conversion of UOM of each item by another one. In Receipt/Issue Transaction List, if you select to view physical/book quantity “with UOM”, the items in report will be shown with the UOM entered in voucher. On the contrary, they will be shown with the original UOM.

Field descriptions

Item

The item of which the unit of measure will be converted
An item can be controlled by many units of measure if there is check in “Multi-UOM” option in Item List.

UOM

The unit of measure converted from the original one; this UOM is now used in sales/purchase orders or actual receipt/issue only, not in inventory reports. In sales/purchase orders or receipt/issue entries, the original UOM is default. You have to select the converted UOM if necessary.

Coefficient

The number of items in original UOM equivalent to 1 item in conversion UOM

Example:
Base UOM of item 00007 (in item master) is "kg", it has also other UOM (pack) with coefficient (conversion rate: 1 pack=7 kgs).
This item has been inputed as 100 packs.
Whenever we view report:
- If choose "data type" with "invoice (physics)", then the result will be 700 (packs).
- If choose "data type" with "invoice" (physics) - UOM", then the result will be 100 (kgs).
