Cash Disbursement

Cash Disbursement Entry – Transaction code 1

Voucher type 1- “Disbursement detailed by Invoice” is used in case payment control is detailed by invoice and cash accountant also follows payment control by invoice from cash disbursement. When paying by invoice, it is required to define for which invoice the amount is paid. If paying for many invoices, the paid amount must be split to each invoice.
To find the required invoice, press "Ctrl+Insert” or click "” - "Insert” to create a new detail row. Then, key in the invoice number or click search icon.

Field Descriptions

Invoice No.

Number of invoices of supplier which haven’t been paid or partial paid by enterprise. Invoice number will be extracted from list of invoices of supplier which haven’t been paid.

Invoice Date

Created date in invoice

Debit Account

Debit account in account invoice; automatically extracted and cannot be entered in Disbursement.

FC Amount/ Amount

Current payment in currency of Cash Disbursement

Payment Amount

If the currency in bank credit advice is different from one in invoice, this field is enabled to input amount converted to currency in invoice. On the contrary, this field will not be displayed.

Note:
- In case transaction currency is different from base currency, input “FC Amount” or “Payment Amount” to record original value in ledger, amount in base currency will be input in “Amount” field;
- In case transaction currency is base currency, input amount in base currency in “Amount” field, original value in “Payment Amount”.

For example:
01. Invoice amount USD100, exchange rate VND20.000/USD, payment VND2.000.000: input “Payment Amount” as USD100, “Amount” as VND2.000.000,
02. Invoice amount USD100, exchange rate VND20.000/USD, payment USD100: input “FC Amount” as USD100, “Payment Amount” as USD100, “Amount” as VND2.000.000,
03. Invoice amount VND 1.000.000, payment VND 1.000.000: input “Amount” as VND1.000.000,
04. Invoice amount VND1.100.000, exchange rate VND20.000/USD, payment USD50: input “FC Amount” as USD50, “Payment Amount” as VND1.000.000, “Amount” as VND1.100.000

Description

Transaction reasons/ memo, posted to GL as Note.

Note:
In case of using voucher type 2, the disbursed amount can also be allocated to each invoice but the allocation is processed in Accounts Payable module. The disbursed vouchers in Cash Management and invoices (not written off) of current customers will be listed to allocate the disbursed amount to corresponding invoice.

Cash Disbursement Entry – Transaction code 2

Field Descriptions – Format view

Customer

Paying object. If Disbursement relates to payment, customer ID will be supplier, employee or customer ID. At that time, this disbursed amount will be recorded in payable detailed book. In case of direct disbursement when purchasing goods, services paying immediately, customer ID is only reference information or can be used for VAT voucher list in the future. Customer ID is extracted from Customer/Supplier List.

Receiver

Receiver is used as reference information in Disbursement. Receiver is automatically extracted from chosen customer ID if customer ID has declared “Partner” field and be editable. This field is declared to be used or not in “Transaction Entry” Screen.

Memo

Reference information in bank debit advice

Credit Account

Credit Account usually is account of cash, extracted from Chart of Accounts.

Voucher Type

Type of Disbursement. Different types of Disbursement will be used following specification’s properties. Types of Disbursement are:

    - 2 - “Disbursement to Customer”: for Cash Accounting Dept. to receive from customer following amount. Allocation of payment detailed by invoice (if any) is carried out by specific process department.
    - 3 - “Disbursement to Customers”: for receiving from one representative but controlling many other customers.
    - 5 – “FC Disbursement with Average Book Rate for transaction relating to foreign currency with monthly average book rate.
    - 9 - “Other”: for directly paying, this type is the most popular.

Default type is 2; however, it can be changed in “Transaction Entry” Screen.
Transaction code mustn’t be changed after entering information in grid view. If it is necessary to change, delete all detailed lines.

Note in applying exchange rate difference in selecting voucher type 2 or 5:
- Select type 2: require book exchange when creating voucher and print with calculated exchange rate;
- Select type 5: book exchange rate updated when calculating exchange rate, exchange rate when creating and printing voucher is temporarily used transaction exchange rate.
In receipt/payment voucher, book exchange rate of accounts in foreign currency will be applied to voucher and create the differences if any. Please refer to “Other guides in cash management module” for calculating formula.

Voucher Number / Voucher Date / Posting Date

Number of Disbursement. It is automatically increased 1 when a new voucher entered but it can be changed by user.
Disbursement created date is used only for reference. Program will use GL Date in calculation. GL date must be greater than closing date.

Currency / Book Rate / Ex. Rate

Code of transaction currency of current voucher is selected from Currency List.
Ex. Rate is exchange rate in real transaction, extracted from “FC Exchange List”, bases on exchange rate in latest date before current bank credit advice. If FC code is GL currency, exchange rate will be 1.
Cursor can not be used in this field, click mouse to select type of currency in transaction.

Payment Control

This option is applied to loan receipt as an invoice (in Accounts Receivable) and supports in receivable aging analysing and payment detailed by loan item reports.
The system will auto-copy voucher number, GL date in format view (general information) to “Control No." & "Control Date” fields in “Payment Control” tab and editable. “Dr. Term” is extracted from Customer/Supplier List.

Edit Book Ex. Rate

This option is used to edit book exchange rate. You can change the system-generated book exchange rate into another one. When you run exchange rate revaluation at the end of month, the system will not update exchange rate in this voucher.

Amount

Amount in GL currency. If transaction currency is the same as GL currency, this field will not be displayed. Value is equal to FC amount multiplied by exchange rate but can be changed.

Tax

Tax amount in GL currency. Tax Amount [in GL currency]=Amount [in GL currency] * Tax Rate (corresponding with entered tax code). This field is editable. If transaction currency is the same as GL currency, this field will not appear.

Payment

Total amount in GL currency will be calculated by different formulas depending on tax calculating methods.

Status

There are 3 statuses:

    - No Action: the voucher has not been posted to any book;
    - Release Later: If there is VAT data, the invoice will be shown in VAT reports. If this is AR/AP invoice, it will be shown in AR/AP by invoice reports;
    - Release: the voucher has been posted to all related books.

Grid view

Debit Account

Debit account in transaction; extracted from Chart of Accounts.

Description

Transaction description; posted to GL.

(xxx) Amount

Xxx – code of transaction currency. This is amount in transaction currency. If amount is in foreign currency, it will be calculated by formula [amount xxx * ex. rate]. However, if real amount of transaction is equal to [xxx * Ex. Rate], you can directly enter value in Amount field.

Job

This is an optional field used in case of disbursement by job.

Cash Disbursement Entry – Transaction code 5

Field Descriptions

The same as ones in voucher transaction code 2; however, do not input exchange rate which will be periodically updated when book rate is calculated.

Cash Disbursement Entry – Transaction code 9

Field Descriptions

The same as ones in voucher transaction code 2; however, input VAT information

Report Form

Filter condition when viewing reports in Tax Reports module.
With following forms:

    - 3 - VAT invoice;
    - 4 - Goods, services purchase without invoice;
    - 5 - Sales invoice.

Tax Type

This is VAT of goods, services purchased following State’s regulations; used to classify taxes in “Input VAT invoice listing (Form 01-2/GTGT)”.
With following options:

    - 1 - Goods, services used for VAT taxable business activities with revenue;
    - 2 - Goods, Services used for VAT non-taxable business activities;
    - 3 - Goods, Services used for both VAT taxable and non-taxable business activities;
    - 4 - Goods, services used for investment projects which meet tax deduction condition;
    - 5 - Goods, services not be declared in Form No. 01/GTGT.

Invoice Form

This "Invoice Form" is default as one in optional parameter but editable.
The "Invoice Form" will be used in tax reports by the following priority:

    - The "Invoice Form" input in this voucher is the first priority;
    - If this field is blank, the system will get the "Default Input Invoice Form" in optional parameter;
    - If the "Default Input Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Invoice Number / Serial Number

Number and serial number of invoice of supplier

Invoice Date

The date the invoice of supplier is recorded

Supplier ID

Code identify the goods, services supplier, default as Customer ID in Bank Debit Advice but editable. This ID will be extracted from Customer List. After entering ID, program will update Name, Address, and Tax Code of suppliers.
ID may not be entered if you want to free enter these fields in case of retail suppliers.

Supplier Name

Name of goods and services suppliers can be directly entered if there is no tax code.

Supplier Tax Code

Tax code of the supplier

Goods - Services

Name (group) of purchased goods, services.

Tax Code / Tax Rate

VAT tax code is extracted from Tax Rate List. Tax rate and Tax account declared in Tax Code List will be defined after tax code entered.

(xxx)Tax

Tax amount in transaction currency is calculated basing on tax rate and type of invoice and can be edited.

Tax Authority

Code of tax office where enterprise registers tax declaration; extracted from Customer List.
This field is for of controlling many different tax offices and usually occurs to construction companies having work wide spread in many towns.

Amount

Total payment amount in transaction currency

Note:
- In case of transaction code 2, paid amount can be allocated to each invoice but the allocation can be carried out in Accounts Payable. The system will list bank debit advice notes disbursed in Cash Management and unpaid invoices of customer to allocate amount disbursed to correlative invoices.


Cash Disbursement Entry – Transaction code 3

Field Descriptions

“Pay for many Customers” type is the same as “Pay for 1 Customer” type. The only difference is that you can enter different customer ID in each detail row. Besides, to foreign currency transactions in this transaction type, the system will record and account by transaction exchange rate only, not create exchange rate difference

Some tasks in voucher entry (voucher type 3)

Import data from excel file

The system is enabled to get data from excel spreadsheet to save hours of manual effort.

The data importing process:

    - Enter the voucher;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to upload data to the system, click the icon , select the file, the system will get data from this file to grid view of the voucher.

Notes of inputting data in template file:
- Customer, Debit Account, Job, Department, MO, Product, Contract, Expense, Loan Contract: The input code must be one existing in corresponding list with the same length in add (edit) screen in list;
- Account Name, Customer Name, Description: manually input with the same length in add (edit) screen in lists - Amount: number format, 14 digits maximum and 2 decimal digits (standard format).

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists (Customer, Account, Job...). In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field.

Other notes:
- The function is only supported to get data for each voucher, used in case of frequently inputting the similar periodic transactions.
- It is not required to keying object name in lists, the system will auto update following their codes.

“Payment” information

This is used to input “Payment Term”, “Voucher Number” and “Voucher Date” in case of voucher type 2 with active “Payment Control” option.
This information will be saved in payment details and processed as an invoice.

“Include” information

This is used to input number of enclosed documents and descriptions which will be used in voucher printing.

In receipt/payment voucher, book exchange rate of accounts in foreign currency will be applied to voucher and create the differences if any. Please refer to “Other guides in cash management module” for calculating formula.

Some tasks in voucher entry

Import data from excel file (mass importing)

The system is enabled to get data from excel spreadsheet to save hours of manual effort.

The data importing process:

    - Enter "Bank Debit Advice";
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to upload data to the system, click the icon , select the file, the system will import data from this file.

Notes of inputting data in template file:
- Unit, Debit Account, Credit Account, Customer: account must be detailed accounts; unit must be assigned units with unit locking date before invoice date;
- Voucher Number: input in form of string, 12 characters maximum;
- Voucher Date: input in date format. Voucher date must be after data closing date, unit and voucher locking date;
- Voucher Type: only voucher type 2, 3 and 9;
- Amount: input in number format, 14 digits maximum and 2 decimal digits (standard format);
- Customer, Credit Account, Job, Department, MO, Product, Contract, Expense, Loan Contract: The input code must be one existing in corresponding list with the same length in add (edit) screen in list.

The system will give error message in some following cases:
- Data in column containing customer, account do not exist in the corresponding lists. Account is general account. In this case, the system will indicate the specific invalid cell;
- The required fields (Voucher Number, Unit...) are blank;
- String data is longer than the available length of field; the field length is the same as one manually input;
- There is tax information in spreadsheet to Bank Debit Advice voucher type 2; it's because there is no "Tax" tab in this voucher entry;
- In case the "Check for duplicate invoices in Input VAT Vouchers" parameter is "Yes"; if the voucher has the same serial number, invoice number and invoice date with the available input VAT vouchers, the system will give error message and disable to import.

Other notes:
- Cannot import data after system data closing, unit or voucher locking date (the data closing date after the beginning input date);
- The current user must have the "New" right in "Bank Credit Advice" menu;
- The current user must have the "New" right in "Access Right by Unit" menu;
- Check the duplicate vouchers by number only (if available);
- In order to import a voucher with many grid rows, you can input many rows in spreadsheet with the same voucher number and date. The system will create one voucher with these grid rows;
- The system is not enabled to import many input VAT invoices by each item (goods or services), you can only manually input or adjust the imported data;
- Enable to import data in base currency only;
- Enable to import voucher in "No Action" status only; you can edit the imported voucher to post to ledger;
- It is not required to keying object name in lists, the system will auto update following their codes;
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
