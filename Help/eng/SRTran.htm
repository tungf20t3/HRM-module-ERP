Sales Return

Entry screen

Field descriptions - format view

Customer

Customer ID and name, extracted from Customer/Supplier List, is usually used in debt control in case of Issue related to debt transactions as lending items from other units. In case transactions which are not related to debt control, this information is used to control internal objects as production line, shop floor…

Delivered by

If “contact person of trans” field is declared in “voucher book list”, this field will be extracted from supplier list by default, but editable.

Transaction Code

There is only one type “1 – Sales Return” now.

Credit Account

Credit account is corresponding with sales return account in sales return entry. This is a required field.
Credit Account is extracted from supplier list at the “default Account” field, but editable.

Invoice Number / Serial Number / Invoice Date

Number / Serial number / Created Date of sale return invoice created by customer, used in output VAT list (VAT 02).

Invoice Form

This "Invoice Form" is default as one in optional parameter but editable.
The "Invoice Form" will be used in tax reports by the following priority:

    - The "Invoice Form" input in this voucher is the first priority;
    - If this field is blank, the system will get the "Default Output Invoice Form" in optional parameter;
    - If the "Default Output Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Memo

Description of arising transaction content

Voucher Number

Number of sales return voucher; you can enter or the program automatically gets from declaration in system.

Invoice Date / Posting Date

The date the voucher is created and the date the voucher is posted to GL.

Ex. Rate

Foreign currency exchange rate, used in balance tracking. Select FC code, the program will return to exchange rate in latest date, but it is editable.

Status

There are 3 statuses:

    - No Action: the voucher has not been posted to any book;
    - Release Later: If there is VAT data, the invoice will be shown in VAT reports. If this is AR/AP invoice, it will be shown in AR/AP by invoice reports;
    - Release: the voucher has been posted to all related books.

Tax Code / Tax Account

Code and account of deducted output VAT; extracted from “Tax Rate List”.

Tax Authority

State Tax Authority, where company registers tax declaration
If tax account is AR/AP Account, Tax Authority is a required field.

Update SO Status

You can update SO with returned quantity. If you choose this option, the program will update ordered quantity and SO status corresponding with quantity of returned goods.

Field descriptions - “Detail” tab

Item code

Code of items returned by customer

Promotion

See “Field descriptions” of grid view in “Sales Invoice” for details

Site / Location

Site / Location where sales returned goods stored; in case company has stocks controlled by location, shelves… data are entered and automatically returned to Pick List or Sales Invoice if they are extracted from Pick List or Sales Invoice. However, they can be edited.
Site / location codes can also be directly extracted from Site / Location List.

Lot / Quantity

Lot code and quantity of sales return goods will be automatically returned to Pick List or Sales Invoice if data are extracted from Pick List or Sales Invoice.
Besides cases above, data are directly entered in screen; lot code is extracted from Lot Master.

Average Cost

This option is used to define cost price of goods restored in case issued cost price is applied when calculating average price. If you don’t select this option, you have to enter price directly in screen.

Sales Price/ Amount/ FC Sale Price/ FC Amount/ Unit Cost/ Capital/ FC Unit Cost / FC Capital

Sales Price, Cost Price, Receivable/ Payable Amount and Capital in currency and foreign currency type in GL. Data of foreign currency are entered depending on FC code chosen in format view.

Return Account / Inventory Account / COGS Account

GL Accounts in Sales Return Entry; extracted from “Item Master”. Return Acct and Cost Price Acct can be changed. These are required fields.

Invoice No./ Line / Pick List No. / Line

Number of invoice, pick list and correlative line number in invoice, pick list; values in these fields appear only when data are extracted from Invoice or Pick List.

Job / Department / MO / Product / Contract / Expense / Loan Contract

More fields for advanced management. It depends on management matter of specific organizer.

Field descriptions - “Other Information” tab

Item Group

Item name in Output VAT List. If you do not input data in this field, the system will auto get name of the item with greatest sales figures.

"Post to Input VAT" option

When this option is checked, the system will check tax information and post to Input VAT listing. Otherwise, tax information of this voucher will be posted to Output VAT listing (as default).

For details of output VAT data entry, refer to Output VAT Invoice Entry

Some common task

Create Return - Extract data from Invoice

Create a new Return, click on function “ - Extract Data", select “Extract from Invoice” function and type start date. Lookup and choose Invoices, type return quantity then exit Invoice List screen. Data in selected rows in this list will be system-updated to Return.

Create Return - Extract data from Pick List

Create a new Return, click on function “ - Extract Data", select “Extract from Pick List” function and type start date. Lookup and choose Pick List, type return quantity then exit Pick List screen. Data in selected rows in this list will be system-updated to Return.

See “Hints and Tips in voucher” for details of common tasks
