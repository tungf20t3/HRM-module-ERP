Input VAT Invoice Entry

Path: Modules → General → Tax → Input VAT Invoice Entry
Beside available VAT tabs in data entry screens, “Input VAT Invoice Entry” menu is used in some special cases that VAT invoices are entered by staffs who are responsible for tax, instead of professional accountants, because of work assignment in Accounting department.



Other notes

Some notes in input VAT voucher entry:

    - Input VAT vouchers can be listed in many input VAT reports, so the corresponding report form should be defined in input VAT entry screens by select the code in “Report Form” field.
    - For correct and automatic VAT reports, the output VAT account must be divided into sub-accounts by the principles mentioned in Organization of Chart of Accounts.

Input VAT accounting must obey following order:

    - Purchase goods, services with VAT: Debit on account 133 / Credit on account 331, 111, 112, 141, …
    - At the end of period, close and transfer the un-deducted VAT (if any): Debit on account 142, 632…/ Credit on account 133
    - If the total payable output VAT amount is greater than the deductible input one, transfer the total debit amount of account 133 to account 33311; on the contrary, if the total output VAT amount is less than the input one, transfer the total credit amount of account 33311 to account 133.

… to achieve the correct “VAT of purchased goods, services in period” article in Tax Declaration.
In case of return to supplier, input VAT accounting must obey the following order:

    - When issuing to return to supplier: Debit on account 331/ Credit on account 13313;
    - At the end of month, closing: Debit on account 13313/ Credit on account 13311 …

… to create VAT detail book of goods returned to supplier and post the returned goods VAT articles in some tax reports.

The "Invoice Form" field is default as one in optional parameter but editable.
This "Invoice Form" will be used in tax reports by the following priority:
- The "Invoice Form" input in voucher is the first priority;
- If this field is not defined in voucher, the system will get the "Default Input Invoice Form" in optional parameter;
- If the "Default Input Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.
