Sales Order

Field descriptions - format view

Customer

Extracted from “Supplier/Customer List”, is a required field.
Entered when you create new voucher and unchangeable during data entry. If you want to change, delete all details and enter other ID.

Transaction

Select 1 – SO (default), 2 - Contract or 3 – Immediate Invoice. This field is a required field.
Transaction code will be selected according to sales process. Company selling by SO use only “Sales Order” or “Contract”. On the contrary, to company selling retail, it is required to input data of sold goods into a Sales Order (to manage sales turnover of Sales Dept.)- AI ERP defined these SO as “Immediate Invoice”.
Transaction code of voucher is only entered first when creating voucher, mustn’t be changed in adjusting voucher. If it is necessary to change, you must delete voucher and enter other code

Credit Term

Credit term of sales order or contract; extracted from “Payment Term List” but optional field

Effective Date

The expiry of contract; entered only when transaction code is 2- Contract.

SO No.

SO/Contract number; automatically numbered by the system as declaration in voucher list system or entered by user.

SO Date

The date the SO/Contract is created

Ex. Rate

Select FC Code and declare exchange rate in SO/Contract creating date.

Blanket No.

Contract number is valid only when SO is created from contract.
Confirmed long-term order by a buyer to a seller for supplying specified good(s) or service(s), for a fixed period or in a fixed quantity, at agreed-on prices or pricing method. After its acceptance by the supplier, supplies may be made against it periodically, on as-and-when-required basis, or as specified in the order, without calling for new purchase orders.

Field descriptions - “Details” Tab

Item Code

Directly extracted from Item Master (to SO with Immediate Invoice) or from Quotation.

UOM

Default UOM in Item Master but can be changed by conversion of units of measurement in some special cases.

Site ID

Directly extracted from Site List (to items with immediate invoice) or from Quotation.

Quantity

Quantity of item ordered (is a required field).

FC Sales Price

Purchase price of item in foreign currency, appears only when currency field in format view is foreign currency. In case sales with discount which is not defined to “Apply to Extended Amount”, sales price will be equal to base price minus discount.

FC Amount

Arising amount in foreign currency. If there is adjustment of FC amount, amount will be equal to FC amount multiplied by exchange rate.

FC Discount

Discount amount in case sales with discount which is defined to “Apply to Extended Amount”.

Sales Price

Sales price of item in GL currency. If sales order with transaction code 3 – Immediate Invoicing, sales price will be equal to base price. If sales price is extracted from quotation, it will be equal to quoted price in quotation. In case sales with discount which is not defined to “Apply to Extended Amount”, sales price will be equal to base price minus discount

Discount

Discount amount in case sales with discount which is defined to “Apply to Extended Amount”.

Tax Code/ Tax Rate

VAT Code and VAT Rate of item; extracted from Item Master or selected from Tax Code List.

Due Date

Date when delivering goods, extracted from Quotation

Tax

Amount of VAT in VND; used to replace Tax (USD) field in format view - VND.

Issued Qty

Quantity of item issued to SO; be displayed when items in SO are created Pick List. It is used to check in SO, un-editable.

Invoice Qty

Invoiced quantity of item to SO; be displayed when items in SO are created Sales Invoice. It is used to check in SO, un-editable.

Job

Code of job; used to control a sale process from this quotation. If next steps are created from data of this quotation, job will be automatically enclosed.

Field descriptions - “Shipping Information” Tab

Ship-to / Address

Place/ address receiving goods, extracted from Delivery Address List. This field can be entered only when customer’s address is declared in Delivery Address List.

Ship via

Means of shipping goods; extracted from Transport Type List

Field descriptions - “Charges” Tab

“Charges” Tab in Contract is declared as information for reference, used to declare predicted charges which can be arise. There is no interaction to other next steps.

Charge Code

Charge code attached to Contract, extracted from Charge List. This is an optional field.

Amount

Amount of charge related to sales if SO is carried out. Amount of each charge code will be allocated to each item in quotation by specific formula of each code. Allocation formula depends on Charge List creating.

Field descriptions - “Customer Information” Tab

Customer Name / Address / Telephone / Fax

Information of customer; extracted from Customer/Supplier List, used for reference.

Attachments

The program allows users to add attachment files to a voucher. See more "Working with attachments".

Field descriptions - “Other Information” Tab

Sales Employee

Employee in charge of SO/Contract; extracted from Sales Employee List

Inv. No. / Inv. Date

Sales Dept. enters number and date of invoice into the program in case consumption dept. can invoice but can not create entry.

Blanket Date

The date the contract is created; appears when SO is extracted from contract.

Effective date

Effective date of contract; appears when SO is extracted from contract.

Status

Contract status; appears when SO is extracted from contract.

Task

Extract Data from Blanket Order

Click on " - Extract Data" icon, then choose "Extract Data from Blanket Order". Type "Date From", lookup and specify the Blanket Order Number.

Extract Data from Quotation

Click on " - Extract Data" icon, then choose "Extract Data from Quotation". Type "Date From", lookup and specify the Quotation Number.

Working with attachments

The program allows users to add attachment files to a voucher. To add an attachment file, click on the icon then choose file. To download an attached file, click on the icon . To remove an attached file on a voucher, click on the icon .
Program allows attachments up to 10 MB, the maximum size of each file can be declared. (Refer this function to counsellors)
