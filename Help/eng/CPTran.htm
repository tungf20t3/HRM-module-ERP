Bank Debit Advice

Bank Debit Advice Entry – Voucher Type 2, 5 and 9

Field descriptions- Format View

Customer

Customer is payable object, extracted from Customer List. If bank debit advice relates to payment, customer will be supplier, employee or customer. At that time, this disbursed amount will be recorded in payable detailed book. In case of direct disbursement when purchasing goods, services immediately paying, customer is only reference information or used for VAT voucher list.

Address/ Receiver

Address and receiver are reference information in bank debit advice, extracted from Customer List/ Partner field (if defined) in accordance with the customer above.

Memo

This is reference information in bank debit advice.

Voucher Type

Bank Debit Advice types are differently used in accordance with each transaction’s properties. Including:

    - 1- “Disbursement detailed by invoice”: used in case payment control is detailed by invoice and cash accountant also follows payment control by invoice from bank credit advice.
    - 2- “Disbursement to Customer”: used in case cash accountant pay for supplier by approved amount only. The allocation of this amount by invoice (if any) is carried out by specific payment processing section.
    - 3- “Disbursement to Customers”: used in case payment is paid for a representative but controlled by many suppliers.
    - 5 – “FC Disbursement with Average Book Rate used in case of arising transaction relating to foreign currency with monthly average book exchange rate.
    - 9- “Other”: used in case of directly disbursement. This type is most popular, specially used for voucher with VAT invoice.

Credit Account

Credit account is bank deposits account, extracted from Chart of Accounts.

Voucher Number/ Voucher Date/ Posting Date

Number of bank debit advice; it is system-generated increasing 1 when a new voucher is created but editable.
The date when the bank debit advice is created is reference information only. In calculation, the system will use posting date. The posting date must be after the closing date.

Currency/ Ex. Rate/ Book Rate

Transaction currency of the current voucher; extracted from Currency List.
Book exchange rate will be system-calculated by method selected in Chart of Accounts. In case currency is base currency, exchange rate will be 1.

Note:
Exchange rate and book rate will be active or inactive in each specific voucher type: - Type 1/2/5: Book rate;
- Type 3/9: Exchange rate.

Status

There are 3 statuses:

    - No Action: the voucher has not been posted to any book;
    - Release Later: If there is VAT data, the invoice will be shown in VAT reports. If this is AR/AP invoice, it will be shown in AR/AP by invoice reports;
    - Release: the voucher has been posted to all related books.

Detail

Debit Account

Credit account in arising transaction; extracted from Chart of Accounts.

Description

Content of arising transaction; will be posted to GL as voucher description.

(Xxx) Amount

Xxx is transaction currency and amount is arising amount of transaction in this currency. In case of foreign currency, the amount will be equal to xxx amount multiplied by exchange rate. If the transaction amount is already equal to xxx amount multiplied by exchange rate, directly input in this field.

Amount

Arising amount in base currency. If transaction currency is base currency, this field will disappear. This amount will be equal to FC amount multiplied by book rate but editable.

Tax

Report Form

Used as filter condition in tax reports.

    There are 3 forms:
    - 3- VAT Invoice;
    - 4- Goods, Services Purchased without Invoice;
    - 5- Sales Invoice.

Tax Type

This is VAT of goods, services purchased following State’s regulations; used to classify taxes in “Input VAT invoice listing (Form 01-2/GTGT)”.
With following options:

    - 1 - Goods, services used for VAT taxable business activities with revenue;
    - 2 - Goods, Services used for VAT non-taxable business activities;
    - 3 - Goods, Services used for both VAT taxable and non-taxable business activities;
    - 4 - Goods, services used for investment projects which meet tax deduction condition;
    - 5 - Goods, services not be declared in Form No. 01/GTGT.

Invoice Form

This "Invoice Form" is default as one in optional parameter but editable.
The "Invoice Form" will be used in tax reports by the following priority:

    - The "Invoice Form" input in this voucher is the first priority;
    - If this field is blank, the system will get the "Default Input Invoice Form" in optional parameter;
    - If the "Default Input Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Invoice Number/ Serial Number

Number and serial number of invoice from supplier

Invoice Date

The date when invoice is created by supplier

Supplier ID

Code to identify of goods, services supplier; extracted from Customer List.

Supplier Name

Name of goods, services supplier; manually keyed in if there is no supplier ID

Supplier Tax Code

Tax code of the supplier

Goods, Services

Name (group) of purchased goods, services

Tax Authority

Code of the Tax Authority where your business registers tax deduction; extracted from Customer List.
This field is used in case a business has to register deduction in many Tax Authorities like construction companies with many projects in many towns.

Other

Payment Control

This option is applied to loan, advance disbursement as an invoice (in Accounts Receivable module) and supports in receivable aging analyzing and payment detailed by loan, advance item reports.
Voucher number, posting date in format view will be auto-copied to voucher number, voucher date in “Payment” information but editable. “Payment Term” is extracted from Payment Term List.

Edit Book Exchange Rate

This option is used to edit book exchange rate of cash account. You can change system-generated book exchange rate into another one. When you run exchange rate revaluation at the end of month, the system will not update exchange rate in these vouchers.

Note:
- Only VAT vouchers are input in bank debit advice type 9 because it is used for immediately disbursement.
- In case of voucher type 2, disbursed amount can also be allocated to invoices but it is processed in Accounts Payable module. Disbursed debit notes and unpaid invoices relating to current customer will be listed so that the allocation can be processed to required invoices.

Bank Debit Advice Entry – Voucher Type 1

Voucher type 1- “Disbursement detailed by invoice” used in case payment control is detailed by invoice and cash accountant also follows payment control by invoice from bank credit advice. When disbursing by invoice, it is required to define to which invoice the amount is disbursed. If disbursing to many invoices, the disbursed amount must be split to each invoice.
To find the required invoice, press “Ctrl+Insert” or click “” – “Insert” to create a new detail row. Then, key in the invoice number or click search icon.

Bank Debit Advice Entry – Voucher Type 3

“Disbursement to Customer” type is the same as “Disbursement to Customers” one. The only difference is that you can enter different customer in each detail row. Besides, to foreign currency transactions using this voucher type, the system will record and account by transaction exchange rate only, not create exchange rate difference.

Some tasks in voucher entry (voucher type 3)

Import data from excel file

The system is enabled to get data from excel spreadsheet to save hours of manual effort.

The data importing process:

    - Enter the voucher;
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to upload data to the system, click the icon , select the file, the system will get data from this file to grid view of the voucher.

Notes of inputting data in template file:
- Customer, Debit Account, Job, Department, MO, Product, Contract, Expense, Loan Contract: The input code must be one existing in corresponding list with the same length in add (edit) screen in list;
- Account Name, Customer Name, Description: manually input with the same length in add (edit) screen in lists;
- Amount: number format, 14 digits maximum and 2 decimal digits (standard format).

The system will give error message in some following cases:
- Data in columns containing codes of related objects do not exist in current lists (Customer, Account, Job...). In this case, the system will indicate the specific invalid cell;
- String data is longer than the available length of field.

Other notes:
- The function is only supported to get data for each voucher, used in case of frequently inputting the similar periodic transactions.
- It is not required to keying object name in lists, the system will auto update following their codes.

“Payment Order” information

This is used to input number of enclosed documents and descriptions which will be used in voucher printing. Including: Company, Account No., Bank, Province/City and Memo.

“Payment” information

This is used to input “Payment Term”, “Voucher Number” and “Voucher Date” in case of voucher type 2 with active “Payment Control” option.
This information will be saved in payment details and processed as an invoice.

In receipt, disbursement vouchers, book exchange rate of foreign currency accounts will be used, see “Other guides in cash management module”.

Some tasks in voucher entry

Import data from excel file (mass importing)

The system is enabled to get data from excel spreadsheet to save hours of manual effort.

The data importing process:

    - Enter "Bank Debit Advice";
    - Click the icon to download template file to put data in;
    - After keying data in the spreadsheet, to upload data to the system, click the icon , select the file, the system will import data from this file.

Notes of inputting data in template file:
- Unit, Debit Account, Credit Account, Customer: account must be detailed accounts; unit must be assigned units with unit locking date before invoice date;
- Voucher Number: input in form of string, 12 characters maximum;
- Voucher Date: input in date format. Voucher date must be after data closing date, unit and voucher locking date;
- Voucher Type: only voucher type 2, 3 and 9;
- Amount: input in number format, 14 digits maximum and 2 decimal digits (standard format);
- Customer, Credit Account, Job, Department, MO, Product, Contract, Expense, Loan Contract: The input code must be one existing in corresponding list with the same length in add (edit) screen in list.

The system will give error message in some following cases:
- Data in column containing customer, account do not exist in the corresponding lists. Account is general account. In this case, the system will indicate the specific invalid cell;
- The required fields (Voucher Number, Unit...) are blank;
- String data is longer than the available length of field; the field length is the same as one manually input;
- There is tax information in spreadsheet to Bank Debit Advice voucher type 2; it's because there is no "Tax" tab in this voucher entry;
- In case the "Check for duplicate invoices in Input VAT Vouchers" parameter is "Yes"; if the voucher has the same serial number, invoice number and invoice date with the available input VAT vouchers, the system will give error message and disable to import.

Other notes:
- Cannot import data after system data closing, unit or voucher locking date (the data closing date after the beginning input date);
- The current user must have the "New" right in "Bank Credit Advice" menu;
- The current user must have the "New" right in "Access Right by Unit" menu;
- Check the duplicate vouchers by number only (if available);
- In order to import a voucher with many grid rows, you can input many rows in spreadsheet with the same voucher number and date. The system will create one voucher with these grid rows;
- The system is not enabled to import many input VAT invoices by each item (goods or services), you can only manually input or adjust the imported data;
- Enable to import data in base currency only;
- Enable to import voucher in "No Action" status only; you can edit the imported voucher to post to ledger;
- It is not required to keying object name in lists, the system will auto update following their codes;
- Cannot import data from spreadsheet with macro;
- Only support for spreadsheet in Excel 2007 or higher.
