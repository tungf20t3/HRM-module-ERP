Discount Invoice

This voucher is used to give discount on created sales invoices.
This is also used in case amount in issued invoice is greater than amount of goods, services in delivery and customer requires an invoice for the adjustment.

Field descriptions – format view

Customer

Customer ID, extracted from Customer/Supplier List, is usually used in debt control in case of Issue related to debt transactions as lending items from other units. In case transactions which are not related to debt control, this information is used to control internal objects as production line, shop floor…

Buyer

Representative of customer; extracted from Customer List (“Partner” field) in accordance with customer ID or manually entered. This field is defined to be used or not in “Voucher information”.

Transaction Code

There is only one type “1 – Discount” now.

Credit Account

Credit account is receivable account, reducing receivable from customers.

Invoice Number

Number of the invoice being discounted; extracted following the selected invoice

Invoice Date

Date of the invoice being discounted; extracted following the selected invoice

Invoice Serial Number

Serial number of the invoice being discounted; extracted following the selected invoice

Memo

More information about arising transaction

Voucher Number

Number of discount invoice. In case of using voucher book, this number will be auto-increased 1 when a new voucher is created but editable. The system will check the number duplicate when saving.

- If the invoice is numbered by voucher book which has been defined the "Invoice Form", this "Invoice Form" will be used in tax reports.
- If the invoice is numbered by voucher book which has not been defined the "Invoice Form", the system will get the "Default Output Invoice Form" in optional parameter.
- If the "Default Output Invoice Form" parameter is also blank, there will be no "Invoice Form" in tax reports.

Serial Number

Serial number of invoice; user-defined

Posting Date

The date the voucher is posted; used in calculating and default as created date. GL date must be greater than closing date. This field can be updated only when you click mouse.

Ex. Rate

If transaction currency is the same as GL currency, exchange rate will be 1 and not be updated. If exchange rate is changed, the program will recalculate value in foreign and GL currency.

Status

There are 3 statuses:

    - No Action: the voucher has not been posted to any book;
    - Release Later: If there is VAT data, the invoice will be shown in VAT reports. If this is AR/AP invoice, it will be shown in AR/AP by invoice reports;
    - Release: the voucher has been posted to all related books.

Field descriptions – Grid view

Debit Account

Code of discount account

Amount

Discount amount

Tax Code

Code of tax rate. If this field is blank, “Tax rate”, “Tax Account” and “Tax” fields will not be generated.

Tax Rate

Tax rate appears after tax code is entered; auto-displayed in accordance with tax code.

Tax Account

Tax account appears after tax code is entered. When you give enough required information, the system will make an entry: Debit on 5321, Debit on 33313 – Credit on 131111.

Tax

Tax amount in transaction currency. This value is equal to amount in foreign currency (FC) multiplied by tax rate, but editable. If there is any change in tax amount in FC, tax amount will be equal to FC tax amount multiplied by exchange rate. Relating to output VAT, sales turnover and payable VAT amount are entered in negative number in output VAT invoice listing. Invoice number is number in discount invoice, number of the issued sales invoice is recorded in note.

Tax Authority

See “Field descriptions” of grid view in “Sales Invoice” for details

Description

Description about the transaction

Field descriptions – “Allocation” tab

Select Invoice

Click “ - Select Invoice” button to select issued sales invoices to reduce sales turnover, tax amount in sales reports by item. Type customer code, then click on button to search invoice will be allocated.

Note:
If you want to control payment by invoice, this voucher is used to adjust receivable amount of issued invoices for detail receivable reporting by invoice. This adjustment occurs in “Accounts Receivable/ Date Entry/ Upset Unapplied Transaction”.
